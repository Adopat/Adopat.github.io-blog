    # 计算机网络

    # 1. 网络体系结构

    # 1.1 网络体系结构概念

    网络体系结构(Network Architecture)是指将计算机网络划分为不同层次的体系结构，每一层都有特定的功能和协议。常见的网络体系结构包括OSI参考模型、TCP/IP模型等。

    # 1.2 网络体系结构分类

    # 1.2.1 OSI参考模型

    OSI参考模型将计算机网络分为七个层次，从下到上分别是物理层、数据链路层、网络层、传输层、会话层、表示层和应用层。每一层都有特定的功能和协议。

    • 物理层

      • 概念

        物理层是OSI (opens new window)的第一层，它虽然处于最底层，却是整个开放系统的基础。物理层为设备之间的数据通信 (opens new window)提供传输媒体及互连设备，为数据传输 (opens new window)提供可靠的环境。如果您想要用尽量少的词来记住这个第一层，那就是“信号 (opens new window)和介质 (opens new window)”。物理层主要定义了物理设备的标准，如网线的类型，光纤的接口类型，各种传输介质的传输速率。

      • 功能

        主要功能是传输比特流（0101二进制数据），将比特流转化为电流强弱传输，到达目的后再转化为比特流，即常说的数模转化和模数转换。物理层解决了两台物理机的通信需求，具体就是机器A往机器B发送比特流，机器B能收到比特流。

      • 常见的物理层设备

        • 网卡 (opens new window)

        • 光纤 (opens new window)

        • CAT-5 (opens new window)线

          • RJ-45 (opens new window)接头

        • 集线器 (opens new window)有整波作用。

          集线器和交换机区别 (opens new window)

        • Repeater加强信号。

        • 串口

        • 并口

      • 参考资料

        物理层 (opens new window)

    • 数据链路层

      • 概念

        数据链路层是OSI参考模型 (opens new window)中的第二层，介乎于物理层 (opens new window)和网络层 (opens new window)之间。数据链路层在物理层提供的服务的基础上向网络层提供服务，其最基本的服务是将源自物理层来的数据 (opens new window)可靠地传输 (opens new window)到相邻节点 (opens new window)的目标机网络层。

      • 功能

        在传输比特流的过程中，会产生错传、数据传输不完整的可能。所以数据链路层主要有两个功能 ：帧编码和误差纠正控制。帧编码意味着定义一个包含信息频率、位同步、源地址、目标地址以及其他控制信息的数据包。数据链路层协议又被分为两个子层 ：逻辑链路控制 (opens new window)（LLC）协议和媒体访问控制 (opens new window)（MAC）协议。本层将比特数据组成帧，交换机工作在这层，对帧解码，并根据帧中包含的信息把数据发送到正确的接收方。

        数据链路层会将0、1序列划分为具有意义的数据帧传送给对端（数据帧的生成与接收）

      • 常见的数据链路层设备

        • 交换机

      • 常见协议

        • 点对点协议 (opens new window)（Point-to-Point Protocol）；
        • 以太网 (opens new window)（Ethernet）；
        • 高级数据链路协议（High-Level Data Link Protocol）；
        • 帧中继 (opens new window)（Frame Relay）；
        • 异步传输模式 (opens new window)（Asynchronous Transfer Mode）；

      • 参考资料

        数据链路层 (opens new window)

    • 网络层

      • 概念

        随着网络节点的不断增加，点对点通讯需要通过多个节点，如何找到目标节点，如何选择最佳路径成为首要需求。，网络层是OSI参考模型 (opens new window)中的第三层，介于传输层 (opens new window)和数据链路层 (opens new window)之间，它在数据链路层提供的两个相邻端点之间的数据帧 (opens new window)的传送功能上，进一步管理网络中的数据通信 (opens new window)，将数据设法从源端经过若干个中间节点传送到目的端，从而向传输层提供最基本的端到端的数据传送 (opens new window)服务

      • 功能

        网络层主要功能是将网络地址转化为对应的物理地址(通过ARP协议 (opens new window))，并决定如何将数据从发送方路由到接收方。网络层通过综合考虑发送优先权、网络拥塞程度、服务质量以及可选路由的花费来决定从一个网络中节点A到另一个网络中节点B的最佳路径。网络层将数据从发送端的主机发送到接收端的主机，两台主机间可能会存在很多数据链路，但网络层就是负责找出一条相对顺畅的通路将数据传递过去。传输的地址使用的是IP地址。IP地址通过不断转发到更近的IP地址，最终可以到达目标地址.

        此层的数据称之为数据包。本层需要关注的协议TCP/IP协议中的IP协议。

      • 常见网络层设备

        • 路由器

          由于网络层处理并智能指导数据传送，路由器连接网络隔断，所以路由器属于网络层。

      • 常见协议

        • IP协议
        • 地址解析协议(ARP)
        • ICMP协议
        • IGMP协议

      • 参考资料

        网络层 (opens new window)

    • 传输层

      • 概念

        随着网络通信需求的进一步扩大，通信过程中需要发送大量的数据，如海量文件传输，可能需要很长时间，网络在通信的过程中会中断很多次，此时为了保证传输大量文件时的准确性，需要对发送出去的数据进行切分，切割为一个一个的段落（Segement）发送，其中一个段落丢失是否重传，段落是否按顺序到达，是传输层需要考虑的问题。

      • 功能

        传输层解决了主机间的数据传输，数据间的传输可以是不同网络，并且传输层解决了传输质量的问题。有以下几个功能

        分割与重组数据

        按端口号 (opens new window)寻址 (opens new window)

        连接管理

        差错控制 (opens new window)和流量控制 (opens new window),纠错的功能

        传输层要向会话层 (opens new window)提供通信服务的可靠性 (opens new window)，避免报文 (opens new window)的出错、丢失、延迟时间紊乱、重复、乱序等差错。

      • 常见协议

        • TCP
        • UDP

      • 参考资料

        传输层 (opens new window)

    • 会话层

      • 概念

        会话层(Session)是建立在传输层之上，利用传输层提供的服务，使应用建立和维持会话，并能使会话获得同步。会话层使用校验点可使通信会话在通信失效时从校验点继续恢复通信。这种能力对于传送大的文件极为重要。

      • 功能

        会话层作用是负责建立和断开通信连接

      • 常见协议

        • ADSP
        • RPC

      • 参考资料

        会话层 (opens new window)

    • 表示层

      • 概念

        表示层向上对应用层服务，向下接受来自会话层的服务。表示层为在应用过程之间传送的信息提供表示方法的服务，它只关心信息发出的语法和语义。

      • 功能

        解决不同系统之间通信语法问题，在表示层数据将按照网络能理解的方案进行格式化，格式化因所使用网络的不同而不同。它主要负责数据格式的转换。具体来说，就是讲设备固有的数据格式转换为网络标准格式。常见的功能有以下三个

        语法转换：语法转换涉及代码转换和字符集的转换，数据格式的修改、数据结构操作的适配、数据压缩、数据加密等。

        语法选择：语法选择是提供初始选择的一种语法和随后修改这种选择的手段。

        联接管理：利用会话层提供的服务建立表示联接，管理在这一联接之上的数据运输和同步控制，以及正常或非正常地终止联接。

        Linux给 WIndows 发包，不同系统语法不一致，如exe不能在Linux下执行，shell不能在Windows不能直接运行。于是需要表示层。

      • 常见协议

        • ASCII
        • SSL/TLS

      • 参考资料

        表示层 (opens new window)

    • 应用层

      • 概念

        应用层（Application layer）是七层OSI模型的第七层。应用层直接和应用程序接口 (opens new window)并提供常见的网络应用服务 (opens new window)。应用层也向表示层 (opens new window)发出请求。应用层是开放系统的最高层,是直接为应用进程提供服务的。

      • 功能

        应用层旨在更方便应用从网络中接收的数据，规定发送方和接收方必须使用一个固定长度的消息头，消息头必须使用某种固定的组成，消息头中必须记录消息体的长度等信息，方便接收方正确解析发送方发送的数据。

      • 常见协议

        • HTTP
        • FTP
        • SMTP
        • POP3

      • 参考资料

        应用层 (opens new window)

    • 总结

      • 各层传输的数据

        四层传输层数据被称作段（Segments）；

        三层网络层数据被称做包（Packages）；

        二层数据链路层时数据被称为帧（Frames）；

        一层物理层时数据被称为比特流（Bits）。

    # 1.2.2 TCP/IP模型

    TCP/IP模型将计算机网络分为四层，从下到上分别是网络接口层、网际层、传输层和应用层。每一层都有特定的功能和协议。

    • 应用层

      应用层，主要有负责web浏览器的HTTP协议， 文件传输的FTP协议，负责电子邮件的SMTP协议，负责域名系统的DNS等

    • 传输层

      传输层，主要是有可靠传输的TCP协议，特别高效的UDP协议。主要负责传输应用层的数据包。传输层（Transport Layer）是为应用层提供网络支持的。

    • 网络层

      网络层，主要是IP协议。主要负责寻址（找到目标设备的位置）

    • 数据链路层

      数据链路层，主要是负责转换数字信号和物理二进制信号。数据链路层也叫网络接口层.

    • 总结

      • OSI七层模型和TCP/IP模型的区别

        OSI模型注重通信协议必要的功能；TCP/IP更强调在计算机上实现协议应该开发哪种程序。

      • 四层模型

      • 每一层的封装格式

      • 四层网络协议的作用

        • 发送端是由上至下，把上层来的数据在头部加上各层协议的数据（部首）再下发给下层。
        • 接受端则由下而上，把从下层接受到的数据进行解密和去掉头部的部首后再发送给上层。
        • 层层加密和解密后，应用层最终拿到了需要的数据。

      • 从键入URL到网页显示发生了什么

        URL解析-->DNS解析(获取IP地址)-->协议栈-->TCP(传输)-->IP寻址-->MAC(获取物理地址)-->网卡-->显示

        了解每一层的功能，就能把数据流图串起来。

      • 报文格式

      • TCP/IP协议族

        从字面意义上来讲，TCP/IP是指传输层的TCP协议和网络层的IP协议。

        实际上，TCP/IP只是利用 IP 进行通信时所必须用到的协议群的统称。

        具体来说，在网络层是IP/ICMP协议、在传输层是TCP/UDP协议、在应用层是SMTP、FTP、以及 HTTP 等。他们都属于 TCP/IP 协议。

    # 2. 网络协议

    # 2.1 网络协议概念

    网络协议是指在计算机网络中，用于实现数据传输和通信的规则、标准和约定。网络协议定义了数据在网络中的传输方式、格式、顺序、错误处理等方面的细节，使得不同的设备和系统可以进行有效的通信。

    # 2.2 常见的通信模式

    通信模式是指在信息传输过程中采用的特定方式和方法，包括通信协议、通信媒介、通信方式等。常见的通信模式有以下几种：

    • 单工通信模式(One-way Communication Mode):指在一次通信过程中，只能有一个方向进行信息传输，而不能在同一时间进行双向的通信。如传统的电话通信。
    • 双向通信模式(Two-way Communication Mode):指在一次通信过程中，可以同时进行发送和接收信息的交互，从而实现更加复杂的通信场景，如视频会议、语音聊天等。
    • 广播通信模式(Broadcast Communication Mode):指将信息发送到一个或多个接收方，而不需要与每个接收方建立单独的连接。这种模式通常用于广播电台、电视节目等场景。
    • 点对点通信模式(Point-to-point Communication Mode):指两个设备之间直接建立连接进行通信的方式，不需要通过其他设备中转。这种模式通常用于互联网上的数据传输、蓝牙连接等场景。
    • 多点通信模式(Multipoint Communication Mode):指多个设备之间建立连接进行通信的方式，需要通过其他设备中转。这种模式通常用于计算机网络中的网络拓扑结构、局域网(LAN)等场景。

    总之，不同的通信模式适用于不同的场景和应用需求。在选择通信模式时，需要考虑通信的距离、速度、安全性等因素，并根据实际需求选择最合适的通信方式。注意TCP 是一个双工协议，数据任何时候都可以双向传输。

    # 2.3 常见的网络协议

    • 按照用途分类：

      • 应用层协议：如HTTP、FTP、SMTP等。
      • 传输层协议：如TCP、UDP等。
      • 网络层协议：如IP、ICMP等。
      • 数据链路层协议：如Ethernet、PPP等。

    • 按照功能分类：

      • 控制协议：用于控制和管理网络通信的协议，如TCP/IP协议族中的控制协议TCP。
      • 数据链路协议：用于在物理层和数据链路层之间进行数据传输的协议，如Ethernet、PPP等。
      • 网络层协议：用于在网络层之间进行数据传输的协议，如IP、ICMP等。
      • 传输层协议：用于在传输层之间进行可靠数据传输的协议，如TCP、UDP等。

    • 按照层次结构分类：

      • 物理层协议：用于定义物理介质和电气特性的协议，如Ethernet、PPP等。
      • 数据链路层协议：用于在物理层和网络层之间进行数据传输的协议，如Ethernet、PPP等。
      • 网络层协议：用于在网络层之间进行数据传输的协议，如IP、ICMP等。
      • 传输层协议：用于在传输层之间进行可靠数据传输的协议，如TCP、UDP等。
      • 应用层协议：用于实现具体应用的协议，如HTTP、FTP、SMTP等。

    # 2.4 常见的网络协议解释

    • HTTP(Hypertext Transfer Protocol):超文本传输协议，是用于Web浏览器和Web服务器之间通信的协议。HTTP定义了客户端和服务器之间的消息格式、请求方法、响应状态码等规范，使得Web应用程序能够通过Internet进行数据传输。
    • FTP(File Transfer Protocol):文件传输协议，用于在网络上进行文件的上传和下载。FTP使用客户端-服务器模型，客户端向服务器发送命令，服务器响应并返回文件内容。FTP支持匿名访问和身份验证两种方式。
    • SMTP(Simple Mail Transfer Protocol):简单邮件传输协议，是用于电子邮件发送的协议。SMTP规定了邮件发送方和接收方之间的通信格式、命令语法、认证方式等规范，使得电子邮件能够通过Internet进行传输。
    • TCP(Transmission Control Protocol):传输控制协议，是用于在网络中建立可靠连接的协议。TCP提供了一种可靠的数据传输机制，保证数据的完整性和顺序性。TCP还提供了流量控制和拥塞控制等功能，以确保网络的稳定性。
    • UDP(User Datagram Protocol):用户数据报协议，是用于在网络中进行无连接的数据传输的协议。UDP不提供数据的可靠性和顺序性保障，但具有较低的时延和较小的带宽占用。UDP常用于实时应用，如语音通话、视频流等。
    • IP(Internet Protocol):互联网协议，是用于在网络中进行数据包交换的协议。IP定义了数据包的结构和寻址方式，使得不同的网络设备可以相互通信。IP还提供了路由选择和分组转发等功能，以确保数据包能够到达目标设备。
    • ICMP(Internet Control Message Protocol):互联网控制报文协议，是用于在网络中传递控制信息的协议。ICMP提供了一种简单的机制，用于诊断网络故障、测量网络性能等。
    • Ethernet:局域网技术标准，是一种高速、可靠的数字信号传输技术。Ethernet使用CSMA/CD(载波侦听多址冲突检测)技术来避免数据冲突，提供了全双工通信和自动重传机制等功能。
    • PPP(Point-to-Point Protocol):点对点协议，是一种用于在点对点网络中建立连接的协议。PPP支持多种物理层接口和加密方式，提供了可靠的数据传输和身份验证功能。
    • HTTPS(Hypertext Transfer Protocol Secure)是一种安全的Web传输协议，是在HTTP协议基础上添加了SSL/TLS加密通信协议来保证数据传输的安全性和完整性。
    • DNS协议：用于将域名解析为IP地址，以便在Internet上找到对应的服务器。DNS协议采用递归查询的方式，将用户请求发送给根域名服务器，然后逐级向下查询，直到找到对应的IP地址。
    • ARP协议：ARP协议是一种地址解析协议，它用于将IP地址映射到物理地址(MAC地址)。当计算机需要与另一个计算机通信时，它会首先检查目标计算机的IP地址是否已知，如果未知则会向本地网络中的ARP缓存中查找该IP地址对应的MAC地址。如果缓存中没有该MAC地址，则会向相邻的路由器发送一个ARP请求，以获取该MAC地址。
    • URL协议：用于在Web浏览器和Web服务器之间传输数据，它定义了如何表示和访问Web资源(如网页、图片、视频等)。
    • Wi-Fi协议是一种无线局域网(WLAN)通信协议，它允许计算机和其他设备通过无线电波在无线网络中进行通信。Wi-Fi协议通常使用IEEE 802.11标准，该标准定义了多种不同的无线网络技术，如802.11a、802.11b、802.11g、802.11n和802.11ac等。Wi-Fi协议提供了一种方便的、易于使用的无线网络接入方式，使得用户可以在任何地方访问互联网。

    # 2.5 HTTPS协议与HTTP协议区别

    • HTTPS使用SSL/TLS协议进行加密通信，而HTTP则不使用任何加密机制。
    • HTTPS使用数字证书对服务器进行身份验证，而HTTP则没有这个功能。
    • HTTPS在客户端和服务器之间建立一个安全的隧道，可以防止中间人攻击等安全问题。

    具体来说，HTTPS的通信流程如下：

    1. 客户端向服务器发送一个请求。
    2. 服务器返回一个响应，并在响应头中包含数字证书。
    3. 客户端通过验证服务器的数字证书来确认服务器的身份。
    4. 客户端和服务器之间建立一个SSL/TLS连接，使用对称密钥或非对称密钥进行加密通信。
    5. 客户端和服务器之间的通信内容被加密，保证了数据的安全性和完整性。

    总之，HTTPS协议是一种安全可靠的Web传输协议，可以有效地保护用户的数据隐私和网络安全。

    # 2.6 三次握手四次挥手

    三次握手和四次挥手是TCP协议中用于建立和断开连接的过程。具体解释如下：

    三次握手：

    1. 客户端向服务器发送一个SYN包，表示请求建立连接。
    2. 服务器收到SYN包后，回复一个SYN+ACK包，表示确认收到请求，并请求建立连接。
    3. 客户端收到SYN+ACK包后，再回复一个ACK包，表示确认建立连接。

    四次挥手：

    1. 客户端向服务器发送一个FIN包，表示请求断开连接。
    2. 服务器收到FIN包后，回复一个ACK包，表示确认收到请求。
    3. 服务器向客户端发送一个FIN包，表示请求断开连接。
    4. 客户端收到FIN包后，回复一个ACK包，表示确认断开连接。

    # 2.7 三次握手四次挥手的原因

    三次握手和四次挥手的原因是因为TCP协议是面向连接的协议，需要在通信前建立连接，通信后断开连接。三次握手是为了确保双方都能收发数据，四次挥手是为了确保双方都能正常关闭连接。具体原因如下：

    三次握手：

    1. 第一次握手：客户端向服务器发送一个SYN包，表示请求建立连接。此时客户端进入SYN_SENT状态。
    2. 第二次握手：服务器收到SYN包后，回复一个SYN+ACK包，表示确认收到请求，并请求建立连接。此时服务器进入SYN_RCVD状态。
    3. 第三次握手：客户端收到SYN+ACK包后，再回复一个ACK包，表示确认建立连接。此时客户端和服务器都进入ESTABLISHED状态，可以开始通信。

    三次握手的原因是为了防止已失效的连接请求报文段突然又传送到了服务器，从而产生错误。如果只有两次握手，那么可能会出现这种情况：客户端发送了一个连接请求，但是因为网络延迟等原因，客户端没有收到服务器的回复，此时客户端会认为连接已经建立，但是服务器并没有收到连接请求，因此不会建立连接。如果此时客户端发送数据，服务器会认为这是一个错误的连接请求，从而拒绝连接。因此需要三次握手来确保双方都能收发数据。

    四次挥手：

    1. 第一次挥手：客户端向服务器发送一个FIN包，表示请求断开连接。此时客户端进入FIN_WAIT_1状态。
    2. 第二次挥手：服务器收到FIN包后，回复一个ACK包，表示确认收到请求。此时服务器进入CLOSE_WAIT状态，客户端进入FIN_WAIT_2状态。
    3. 第三次挥手：服务器向客户端发送一个FIN包，表示请求断开连接。此时服务器进入LAST_ACK状态。
    4. 第四次挥手：客户端收到FIN包后，回复一个ACK包，表示确认断开连接。此时客户端进入TIME_WAIT状态，等待2MSL后进入CLOSED状态，服务器也进入CLOSED状态。

    四次挥手的原因是因为TCP是全双工的协议，双方都可以发送和接收数据。因此在断开连接时，需要双方都发送FIN包来表示断开连接。第一次挥手是客户端发送FIN包，第二次挥手是服务器回复ACK包，表示确认收到请求。第三次挥手是服务器发送FIN包，第四次挥手是客户端回复ACK包，表示确认断开连接。需要四次挥手是因为在断开连接时，可能还有未发送完的数据需要传输，需要等待数据传输完成后再断开连接。因此需要等待一段时间，等待数据传输完成后再发送FIN包。

    # 3. 网络拓补

    # 3.1 网络拓扑概念

    网络拓扑(Network Topology)结构是指用传输介质 (opens new window)互连各种设备 (opens new window)的物理布局。指构成网络的成员间特定的物理的即真实的、或者逻辑 (opens new window)的即虚拟 (opens new window)的排列方式。如果两个网络 (opens new window)的连接结构相同我们就说它们的网络拓扑相同，尽管它们各自内部的物理接线 (opens new window)、节点 (opens new window)间距离可能会有不同。

    # 3.2 常见的网络拓扑

    • 常见的网络拓扑

    • 总线型拓扑结构

      总线型拓扑结构是指所有节点都连接在一条共享的通信线路上，数据可以在节点之间直接传输。总线型拓扑结构简单、易于实现，但当通信线路出现故障时，整个网络都会受到影响。因此，总线型拓扑结构适用于小型局域网。

    • 环型拓扑结构

      环型拓扑结构是指所有节点都连接成一个环形，数据沿着环形路径传输。环型拓扑结构的特点是可靠性高，一旦某个节点出现故障，不会影响整个网络的正常运行。但是，环型拓扑结构的缺点是效率低下，因为数据需要沿着环形路径依次传输。因此，环型拓扑结构适用于需要高可靠性的应用场景，如金融交易系统等。

    • 星型拓扑结构

      星型拓扑结构是指所有节点都连接到一个中心节点(即交换机),数据通过中心节点进行转发和路由。星型拓扑结构的特点是易于扩展和管理，因为中心节点可以随时增加或减少。但是，星型拓扑结构的缺点是如果中心节点出现故障，整个网络都会瘫痪。因此，星型拓扑结构适用于中小型企业内部网络。

    • 树型拓扑结构

      树型拓扑结构是指所有节点都连接成一棵树形结构，数据沿着分支和叶子节点进行传输。树型拓扑结构的特点是灵活性高，因为可以根据需要添加或删除节点。但是，树型拓扑结构的缺点是容易出现单点故障，因为所有的数据都需要经过根节点进行传输。因此，树型拓扑结构适用于大型企业内部网络和互联网服务提供商等场景。

    • 网状拓扑结构

      网状拓扑结构是指所有节点都可以相互连接，形成一个复杂的网状结构。网状拓扑结构的特点是可靠性高、可扩展性强,但是实现复杂、成本高昂。网状拓扑结构适用于需要高度可靠性和可扩展性的应用场景，如大型企业内部网络、数据中心等。

    不同的拓扑结构有不同的优缺点，因此在选择拓扑结构时需要根据具体的应用场景进行综合考虑。同时，还需要考虑网络的可靠性、性能、安全性等因素，以保证网络的稳定运行和数据的安全传输。

    # 4. 网络安全

    # 4.1 网络安全的基本概念

    网络安全是指保护计算机网络系统和数据免受未经授权的访问、使用、泄露、破坏、篡改等威胁的过程。网络安全的基本概念包括：

    • 保密性(Confidentiality):确保网络通信内容不被未经授权的个人或实体窃取或泄露。
    • 完整性(Integrity):确保网络通信内容没有被篡改、损坏或伪造，以保持其真实性和准确性。
    • 可用性(Availability):确保网络资源始终处于可用状态，以满足用户需求。
    • 身份认证(Authentication):确保用户能够通过正确的身份验证信息访问网络资源。
    • 授权(Authorization):确保只有经过授权的用户或实体才能访问特定的网络资源。

    # 4.2 网络攻击方式

    常见的网络攻击方式包括：

    • 病毒(Virus):一种具有自我复制能力的恶意软件，通过感染其他程序或文件传播自身。
    • 蠕虫(Worm):一种无需人工干预就能自动传播的恶意软件，通过网络共享传播。
    • 木马(Trojan):一种伪装成合法程序的恶意软件，一旦被执行，会损害目标系统。
    • DoS(Denial-of-Service)攻击：通过发送大量请求导致目标服务器过载，使其无法正常提供服务。
    • DDoS(Distributed Denial-of-Service)攻击：利用多个受感染计算机共同发起的攻击，使目标服务器瘫痪。
    • SQL注入(SQL Injection):通过操纵输入的数据，绕过数据库安全防护措施，获取敏感信息。
    • 跨站脚本攻击(Cross-Site Scripting,XSS):通过在受害者浏览器上执行恶意脚本，窃取用户信息。
    • 钓鱼攻击(Phishing):通过伪装成可信任来源，诱骗用户提供敏感信息。

    # 4.3 网络安全防范措施

    为了保护网络安全，可以采取以下防范措施：

    • 安装并更新杀毒软件和防火墙。
    • 为操作系统和应用程序打补丁。
    • 强化密码策略，定期更改密码，使用复杂且不易猜测的密码。
    • 避免点击不明链接和下载未知来源的附件
    • 使用VPN(Virtual Private Network,虚拟专用网络)加密网络通信。
    • 实施访问控制策略，只允许经过授权的用户或设备访问网络资源。
    • 对敏感数据进行加密存储和传输。
    • 定期备份关键数据，以防止数据丢失或损坏。
    • 对员工进行网络安全培训，提高其安全意识和防范能力。
    • 监控网络活动，及时发现并应对潜在威胁。
    • 采用多层安全防护措施，形成有效的网络安全防线。

    # 5. 网络编程

    # 5.1 网络编程的概念

    网络编程是指利用计算机网络进行程序设计和开发的过程。它是计算机科学中的一个重要分支，主要涉及网络通信、协议、数据传输等方面的知识。

    # 5.2 常见的网络编程

    常见的网络编程分类包括：

    • 套接字编程：套接字是一种通信机制，可以在不同的计算机之间进行数据传输。套接字编程是指利用套接字进行网络编程的过程。套接字编程通常采用基于TCP/IP协议的数据传输方式。常见的套接字编程语言包括C、C++、Java等。套接字编程可以用于开发网络聊天室、文件传输程序等
    • Web编程：Web编程是指利用Web技术进行网络编程的过程。Web编程通常采用基于HTTP协议的数据传输方式。Web编程主要涉及HTML、CSS、JavaScript等技术，可以用于开发Web应用程序、网站。Web编程可以用于开发网站、Web应用程序等
    • RPC编程：RPC（Remote Procedure Call）是一种远程过程调用机制，可以在不同的计算机之间进行函数调用。RPC编程是指利用RPC进行网络编程的过程。常见的RPC框架包括gRPC、Thrift等。RPC编程可以用于开发分布式系统、微服务等
    • RESTful编程：RESTful（Representational State Transfer）是一种基于HTTP协议的Web服务架构风格，可以用于开发Web API等。RESTful编程是指利用RESTful进行网络编程的过程。RESTful编程可以用于开发Web AP、移动应用程序等。

    # 6. 网络设备

    # 6.1 网络设备的概念

    网络设备是指用于构建、管理和连接计算机网络的硬件设备。

    # 6.2 常见的网络设备

    常见的网络设备包括

    • 路由器(Router)

      路由器是一种用于在不同网络之间转发数据包的设备。它可以根据IP地址和端口号来决定将数据包转发到哪个网络或计算机上。

    • 交换机(Switch)

      交换机是一种用于在局域网内转发数据包的设备。它可以根据MAC地址来决定将数据包转发到哪个端口上，从而实现快速的数据传输。

    • 防火墙(Firewall)

      防火墙是一种用于保护网络安全的设备。它可以监控进出网络的数据流，并根据预先设定的规则来允许或阻止数据包的传输。

    • 集线器(Hub)

      集线器是一种用于扩展局域网的设备。它可以将多个计算机连接在一起，但由于广播方式传输数据包，容易导致网络拥堵和安全问题。

    • 网卡(网络适配器)(Network Adapter)

      网络适配器是一种用于将计算机连接到局域网或广域网的设备。它可以将计算机的数据转换成网络协议，并通过物理介质传输到目标设备。

    • 中继器(Repeater)

      中继器是一种用于放大信号的设备。它可以将信号从一个位置放大后传输到另一个位置，从而扩大了信号覆盖范围。

    • 网关(Gateway)

      网关是一种用于连接不同网络的设备。它可以将来自不同网络的数据包进行路由和转发，从而实现不同网络之间的通信。

    • 网桥(Bridge)

      网桥是一种用于连接两个或多个局域网的设备，它可以将不同的局域网连接在一起，实现数据共享。

    • 负载均衡器(Load Balancer)

      负载均衡器是一种用于平衡网络流量的设备，它可以将流量分配到多个服务器上，以提高网络性能和可靠性。

    # 7. 网络性能优化

    # 7.1 网络性能优化概念

    网络性能优化是指通过一系列技术手段和管理措施，提高计算机网络的性能和可靠性的过程。网络性能优化的目的是提高网络的吞吐量、响应时间、可用性和安全性等方面的指标，以满足用户的需求和业务的要求。

    以下是一些情况下需要进行网络性能优化的情况：

    1. 当网络负载过高时，需要对网络进行优化以提高其吞吐量和响应时间。
    2. 当网络出现故障或延迟时，需要对网络进行优化以减少故障和延迟。
    3. 当网络的安全性和稳定性受到威胁时，需要对网络进行优化以提高其安全性和稳定性。
    4. 当用户对网络的可用性要求较高时，需要对网络进行优化以提高其可用性。

    # 7.2 常见的网络性能指标

    以下是一些常见的网络性能指标：

    1. 带宽(Bandwidth):指网络传输数据的能力，通常以比特/秒(bps)或兆比特/秒(Mbps)为单位。

      可以使用网络测试工具(如iperf)来测试网络的带宽。

    2. 延迟(Latency):指数据从发送端到接收端所需的时间，通常以毫秒(ms)为单位。延迟越小，网络性能越好。

      可以使用ping命令或网络测试工具(如traceroute、mtr-cmd等)来测试网络的延迟。

    3. 吞吐量(Throughput):指网络在一定时间内传输的数据量，通常以比特/秒(bps)或字节/秒(Bps)为单位。

      可以使用网络测试工具(如iperf、netcat等)来测试网络的吞吐量。

    4. 丢包率(Packet Loss Rate):指在网络传输过程中丢失的数据包占总数据包数的比例，通常以百分比(%)为单位。

      可以使用网络测试工具(如iperf、netcat等)来测试网络的丢包率。

    5. 连接数(Connections):指同时连接到网络上的设备数量。

      可以使用系统监视器或网络管理工具来查看连接到网络上的设备数量。

    6. 负载均衡(Load Balancing):指将网络流量分配到多个服务器上，以提高网络的可用性和性能。

      可以使用负载均衡软件或网络管理工具来查看网络负载均衡的状态。

    7. QoS(Quality of Service):指对不同类型的数据流设置不同的优先级，以保证重要数据的及时传输。

      可以使用网络管理工具或QoS软件来设置和查看网络的QoS优先级。

    # 7.3 带宽和宽带

    • 区别

      带宽大小和宽带大小都是与网络通信相关的概念，但它们的含义和作用有所不同。

      带宽大小指的是网络传输数据的能力，通常用单位为比特每秒（bps）来表示。带宽大小越大，网络传输数据的速度就越快，可以支持更多的数据传输。带宽大小可以通过网络测试工具进行测量，例如ping、traceroute等。

      宽带大小指的是宽带接入网络的速度，通常用单位为兆比特每秒（Mbps）来表示。宽带大小越大，用户接入网络的速度就越快，可以更快地访问互联网、下载文件等。宽带大小可以通过网络服务提供商（ISP）提供的服务套餐来选择，例如10Mbps、50Mbps等。

      因此，带宽大小和宽带大小的区别在于它们的含义和作用。带宽大小主要用于衡量网络传输数据的能力，而宽带大小主要用于衡量用户接入网络的速度。它们的联系在于，宽带大小需要依赖于带宽大小来实现，即网络传输数据的能力越大，用户接入网络的速度就越快。

    • 带宽的影响因素

      带宽大小由多个因素决定，包括网络设备、网络拓扑、网络协议、网络负载等。具体来说，以下是影响带宽大小的因素：

      1. 网络设备：网络设备的性能和配置会影响带宽大小。例如，路由器、交换机等设备的处理能力和带宽容量会影响网络传输数据的能力。
      2. 网络拓扑：网络拓扑的结构和布局会影响带宽大小。例如，星型拓扑、环形拓扑等不同的拓扑结构会影响数据传输的效率和速度。
      3. 网络协议：网络协议的类型和版本会影响带宽大小。例如，TCP协议和UDP协议的传输效率和速度不同，会影响带宽大小。
      4. 网络负载：网络负载的大小和类型会影响带宽大小。例如，网络流量的大小和类型会影响网络传输数据的能力。

    # 7.4 网络负载

    • 概念

      网络负载是指在网络中传输数据的总量，包括数据包的数量、大小和传输时间等。网络负载的大小直接影响网络的性能和稳定性。

    • 查询网络负载

      以下是一些常用的方法来判断查询网络负载：

      1. 使用系统监视器：在Windows系统中，可以通过任务管理器或资源监视器来查看网络负载。打开任务管理器，切换到“性能”选项卡，可以看到网络适配器的实时流量信息。
      2. 使用命令行工具：在Windows系统中，可以使用netstat命令来查看网络连接的状态和流量信息。例如，输入netstat -n可以显示当前活动的TCP连接和UDP连接的数量和带宽占用情况。
      3. 使用网络监控软件：网络监控软件可以实时监测网络流量和负载情况，并提供详细的报告和分析。常见的网络监控软件包括Wireshark、PRTG Network Monitor、Nagios等。
      4. 使用路由器管理界面：如果使用的是路由器作为网络负载均衡器或防火墙，可以在路由器管理界面中查看实时的流量信息和负载情况。

      无论使用哪种方法，都需要注意选择合适的时间点进行查询，以获得准确的结果。同时，也需要考虑其他因素对网络负载的影响，如网络拥塞、设备故障等。

    • 网络负载过高的表现

      1. 网络延迟增加：当网络负载过高时，数据包在传输过程中需要经过更多的路由器和交换机，从而导致网络延迟增加。
      2. 丢包率增加：当网络负载过高时，网络中的数据包容易丢失或被阻塞，从而导致丢包率增加。
      3. 网络拥堵：当网络负载过高时，网络中的数据流量过大，超过了网络设备的处理能力，从而导致网络拥堵。
      4. 网络故障增多：当网络负载过高时，网络设备容易出现故障或崩溃，从而导致网络故障增多。

      如果发现以上情况，就可能存在网络负载过高的问题。可以通过监控网络设备的状态、分析网络流量和性能指标等方式来进一步判断和诊断问题。同时，也可以采取一些措施来降低网络负载，例如优化路由算法、增加带宽、使用缓存技术等。

    # 7.5 网卡常见的性能指标

    网卡的性能指标通常包括以下几个方面：

    1. 传输速率：指网卡在传输数据时的速率，通常以Mbps或Gbps为单位。传输速率越高，网卡的传输效率越高。
    2. 带宽：指网卡能够支持的最大传输速率，通常以Mbps或Gbps为单位。带宽越大，网卡的传输能力越强。
    3. 缓存大小：指网卡内置的缓存大小，通常以KB或MB为单位。缓存大小越大，网卡在传输数据时的缓存能力越强，可以提高传输效率。
    4. 支持的协议：指网卡支持的网络协议类型，如TCP/IP、UDP等。支持的协议越多，网卡的适用范围越广。
    5. 支持的操作系统：指网卡支持的操作系统类型，如Windows、Linux等。支持的操作系统越多，网卡的兼容性越好。

    举例来说，一款网卡的性能指标为传输速率为1Gbps，带宽为10Gbps，缓存大小为256MB，支持的协议为TCP/IP和UDP，支持的操作系统为Windows和Linux。这款网卡适用于需要高速传输数据的场景，如数据中心、云计算等领域。同时，它的缓存能力和协议支持也能够满足不同的应用需求。

    # 7.6 网卡和NIC区别和联系

    网卡和网络接口控制器(NIC)是两个不同的概念，但它们之间有联系。

    网卡是计算机中用于连接网络的硬件设备，它通常是一个插在计算机主板上的扩展卡，也可以是集成在主板上的芯片。网卡的主要功能是将计算机中的数据转换成网络中的数据，并将网络中的数据转换成计算机中的数据。网卡通常包括一个MAC地址，用于在网络中唯一标识该网卡。

    网络接口控制器(NIC)是一种控制器芯片，它通常集成在网卡中，用于控制网卡的工作。NIC通常包括一个或多个物理接口，用于连接计算机和网络。NIC还可以包括一些其他的功能，如数据包过滤、数据包重组、流量控制等。

    因此，网卡和NIC之间的联系是，网卡是用于连接网络的硬件设备，而NIC是用于控制网卡的控制器芯片。在实际应用中，网卡和NIC通常是一体化的，用户可以通过网卡来访问网络，而NIC则负责控制网卡的工作。

    # 7.7 常见的网络命令

    • Windows 下常见的网络命令

      1. ping：用于测试网络连接是否正常，可以测试网络延迟和丢包率等信息。
      2. tracert：用于跟踪数据包在网络中的路径，可以查看数据包经过的路由器和延迟时间等信息。
      3. ipconfig：用于查看本机的IP地址、子网掩码、网关等网络配置信息。
      4. netstat：用于查看网络连接状态，可以查看本机的TCP和UDP连接情况，以及监听的端口等信息。
      5. nslookup：用于查询域名对应的IP地址，可以查看域名解析是否正常。
      6. arp：用于查看本机的ARP缓存表，可以查看本机与其他设备的MAC地址对应关系。
      7. route：用于查看本机的路由表，可以查看本机与其他网络设备的路由信息。

    • Linux 下常见的网络命令

      1. ping：用于测试网络连接是否正常，可以测试网络延迟和丢包率等信息。示例：ping www.baidu.com
      2. traceroute：用于跟踪数据包在网络中的路径，可以查看数据包经过的路由器和延迟时间等信息。示例：traceroute www.baidu.com
      3. ifconfig：用于查看本机的IP地址、子网掩码、网关等网络配置信息。示例：ifconfig
      4. netstat：用于查看网络连接状态，可以查看本机的TCP和``UDP连接情况，以及监听的端口等信息。示例：netstat -an
      5. nslookup：用于查询域名对应的IP地址，可以查看域名解析是否正常。示例：nslookup www.baidu.com
      6. arp：用于查看本机的ARP缓存表，可以查看本机与其他设备的MAC地址对应关系。示例：arp -a
      7. route：用于查看本机的路由表，可以查看本机与其他网络设备的路由信息。示例：route -n
      8. ssh:用于远程登录到另一台计算机，可以在远程计算机上执行命令或者传输文件。示例：ssh username@remote_host
      9. curl:用于发送HTTP请求并获取响应，可以用于测试API接口或者下载文件等。示例：curl https://www.example.com/api
      10. wget:用于下载文件，可以从HTTP、FTP等协议下载文件。示例：wget https://www.example.com/file.zip

      以上是常见的Linux下网络命令，可以通过终端或者Shell等工具来使用。
