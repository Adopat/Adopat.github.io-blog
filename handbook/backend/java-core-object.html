    # 3.1 变量

    1. 变量的类型：

      Java中的变量类型有很多种，包括整型、浮点型、字符型、布尔型等。不同的变量类型适用于不同的场合，学习和掌握变量类型可以让程序更加灵活和高效。

      例如，整型变量可以用来存储整数值：

      1

      double y = 3.14;

    8. Java 变量分类

      局部变量（Local Variables）：定义在方法、代码块或构造器中的变量。局部变量只在定义它的方法、代码块或构造器内部有效，一旦离开该方法、代码块或构造器，该变量就被销毁了。

      成员变量（Member Variables）：定义在类中但在方法之外的变量，也称为实例变量。成员变量可以被类中的所有方法访问，如果不对其进行初始化，成员变量会被自动初始化为默认值。

      静态变量（Static Variables）：定义在类中但使用 static 关键字修饰的变量，也称为类变量。静态变量是类的所有实例共享的，可以通过类名直接访问，不需要创建类的实例。静态变量也会被自动初始化为默认值。

      参数变量（Parameter Variables）：定义在方法签名中的变量，也称为形式参数。参数变量只在方法的执行过程中有效，一旦方法执行完毕，参数变量就被销毁了。

      需要注意的是，Java 变量的作用域和生命周期是由其所在的代码块、方法或类决定的。不同类型的变量具有不同的作用域和生命周期，需要根据具体情况来选择使用。

    1. 参数类型

      Java中的参数类型可以分为三类：基本类型、引用类型和数组类型。基本类型包括int、double、boolean等；引用类型包括String、Object等；数组类型则是一种特殊的引用类型。

    3. 参数传递方式

      注意Java中参数传递只有值传递方式。

    • 值传递：值传递是指在调用函数时将实际参数复制一份传递到函数中，这样在函数中如果对参数进行修改，将不会影响到实际参数。
    • 引用传递：引用传递是指在调用函数时将实际参数的地址传递到函数中，那么在函数中对参数所进行的修改，将影响到实际参数。

    # 3.5 代码块

    1. 代码块的概念

      代码块是Java程序中的一段代码，它可以包含一组语句，用于执行特定的任务。代码块可以是普通代码块、静态代码块、构造代码块和同步代码块。

    2. 代码块的分类

      Java中的代码块分为普通代码块、静态代码块、构造代码块和同步代码块。

    • 普通代码块：用于限制变量的作用范围，只有在代码块内部才能访问定义在代码块内部的变量。

    • 静态代码块：用于初始化类的静态变量，静态代码块只在类加载时执行一次。

    • 构造代码块：用于执行一些初始化操作，在每次创建对象时都会执行。

    • 同步代码块：用于控制线程的并发访问，同步代码块可以保证同一时间只有一个线程能够访问共享资源。

    1. 代码块的作用

      代码块可以用于控制变量的作用范围、初始化类的静态变量、执行一些初始化操作等。

    2. 代码块的语法

      普通代码块和同步代码块使用大括号{}括起来，静态代码块使用static关键字，构造代码块使用类名加{}括起来。

      public class CodeBlockDemo {

    static int num1 = 10;
    int num2 = 20;

    // 静态代码块

    // 构造代码块

        System.out.println("构造代码块执行");
        num2 = 40;
    }

    // 普通代码块
    public void method() {
        System.out.println("普通代码块执行");
        int num3 = 50;
        System.out.println("num3 = " + num3);
    }

    public static void main(String[] args) {
        CodeBlockDemo cbd = new CodeBlockDemo();
        cbd.method();
        System.out.println("num1 = " + num1);
        System.out.println("num2 = " + cbd.num2);
    }

    8. 接口的分类

      1. 普通接口：普通接口是指只包含抽象方法的接口。普通接口用于定义一组方法的规范，实现该接口的类必须实现接口中所有的抽象方法。

      2. 默认方法接口：默认方法接口是指在接口中可以定义实现的方法。在 Java 8 中，接口可以定义默认方法，实现该接口的类可以直接使用默认方法的实现，也可以重写默认方法。

      3. 静态方法接口：静态方法接口是指在接口中可以定义静态方法。在 Java 8 中，接口可以定义静态方法，可以通过接口名直接调用静态方法。

      4. 函数式接口：函数式接口是指只包含一个抽象方法的接口。函数式接口用于定义只有一个抽象方法的接口，可以使用 Lambda 表达式来实现该接口。Java 8 引入了函数式接口，使得 Java 语言可以更方便地支持函数式编程。

      5. 嵌套接口：嵌套接口是指在一个接口中定义另一个接口。嵌套接口可以用于将一组相关的方法和常量组织在一起，方便管理和使用。

      6. 标识接口：标识接口是指不包含任何方法的接口，它只是用来标识一个类是否符合某个特定的规范。在 Java 中，Serializable 和 Cloneable 接口就是标识接口的典型例子。

      7. 监听器接口：监听器接口是指用于处理事件的接口。

      8. 回调接口：回调接口是指用于回调的接口。

      9. 迭代器接口：迭代器接口是指用于遍历集合的接口。

      10. 访问控制接口：访问控制接口是指用于控制访问权限的接口。

      这些接口都是针对特定场景而设计的，用于解决特定的编程问题。

    9. java接口继承和接口实现的区别

      1. 接口继承是指一个接口可以继承另一个接口中的方法和常量，这样子接口就可以拥有父接口中的所有属性和方法。使用关键字extends实现接口继承。
      2. 接口实现是指一个类可以实现一个或多个接口，这样子类就需要实现接口中的所有方法。使用关键字implements实现接口实现。
      3. 接口继承是用于扩展接口的功能，而接口实现是用于实现接口的功能。
      4. 接口继承中的方法和常量是可以被子接口继承的，而接口实现中的方法和常量必须被实现。
      5. 接口继承可以多层继承，而接口实现也可以实现多个接口。
      6. 接口继承是允许方法的默认实现，而接口实现必须实现接口中的所有方法。 总的来说，接口继承和接口实现都是Java中重要的面向对象编程概念，它们的应用场景不同，需要根据具体的需求来选择使用哪种方式。

    # 3.9 多态

    1. 理解多态的概念

      多态是指同一种行为具有多个不同的表现形式或形态的能力。在面向对象编程中，多态是指同一个方法可以在不同的对象上产生不同的行为。多态的实现需要依赖继承和重写的特性。

    2. 掌握继承和重写

      继承是指子类可以继承父类的属性和方法。在子类中，可以通过重写父类的方法来改变方法的行为。重写的方法需要与父类方法具有相同的方法名、参数列表和返回类型。

      下面是一个简单的继承和重写的示例代码：

      class Animal {
    public void makeSound() {
        System.out.println("Animal is making sound.");
    }
}

class Cat extends Animal {
    @Override
    public void makeSound() {
        System.out.println("Cat is meowing.");
    }
}

public class Test {
    public static void main(String[] args) {
        Animal animal = new Animal();
        Cat cat = new Cat();

        animal.makeSound(); // 输出 "Animal is making sound."
        cat.makeSound(); // 输出 "Cat is meowing."
    }
}

    4. 掌握向上转型和向下转型

      向上转型是指将一个子类对象赋给一个父类引用，向下转型是指将一个父类引用强制转换为一个子类引用。向上转型可以实现多态的效果，向下转型可以访问子类特有的属性和方法。

      向下转型是指将父类引用强转为子类类型；这是不安全的，因为有的时候，父类引用指向的是父类对象，向下转型就会抛出 ClassCastException，表示类型转换失败；但如果父类引用指向的是子类对象，那么向下转型就是成功的。

      下面是一个简单的向上转型和向下转型的示例代码：

      class Animal {
    public void makeSound() {
        System.out.println("Animal is making sound.");
    }
}

class Cat extends Animal {
    public void scratch() {
        System.out.println("Cat is scratching.");
    }

    @Override
    public void makeSound() {
        System.out.println("Cat is meowing");
    }
}

public class Test {
    public static void main(String[] args) {
        // 向上转型
        Animal animal = new Cat();
        animal.makeSound(); // 输出 "Cat is meowing."
        // 向下转型
        if (animal instanceof Cat) {
            Cat cat = (Cat) animal;
            cat.scratch(); // 输出 "Cat is scratching."
        } else {
            System.out.println("Animal is not a cat.");
        }
    }
}

    5. 熟悉多态的应用场景

      多态在实际编程中有很多应用场景，比如在集合、框架等方面都有广泛应用。下面是一个使用多态实现简单计算器的示例代码：

      interface Operation {
    double getResult(double num1, double num2);
}

class Add implements Operation {
    @Override
    public double getResult(double num1, double num2) {
        return num1 + num2;
    }
}

class Subtract implements Operation {
    @Override
    public double getResult(double num1, double num2) {
        return num1 - num2;
    }
}

class Multiply implements Operation {
    @Override
    public double getResult(double num1, double num2) {
        return num1 * num2;
    }
}

class Divide implements Operation {
    @Override
    public double getResult(double num1, double num2) {
        if (num2 == 0) {
            throw new IllegalArgumentException("Divisor cannot be zero.");
        }
        return num1 / num2;
    }
}

public class Calculator {
    public static void main(String[] args) {
        Operation add = new Add();
        Operation subtract = new Subtract();
        Operation multiply = new Multiply();
        Operation divide = new Divide();

        double num1 = 10.0;
        double num2 = 5.0;

        System.out.println("Addition result: " + add.getResult(num1, num2));
        System.out.println("Subtraction result: " + subtract.getResult(num1, num2));
        System.out.println("Multiplication result: " + multiply.getResult(num1, num2));
        System.out.println("Division result: " + divide.getResult(num1, num2));
    }
}

    6. 掌握多态的优缺点
