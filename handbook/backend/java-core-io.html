    # 11.1.2 文件过滤器

    了解FileFilter接口和FilenameFilter接口的概念，学会使用这两个接口来过滤文件。

    • FileFilter接口

      Java中的FileFilter接口是一个过滤器接口，用于过滤文件。它只有一个方法，即accept(File pathname)方法，该方法接受一个File对象作为参数，返回一个boolean值，用于判断该文件是否符合过滤条件。如果符合过滤条件，返回true，否则返回false。 使用FileFilter接口可以实现对指定目录下的文件进行过滤，只返回符合条件的文件。例如，可以通过实现FileFilter接口来过滤出指定类型的文件，或者过滤出文件名符合指定规则的文件

      /**
 * @author Adopat
 * @description: FileFilter 接口的使用
 * @date 2023/03/13 16:38
 **/
import java.io.*;
class MyFileFilter implements FileFilter {
    private String fileType;
    public MyFileFilter(String fileType) {
        this.fileType = fileType;
    }
    @Override
    public boolean accept(File file) {
        if (file.isFile() && file.getName().endsWith("." + fileType)) {
            return true;
        }
        return false;
    }
}
public class FileFilterTest {
    public static void main(String[] args) {
        File dir = new File("src/java_io/test");
        // 传入过滤器
        File[] files = dir.listFiles(new MyFileFilter("txt"));
        for (File file : files) {
            System.out.println(file.getAbsolutePath());
        }
    }
}

    • FilenameFilter接口

      Java中的FilenameFilter接口也是一个过滤器接口，它用于过滤文件名。与FileFilter接口不同，FilenameFilter接口只关注文件名，而不关注文件的其他属性。它也只有一个方法，即boolean accept(File dir, String name)方法，该方法接受一个表示目录的File对象和一个表示文件名的字符串作为参数，返回一个boolean值，用于判断该文件是否符合过滤条件。如果符合过滤条件，返回true，否则返回false。 使用FilenameFilter接口可以实现对指定目录下的文件名进行过滤，只返回符合条件的文件。例如，可以通过实现FilenameFilter接口来过滤出文件名符合指定规则的文件。

      /**
 * @author Adopat
 * @description: FilenameFilter的使用
 * @date 2023/03/13 16:51
 **/
public class FileNameFilterTest {
    public static void main(String[] args) {
        File dir = new File("src/java_io/test");
        // 传入过滤器
        String fileType = "txt";
        File[] files = dir.listFiles(new FilenameFilter() {
                                         @Override
                                         public boolean accept(File dir, String name) {
                                             if (name.endsWith("." + fileType)) {
                                                 return true;
                                             }
                                             return false;
                                         }
                                     }
        );
        for (File file : files) {
            System.out.println(file.getAbsolutePath());
        }
    }
}

    • FilenameFilter与FileFilter的联系和区别

      联系：

      1. 都是接口，用于过滤文件。
      2. 都包含一个accept()方法，用于判断文件是否符合过滤条件。
      3. 都可以作为File类的参数，用来过滤出符合条件的文件。

      区别：

      1. FileFilter接口是基于文件本身的属性进行过滤，而FilenameFilter接口是基于文件名进行过滤。
      2. FileFilter接口的accept()方法接受一个File对象作为参数，而FilenameFilter接口的accept()方法接受一个表示目录的File对象和一个表示文件名的字符串作为参数。
      3. 在实现FileFilter接口时，需要判断文件的属性，例如文件是否为目录、文件大小、文件类型等；而在实现FilenameFilter接口时，只需要判断文件名是否符合要求。
      4. 在使用FileFilter接口时，可以通过File类的listFiles(FileFilter filter)方法来过滤文件；在使用FilenameFilter接口时，可以通过File类的listFiles(FilenameFilter filter)方法来过滤文件名。 总之，FileFilter接口和FilenameFilter接口都是非常实用的接口，可以帮助我们快速、方便地过滤出符合条件的文件或文件名。选择哪个接口取决于具体的过滤需求。

    # 11.2.2 字节流概念

    字节流以字节为单位传输数据，通常用于传输二进制数据或图片等非文本数据，而不是纯文本数据。字节流可以处理文件中的所有类型的数据，无论数据是否是文本。

    # 11.2.3 字节流的使用场景及实例

    • 使用场景

      • 文件读写：字节流可以用于读取和写入文件。通过使用FileInputStream和FileOutputStream类，可以打开并读取一个文件，或者将数据写入文件。

      • 网络传输：字节流可以被用于通过网络传输数据。Socket类允许建立一个客户端到服务器之间的连接，然后用InputStream和OutputStream实例读取和写入数据。

      • 图像处理：字节流可以被用于图像处理。BufferedInputStream和BufferedOutputStream类能够缓存数据和提供压缩和解压缩服务，这对于图像处理非常有用。

      • 音频处理：字节流可以被用于音频处理。Java Sound API提供了一些音频处理的类和接口，例如AudioInputStream和AudioSystem类。

      • 数据库访问：字节流可以被用于数据库访问。例如，通过使用Blob类型，可以从数据库中读取和写入二进制数据。

      • 序列化：字节流可以被用于序列化和反序列化对象。ObjectInputStream和ObjectOutputStream类可用于将对象转换为字节流，以及将字节流转换回对象。

      • 压缩/解压缩：字节流可以被用于压缩/解压缩文件和数据。GZIPOutputStream和GZIPInputStream类可用于压缩和解压缩文件、ZipOutputStream和ZipInputStream类可用于压缩和解压缩zip格式的数据。

    • 实例代码

      /**
 * @author Adopat
 * @description: 使用Java字节流中的字节过滤器进行压缩和解压缩的实例代码
 * @date 2023/03/14 16:03
 **/
import java.io.*;
import java.util.zip.GZIPInputStream;
import java.util.zip.GZIPOutputStream;

public class GZipFilterDemo {
    public static void main(String[] args) throws IOException {
        String inputFileName = "src/java_io/demo1/a.txt";
        String compressedFileName = "src/java_io/demo1/compressed.gz";
        String decompressedFileName = "src/java_io/demo1/decompressed.txt";

        // 压缩文件
        FileInputStream inputFile = new FileInputStream(inputFileName);
        GZIPOutputStream compressedFile = new GZIPOutputStream(new FileOutputStream(compressedFileName));
        byte[] buffer = new byte[1024];
        int len;
        while ((len = inputFile.read(buffer)) != -1) {
            compressedFile.write(buffer, 0, len);
        }
        inputFile.close();
        compressedFile.close();

        // 解压缩文件
        GZIPInputStream compressedInputFile = new GZIPInputStream(new FileInputStream(compressedFileName));
        FileOutputStream decompressedFile = new FileOutputStream(decompressedFileName);
        while ((len = compressedInputFile.read(buffer)) != -1) {
            decompressedFile.write(buffer, 0, len);
        }
        compressedInputFile.close();
        decompressedFile.close();
    }
}

    # 11.2.4 基本使用

    • InputStream

      常见方法：

      1. int read()：从输入流中读取一个字节的数据，返回值为读取的字节数据（0-255），如果读取到文件末尾则返回-1。
      2. int read(byte[] b)：从输入流中读取一定数量的字节数据，并将其存储到指定的字节数组b中，返回值为读取的字节数，如果读取到文件末尾则返回-1。
      3. int read(byte[] b, int off, int len)：从输入流中读取最多len个字节的数据，并将其存储到指定的字节数组b中的off位置开始，返回值为读取的字节数，如果读取到文件末尾则返回-1。
      4. long skip(long n)：从输入流中跳过n个字节的数据。
      5. int available()：获取输入流中可读取的字节数。
      6. void close()：关闭输入流。

    • OutputStream

      常见方法:

      1. void write(int b)：将一个字节的数据写入输出流中。
      2. void write(byte[] b)：将指定字节数组b中的所有字节数据写入输出流中。
      3. void write(byte[] b, int off, int len)：将指定字节数组b中的off位置开始的len个字节数据写入输出流中。
      4. void flush()：清空输出流的缓冲区，将所有缓存的数据写入目的地。
      5. void close()：关闭输出流。 在使用InputStream和OutputStream时，需要注意异常处理和资源释放问题，以保证程序的健壮性和资源的及时释放。

    # 11.2.5 常见子类

    • FileInputStream&FileOutputStream 特点

      • FileInputStream用于读取文件数据，FileOutputStream用于向文件写入数据。
      • FileInputStream和FileOutputStream都是阻塞式的I/O操作，读写文件时会阻塞线程。
      • FileInputStream和FileOutputStream都是基于字节流的，读写的数据按照字节为单位进行。
      • FileInputStream和FileOutputStream需要显式地创建和关闭文件流，否则可能会导致文件被占用或者数据丢失等问题。

    • ByteArrayInputStream&ByteArrayOutputStream特点

      • ByteArrayInputStream和ByteArrayOutputStream都是非阻塞式的I/O操作，在内存中读取和写入数据不会阻塞线程。

      • ByteArrayInputStream和ByteArrayOutputStream都是基于字节流的，读写的数据按照字节为单位进行。

      • ByteArrayInputStream和ByteArrayOutputStream不需要显式地创建和关闭流，读写操作完成后即可释放内存。

    • 使用场景

      1. 在内存中处理数据：ByteArrayInputStream和ByteArrayOutputStream都是基于内存的，因此在内存中处理数据时，可以使用它们来读写字节数组，而不必磁盘I/O或网络I/O。
      2. 与其它流进行数据交换：当需要将数据从一个流传输到另一个流时，可以使用ByteArrayInputStream和ByteArrayOutputStream，且它们的性能比FileInputStream和FileOutputStream（基于磁盘的流）高效得多。
      3. 用于序列化和反序列化：序列化和反序列化是将Java对象转换为字节流和将字节流转换回Java对象的过程。ByteArrayInputStream和ByteArrayOutputStream可用于实现这一过程，即将Java对象转换为字节数组，并将字节数组转换回Java对象。
      4. 测试代码：在编写单元测试时，可以使用ByteArrayInputStream和ByteArrayOutputStream来模拟输入输出流，来执行一些常用方法，如read()和write()，以确保代码正确性。

    • 区别

      • FileInputStream和FileOutputStream用于读写文件，而ByteArrayInputStream和ByteArrayOutputStream用于读写内存。
      • FileInputStream和FileOutputStream需要显式地创建和关闭流，而ByteArrayInputStream和ByteArrayOutputStream不需要。
      • FileInputStream和FileOutputStream是阻塞式的I/O操作，而ByteArrayInputStream和ByteArrayOutputStream是非阻塞式的I/O操作。 总的来说，这些子类都是用于读写字节流的，但是它们的应用场景和使用方式有所不同，需要根据具体情况选择合适的子类进行操作。

    • 阻塞式IO和非阻塞式IO的区别

      • 阻塞式I/O（Blocking I/O）

        当应用程序调用系统I/O操作时，线程将被阻塞并等待I/O操作完成。在此期间，该线程不能处理其他任务。如果出现了I/O的阻塞事件，线程就会一直等待下去，无法继续执行，也无法释放CPU资源，直到该I/O操作完成或者出现错误

      • 非阻塞式I/O（Non-blocking I/O）

        当应用程序调用系统I/O操作时，线程将不被阻塞，并立即返回结果。如果I/O操作没有立即完成，则线程将不断查询相关状态，直到操作完成。在此期间，线程可以处理其他任务。这种模型下，线程不会被阻塞，因此它可以一直处于运行状态。

    在实际应用中，阻塞式I/O适用于处理低负载、长连接、且需要数据准确性的场景；而非阻塞式I/O适用于处理高负载、短连接、且需要高并发、高可用性的场景。

    # 11.2.6 字节编码和字符编码

    • 编码

      按照某种规则，将字符存储到计算机中，称为编码

    • 解码

      反之，将存储在计算机中的二进制数按照某种规则解析显示出来，称为解码

    • 编码与解码的区别

      编码:字符(能看懂的)-->字节(看不懂的)

      解码:字节(看不懂的)-->字符(能看懂的)

    • 字节编码和字符编码的联系与区别

      区别:

      1. 字节编码与具体的字符集无关，而字符编码与具体的字符集有关。字节编码只关注二进制数字的组合方式和长度，对于具体字符的含义并不关心；而字符编码则要求对每个字符进行唯一的标识，所以需要对字符集进行选择和定义。

        字节编码是将文本转换成二进制数字的方式，即将每个字符表示为一定数量的字节（通常是1或2个字节）。字节编码可以是固定长度或可变长度的

      2. 字节编码通常用于文件存储和网络传输，而字符编码则主要用于应用程序中进行字符处理。

        字符编码则是将字符映射到数字的方式，即为每个字符指定一个唯一的标识符（通常是整数）

      联系:

      1. 由于文本需要存储在计算机中，所以无论是字节编码还是字符编码，都是必要的。
      2. 在某些情况下，字节编码和字符编码需要相互转换，例如在文本编辑器中打开具有不同字符集的文件时。

    • 常见的字节编码

      1. Base64：将二进制数据转换为ASCII字符的编码方式，用于在网络上传输和存储。
      2. Hex：将二进制数据转换为十六进制字符串的编码方式，用于调试和显示二进制数据。

    • 常见的字符编码

      1. ASCII：将每个字符映射为一个唯一的7位二进制数，适用于英文字符。
      2. Unicode：包含了世界上所有的字符，适用于多语言环境。
      3. UTF-8：一种变长编码方式，用于表示Unicode字符集，适用于在网络传输和存储中使用。
      4. UTF-16：一种定长编码方式，用于表示Unicode字符集，适用于在程序内部使用。
      5. GB2312：一种用于表示简体中文的编码方式，常用于在中国大陆的网站和应用中。
      6. Big5：一种用于表示繁体中文的编码方式，常用于在台湾和香港的网站和应用中。
      7. ISO-8859：国际标准化组织所定义的字符编码集合，分为多个子集，每个子集包含特定语言的字符。

    • GBK、ISO-8859和GB2312 区别和联系

      GBK、ISO-8859和GB2312是三种不同的字符编码标准，它们分别由不同的组织或国家制定和推广，具有不同的特点和用途。

      GBK是一种中文字符编码标准，它是在GB2312标准的基础上进行扩展的。GBK编码支持汉字、英文字母、数字、符号等各种字符，能够表示中国大陆地区的所有汉字，包括繁体字。GBK编码是一种双字节编码，每个汉字占两个字节，可以表示21886个汉字。GBK编码在Windows操作系统中广泛应用，是中文信息处理的主要编码方式之一。

      ISO-8859是一种国际标准字符编码标准，它定义了一系列字符集，包括ISO-8859-1、ISO-8859-2等。ISO-8859-1是最初的版本，支持欧洲主要语言的字符集，包括英语、法语、德语、西班牙语等。ISO-8859编码是单字节编码，每个字符占一个字节，可以表示256个字符。

      GB2312是中国国家标准的中文字符编码标准，它是在1980年制定的，是中国大陆地区最早的中文字符编码标准。GB2312编码支持简体中文字符集，每个汉字占两个字节，可以表示6763个汉字。GB2312编码在中国大陆地区广泛应用，是中文信息处理的主要编码方式之一。

      三种编码的区别和联系：

      1. 编码范围不同：GBK编码支持汉字、英文字母、数字、符号等各种字符，能够表示中国大陆地区的所有汉字；ISO-8859编码支持欧洲主要语言的字符集，包括英语、法语、德语、西班牙语等；GB2312编码支持简体中文字符集，每个汉字占两个字节，可以表示6763个汉字。

      2. 编码方式不同：GBK、GB2312编码是双字节编码，每个汉字占两个字节，而ISO-8859编码是单字节编码，每个字符占一个字节。

      总之，GBK、ISO-8859和GB2312是三种不同的字符编码标准，它们分别适用于不同的应用场景和语言环境，了解其特点和区别，有助于正确选择和应用合适的编码方式。

    # 11.3.1 字符流概念

    字符流以字符为单位传输数据，通常用于传输文本数据。字符流可以处理Unicode字符集（例如UTF-8、UTF-16），因此可以处理所有的文本字符，但不能处理二进制数据。

      • 处理CSV文件，例如读取和写入数据表格.

      • 通过流过滤器来处理文本文件，例如移除HTML标签或者进行文本压缩和解压缩.

      • 读取和写入XML和JSON格式的文本文件。

      • 应用程序日志处理。

    # 11.3.7 Java IO中的常见设计模式

    JavaIO中的设计模式指的是在Java IO库中使用的常见设计模式。这些设计模式通常是为了解决特定的问题而被使用的，例如：如何处理大量的数据、如何处理多种不同类型的数据、如何处理并发访问等等。

    以下是一些常见的JavaIO中的设计模式：

    1. 装饰器模式：Java IO库中的InputSteam和OutputStream类都使用了装饰器模式。装饰器模式允许动态地添加或删除对象的功能，而不会影响其它对象的行为。在Java IO库中，装饰器模式被用来添加或删除一些特定的功能，例如缓存、压缩、加密等等。

    2. 工厂模式：Java IO库中的File类和RandomAccessFile类都使用了工厂模式。工厂模式允许用户通过一个工厂方法来创建对象，而不需要直接调用构造函数。在Java IO库中，工厂模式被用来创建文件对象和随机访问文件对象。

    3. 观察者模式：Java IO库中的FileObserver类使用了观察者模式。观察者模式允许一个对象（被观察者）通知一组对象（观察者）自己的状态发生了改变。在Java IO库中，FileObserver类被用来监视文件系统中文件的变化。

    4. 适配器模式：Java IO库中的InputStreamReader和OutputStreamWriter类使用了适配器模式。适配器模式允许将一个类的接口转换成另一个类的接口，以满足客户端的需求。在Java IO库中，InputStreamReader和OutputStreamWriter类被用来将字节流转换成字符流。

    总之，JavaIO中的设计模式是为了解决特定的问题而被使用的。理解这些设计模式可以帮助开发人员更好地使用Java IO库，并且在自己的项目中应用这些模式。

    # 11.4.3 常见的序列化方式及工具

    • Java原生序列化

      Java原生序列化是Java标准库提供的一种序列化方式。它可以将Java对象序列化为字节流，并且可以将字节流反序列化为Java对象。Java原生序列化使用简单，但是序列化后的字节流比较臃肿，而且序列化和反序列化的性能也不是很高。

      Java原生序列化是Java标准库提供的一种序列化方式，因此不需要引入额外的工具。只需要使用Java IO库中的ObjectInputStream和ObjectOutputStream类即可实现序列化和反序列化操作。

      import java.io.*;

/**
 * @author Adopat
 * @description: 对象的序列化和反序列化
 * @date 2023/02/16 16:08
 **/
public class Object_Oper {
    public static void main(String[] args) throws IOException, ClassNotFoundException {
        String objectFile = "src/java_io/object_oper.txt";
        // 序列化
        serializable(objectFile);
        // 反序列化
        deserializable(objectFile);
    }

    /**
     * 反序列化方法 ObjectInputStream
     *
     * @param objectFile
     */
    private static void deserializable(String objectFile) throws IOException, ClassNotFoundException {
        ObjectInputStream objectInputStream = new ObjectInputStream(new FileInputStream(objectFile));
        Object student = (Student) objectInputStream.readObject();
        System.out.println(student);

        objectInputStream.close();

    }

    /**
     * 序列化方法 ObjectOutputStream
     *
     * @param objectFile
     */
    private static void serializable(String objectFile) throws IOException {
        Student student = new Student("Adopat", 20, 1);
        ObjectOutputStream objectOutputStream = new ObjectOutputStream(new FileOutputStream(objectFile));
        objectOutputStream.writeObject(student);
        objectOutputStream.close();
    }

    private static class Student implements Serializable {
        private String name;
        private int age;
        // transient 关键字可以使一些属性不会被序列化。
        private transient int sex;

        public Student(String name, int age, int sex) {
            this.name = name;
            this.age = age;
            this.sex = sex;
        }

        @Override
        public String toString() {
            return "Student{" +
                    "name='" + name + '\'' +
                    ", age=" + age +
                    ", sex=" + sex +
                    '}';
        }
    }

    # 11.6 NIO&AIO

    Java NIO和AIO是Java中用于实现非阻塞I/O的技术，应该从以下几个方面进行掌握：

    # 11.6.1 基本概念

    了解NIO和AIO的基本概念、原理和实现机制，理解它们与传统I/O的差异和优势。

    • NIO基本概念

      1. 缓冲区（Buffer）：NIO中的缓冲区是一个数组，用于存储数据。缓冲区可以读写数据，并且可以通过通道进行传输。
      2. 通道（Channel）：NIO中的通道是双向的，可以进行读写操作。通道可以实现非阻塞式的I/O操作，提高了程序的并发性能。
      3. 选择器（Selector）：NIO中的选择器是一个可以监控多个通道的对象，可以注册多个通道，并且可以使用一个线程来处理多个通道的I/O操作

    • NIO原理和NIO实现机制

      1. 非阻塞I/O：NIO采用非阻塞式I/O，即当一个线程调用read()或write()方法时，该线程不会等待，而是立即返回，可以继续执行其他任务。
      2. 缓冲区和通道：NIO中的缓冲区和通道可以提高I/O操作的效率，减少系统调用次数。NIO中的通道可以注册到选择器中，实现非阻塞式的I/O操作。
      3. 选择器：NIO中的选择器可以实现多路复用I/O操作，提高了程序的并发性能。选择器可以注册多个通道，并且可以使用一个线程来处理多个通道的I/O操作。

    • AIO基本概念

      1. 异步I/O：AIO采用异步式I/O，即当一个线程调用read()或write()方法时，该线程不会阻塞等待，而是立即返回。当I/O操作完成后，会通知线程进行处理。
      2. CompletionHandler：AIO中的CompletionHandler是一个回调函数，用于处理异步I/O操作完成后的结果。

    • AIO原理和实现机制

      1. 异步I/O：AIO采用异步式I/O，可以提高程序的并发性能。当一个I/O操作完成后，会通知线程进行处理，避免了线程阻塞等待。
      2. CompletionHandler：AIO中的CompletionHandler是一个回调函数，用于处理异步I/O操作完成后的结果。当I/O操作完成后，会调用CompletionHandler的回调方法进行处理。
      3. 操作系统支持：AIO需要操作系统的支持，目前仅在Java 7及以上版本的操作系统中提供了对AIO的支持

    • AIO&NIO总结

      NIO和AIO都可以提高程序的并发性能，适用于高并发的网络应用程序。NIO适用于需要实现高并发的网络应用程序，而**AIO适用于需要处理大量的异步I/O操作**的应用程序。

    • 与传统I/O对比

      1. 阻塞与非阻塞：传统I/O是阻塞式的，即当一个线程调用read()或write()方法时，该线程会一直等待，直到操作完成。而NIO和AIO是非阻塞式的，即当一个线程调用read()或write()方法时，该线程不会等待，而是立即返回，可以继续执行其他任务。
      2. 缓冲区：传统I/O是基于字节流和字符流的，而NIO是基于缓冲区的。NIO中的缓冲区可以读写数据，并且可以通过通道进行传输。缓冲区可以提高I/O操作的效率，减少系统调用次数。
      3. 通道：传统I/O中的输入输出流是单向的，而NIO中的通道是双向的，可以进行读写操作。通道可以实现非阻塞式的I/O操作，提高了程序的并发性能。
      4. 选择器：NIO中的选择器是一个可以监控多个通道的对象，可以注册多个通道，并且可以使用一个线程来处理多个通道的I/O操作，提高了程序的并发性能。

    • Java AIO中的异步I/O的概念

      异步I/O是一种非阻塞I/O的实现方式，它可以让应用程序在等待I/O操作完成的同时，继续执行其他任务，从而提高应用程序的吞吐量和响应速度。在Java AIO中，异步I/O是通过AsynchronousChannel和CompletionHandler接口实现的。

    • Java AIO中的使用方法(给出实例代码)

      使用Java AIO进行异步I/O操作的一般步骤如下：

      1. 打开一个异步通道（如AsynchronousSocketChannel或AsynchronousServerSocketChannel）。
      2. 调用异步通道的read或write方法，传入一个ByteBuffer和一个CompletionHandler对象。
      3. 在CompletionHandler对象中实现completed方法和failed方法，分别处理I/O操作成功和失败的情况。
      4. 在completed方法中，读取或写入数据，并再次调用异步通道的read或write方法，以实现连续的异步I/O操作。

      下面是一个简单的Java AIO客户端的实例代码：

      /**
 * @author Adopat
 * @description: Java AIO客户端代码
 * @date 2023/03/15 16:26
 **/
import java.io.IOException;
import java.net.InetSocketAddress;
import java.nio.ByteBuffer;
import java.nio.channels.AsynchronousSocketChannel;
import java.nio.channels.CompletionHandler;

public class AioClient {
    public static void main(String[] args) throws Exception {
        AsynchronousSocketChannel client = AsynchronousSocketChannel.open();
        InetSocketAddress serverAddr = new InetSocketAddress("localhost", 9999);
        client.connect(serverAddr, null, new CompletionHandler<Void, Void>() {
            @Override
            public void completed(Void result, Void attachment) {
                System.out.println("客户端连接成功");
                ByteBuffer buffer = ByteBuffer.allocate(1024);
                client.read(buffer, buffer, new CompletionHandler<Integer, ByteBuffer>() {
                    @Override
                    public void completed(Integer result, ByteBuffer attachment) {
                        attachment.flip();
                        String message = new String(attachment.array()).trim();
                        System.out.println("客户端收到消息：" + message);
                        try {
                            client.close();
                        } catch (IOException e) {
                            e.printStackTrace();
                        }
                    }
                    @Override
                    public void failed(Throwable exc, ByteBuffer attachment) {
                        System.out.println("客户端读取消息失败：" + exc);
                        try {
                            client.close();
                        } catch (IOException e) {
                            e.printStackTrace();
                        }
                    }
                });
            }
            @Override
            public void failed(Throwable exc, Void attachment) {
                System.out.println("客户端连接失败：" + exc);
                try {
                    client.close();
                } catch (IOException e) {
                    e.printStackTrace();
                }
            }
        });
        Thread.sleep(1000);
    }
}

import java.io.IOException;
import java.net.InetSocketAddress;
import java.nio.ByteBuffer;
import java.nio.channels.AsynchronousServerSocketChannel;
import java.nio.channels.AsynchronousSocketChannel;
import java.nio.channels.CompletionHandler;

public class AioServer {
    public static void main(String[] args) throws Exception {
        AsynchronousServerSocketChannel server = AsynchronousServerSocketChannel.open();
        InetSocketAddress addr = new InetSocketAddress("localhost", 9999);
        server.bind(addr);
        System.out.println("服务器已启动，监听端口 " + addr.getPort());
        server.accept(null, new CompletionHandler<AsynchronousSocketChannel, Void>() {
            @Override
            public void completed(AsynchronousSocketChannel client, Void attachment) {
                System.out.println("客户端连接成功：" + client);
                server.accept(null, this);
                ByteBuffer buffer = ByteBuffer.allocate(1024);
                client.read(buffer, buffer, new CompletionHandler<Integer, ByteBuffer>() {
                    @Override
                    public void completed(Integer result, ByteBuffer attachment) {
                        attachment.flip();
                        String message = new String(attachment.array()).trim();
                        System.out.println("服务器收到消息：" + message);
                        client.write(ByteBuffer.wrap(("服务器收到消息：" + message).getBytes()));
                    }
                    @Override
                    public void failed(Throwable exc, ByteBuffer attachment) {
                        System.out.println("客户端读取消息失败：" + exc);
                        try {
                            client.close();
                        } catch (IOException e) {
                            e.printStackTrace();
                        }
                    }
                });
            }
            @Override
            public void failed(Throwable exc, Void attachment) {
                System.out.println("客户端连接失败：" + exc);
            }
        });
        Thread.sleep(1000);
    }
}
