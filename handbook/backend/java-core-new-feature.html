    # 9.1 Lambda 表达式

    • 概念

      Lambda 允许把函数作为一个方法的参数（函数作为参数传递到方法中）。

    • 语法

      //Lambda表达式的语法比较简单，由一个参数列表、一个箭头符号和一个表达式组成
// 1. 不需要参数,返回值为 5  
() -> 5  
  
// 2. 接收一个参数(数字类型),返回其2倍的值  
x -> 2 * x  
  
// 3. 接受2个参数(数字),并返回他们的差值  
(x, y) -> x – y  
  
// 4. 接收2个int型整数,返回他们的和  
(int x, int y) -> x + y  
  
// 5. 接受一个 string 对象,并在控制台打印,不返回任何值(看起来像是返回void)  
(String s) -> System.out.print(s)

    /**
     * 函数式接口
     */

            @Override
            public int apply(int x, int y) {

        };

        myFunction1.apply(1, 2);

    /**
     * 方法引用
     */
    public static void test3() {
        List<String> list = Arrays.asList("apple", "cherry", "banana");
        // 使用方法引用
        list.forEach(System.out::println);
        // 使用Lambda表达式
        list.forEach(s -> System.out.println(s));
    }

    /**
     * 集合排序
     */
    private static void test4() {
        List<String> list = Arrays.asList("apple", "cherry", "banana");
        list.sort(new Comparator<String>() {
            @Override
            public int compare(String o1, String o2) {
                // 降序排列
                return o2.compareTo(o1);
            }
        });
        System.out.println("不使用Lambda进行降序排序:" + list);
        List<String> list2 = Arrays.asList("apple", "cherry", "banana");
        // 升序
        list2.sort((o1, o2) -> o1.compareTo(o2));
        System.out.println("使用Lambda进行升序排序:" + list2);
    }

}

    # 9.3 函数式接口

    • 概念

      函数式接口是指只有一个抽象方法的接口，但是可以有多个非抽象方法的接口。可以用Lambda表达式来实现。

      在 Java 8 及之后的版本中，推荐使用 Lambda 表达式来实现函数式接口。

    • 语法

      @FunctionalInterface
interface MyFunction {
    int apply(int x, int y);
}
      1
      2
      3
      4

      注解 @FunctionalInterface 的作用

      @FunctionalInterface注解的作用是标识一个接口是函数式接口，这样可以在编译时检查该接口是否符合函数式接口的定义。

      @FunctionalInterface注解有一个规则，即该注解只能标识一个接口，并且该接口只能有一个抽象方法。如果接口中有多个抽象方法，或者没有抽象方法，都会导致编译错误。

      使用Java @FunctionalInterface注解可以在编译时检查接口是否符合函数式接口的定义，从而提高代码的可读性和可维护性。

    • 常见函数式接口

      JDK 1.8 之前已有的函数式接口:

      • java.lang.Runnable(创建线程,重点关注)
      • java.util.concurrent.Callable
      • java.security.PrivilegedAction
      • java.util.Comparator(比较器,重点关注)
      • java.io.FileFilter(文件过滤器，重点关注)
      • java.nio.file.PathMatcher
      • java.lang.reflect.InvocationHandler
      • java.beans.PropertyChangeListener
      • java.awt.event.ActionListener
      • javax.swing.event.ChangeListener

      JDK 1.8 新增加的函数接口：

      • java.util.function

    • 使用

      @FunctionalInterface
interface MyFunction {
    int apply(int x, int y);
}

MyFunction f = (x, y) -> x + y;
int result = f.apply(1, 2);

    # 9.4 默认方法

    • 概念

      默认方法就是接口可以有实现方法，而且不需要实现类去实现其方法。我们只需在方法名前面加个 default 关键字即可实现默认方法。

    • 语法

      public interface Vehicle {
   default void print(){
      System.out.println("我是一辆车!");
   }
    // 静态方法
   static void blowHorn(){
      System.out.println("按喇叭!!!");
   }
}

        Vehicle.blowHorn();
    }

    # 9.5 Stream

    • 概念

      Stream是Java 8中的一个新API，它可以将数据源（如集合、数组）中的元素进行转换、过滤、排序等操作，最终生成一个新的数据流

    • 流的创建方式

      1. 使用Collection接口的stream()方法创建流。
      List<String> list = Arrays.asList("a", "b", "c");
Stream<String> stream = list.stream();
      1
      2
      1. 使用Arrays类的stream()方法创建流。
      String[] array = {"a", "b", "c"};
Stream<String> stream = Arrays.stream(array);
      1
      2
      1. 使用Stream类的of()方法创建流。
      Stream<String> stream = Stream.of("a", "b", "c");
      1
      1. 使用Stream类的generate()方法创建流，该方法接受一个Supplier类型的参数，用于生成流中的元素。
      Stream<String> stream = Stream.generate(() -> "a").limit(3);
      1
      1. 使用Stream类的iterate()方法创建流，该方法接受一个初始值和一个UnaryOperator类型的参数，用于生成流中的元素。
      Stream<Integer> stream = Stream.iterate(0, n -> n + 2).limit(3);
      1
      1. 使用Stream类的concat()方法将两个流合并成一个流。
      Stream<String> stream1 = Stream.of("a", "b");
Stream<String> stream2 = Stream.of("c", "d");
Stream<String> stream3 = Stream.concat(stream1, stream2);
      1
      2
      3
      1. 使用Files类的lines()方法读取文件内容并生成流。
      Stream<String> stream = Files.lines(Paths.get("file.txt"));
      1

      以上是常用的几种创建流的方式，根据不同的场景和需求选择合适的方式。

      以上是Stream的常用操作及对应实例代码，这些操作可以方便地对集合进行转换、过滤、排序等操作。

      1.long count = Stream.of("a", "b", "c").collect(Collectors.counting());

      2.Arrays.asList("a","b","c").stream().count();

      第一行代码中，首先使用Stream类的of创建流，然后使用了Collectors的counting()方法，将流中的元素收集到一个Collector中，并返回元素个数。

      第二行代码中，首先使用List的stream()方法创建流，然后使用了Stream的count()方法，直接返回流中元素的个数。

      在这个例子中，两种方法的结果是相同的，但在其他情况下，它们可能会有所不同。例如，对于一个无限长度的流，使用count()方法可能会导致程序无法结束，而使用Collectors的counting()方法则不会出现这种情况。一般情况下建议使用流工具类对流进行操作，不建议直接使用流的操作方法。

    • Stream的中间操作和终止操作

      • 中间操作

        中间操作是对数据流进行处理的操作，返回一个新的Stream对象。中间操作可以进行链式操作，可以实现一系列的转换和处理，最终生成一个新的数据流。例如，对数据流进行过滤、转换、排序等操作。

        List<Integer> list = Arrays.asList(1, 2, 3, 4, 5);
List<Integer> newList = list.stream()
    .filter(i -> i % 2 == 0)
    .map(i -> i * i)
    .sorted()
    .collect(Collectors.toList());
// newList: [4, 16]

      • 终止操作

        终止操作会触发数据流的处理，返回一个非Stream类型的结果。终止操作可以对数据流进行统计、查找、聚合等操作，最终生成一个结果。例如，对数据流进行统计元素个数、查找最大值、聚合元素求和等操作。

        List<Integer> list = Arrays.asList(1, 2, 3, 4, 5);
long count = list.stream().count();
// count: 5

List<Integer> list = Arrays.asList(1, 2, 3, 4, 5);
Optional<Integer> max = list.stream().max(Integer::compare);
// max: 5

List<Integer> list = Arrays.asList(1, 2, 3, 4, 5);
int sum = list.stream().mapToInt(Integer::intValue).sum();
// sum: 15
List<String> list = Arrays.asList("apple", "banana", "orange");
List<String> newList = list.stream().filter(s -> s.startsWith("a")).collect(Collectors.toList());

    • Stream的并行处理

      • 并行流的创建

        可以通过调用Stream接口的parallel()方法将串行流转换为并行流，也可以通过调用Arrays类或Collection接口的parallelStream()方法创建并行流。

        List<Integer> list = Arrays.asList(1, 2, 3, 4, 5);
Stream<Integer> parallelStream = list.parallelStream();
        1
        2

      • 并行流的操作

        与串行流类似，可以对并行流进行各种操作，如过滤、映射、排序、去重等。需要注意的是，在并行流中使用无状态操作（如filter、map等）可以提高并行效率，而使用有状态操作（如sorted、distinct等）可能会降低并行效率。

        List<Integer> list = Arrays.asList(1, 2, 3, 4, 5);
Stream<Integer> parallelStream = list.parallelStream();
parallelStream.filter(x -> x % 2 == 0).map(x -> x * x).forEach(System.out::println);
        1
        2
        3

      • 并行流的并行度设置

        并行流的操作可能会占用大量的系统资源，因此需要合理地限制并行流的并行度，以避免对系统造成过大的影响。可以通过调用Stream接口的parallel()方法设置并行度，也可以通过系统属性设置并行度。

        /**
 * @author Adopat
 * @description: 并行流处理
 * @date 2023/03/16 15:50
 **/
public class ParallelStreamDemo {
    public static void main(String[] args) {
        // 1.创建并行流
        List<Integer> list = Arrays.asList(1, 2, 3, 4, 5);
        // 2.创建并行流
        Stream<Integer> integerStream = list.parallelStream();
        // 3.设置并行度2
        System.setProperty("java.util.concurrent.ForkJoinPool.common.parallelism", "2");
        List<Integer> collect = integerStream.filter(x -> x % 2 == 0).collect(Collectors.toList());
        //integerStream.filter(x -> x % 2 == 0).forEach(System.out::println);
        System.out.println("并行处理后的结果：" + collect);
    }
}

    # 9.6 Optional类

Optional<String> optOrNull = Optional.ofNullable(name);

    # 9.9 Base64

    • 基本概念

      Base64编码是一种将二进制数据转换为可打印字符的编码方式，常用于在网络传输中传输二进制数据。Base64编码的原理是将3个字节的二进制数据转换为4个可打印字符，每个可打印字符占6位，共24位，不足24位的数据用0补齐。Base64编码的特点是编码后的数据长度比原始数据长度大1/3左右，编码后的数据可以被直接传输和存储，而不需要担心数据丢失或损坏。

      Base64编码要求把3个8位字节（38=24）转化为4个6位的字节（46=24），之后在6位的前面补两个0，形成8位一个字节的形式。 如果剩下的字符不足3个字节，则用0填充，输出字符使用'='，因此编码后输出的文本末尾可能会出现1或2个'='。

    • Java API

      Java API：Java提供了java.util.Base64类来实现Base64编码和解码。该类包括一个静态的getEncoder()方法和getDecoder()方法，分别返回一个Base64.Encoder和一个Base64.Decoder对象，用于进行编码和解码操作。Base64.Encoder和Base64.Decoder类提供了多个重载的encode()和decode()方法，可以实现不同类型的数据的编码和解码。

      String originalString = "B";
String encodedString = Base64.getEncoder().encodeToString(originalString.getBytes());
System.out.println(encodedString);
String decodedString = new String(Base64.getDecoder().decode(encodedString));
System.out.println(decodedString);
// encodedString :Qg==
// decodedString :B

      B 的Base64 编码解释

      B 的 ASCII 码为66

      二进制 01000010 6位进行分组，不足24位用0 补齐 得到结果 010000 100000 000000 000000

      010000 转为10进制对应的码值为16 对应Base64字符为Q

      100000 转为10进制对应的码值为32 对应Base64字符为g

      000000 000000 补充的字符用 ==表示

      所以B 的Base64编码后的字符为 Qg==

      参考 base64编码对照表 (opens new window)

    • 应用场景

      Base64编码在实际应用中有很多常见的场景和用途。例如，在网络传输中，二进制数据不能直接传输，需要将其转换为可打印字符，使用Base64编码可以实现这一目的。在图片、音频、视频等文件中嵌入数据时，也可以使用Base64编码将数据转换为文本格式，方便嵌入到文件中。

    • 安全性考虑

      Base64编码不是加密算法，不能保证数据的绝对安全性。虽然Base64编码可以将二进制数据转换为可打印字符，但仍然可以通过对编码后的数据进行解码，还原出原始的二进制数据。因此，在实际应用中，需要结合其他加密算法来提高数据的安全性，例如使用SSL/TLS协议来保护网络传输数据的安全性。
