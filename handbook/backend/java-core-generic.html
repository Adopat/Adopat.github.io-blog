    # 6.1 泛型概念

    泛型是一种参数化类型的机制，可以在类定义、方法定义、接口定义中使用。通过使用泛型，可以使代码更加灵活、可重用、类型安全。

    # 6.3 泛型类型擦除

    • 泛型类型擦除概念

      Java 泛型在编译时会进行类型擦除，即将泛型类型转换为原始类型。例如，MyGenericClass 在编译时会被转换为 MyGenericClass。

    • Java泛型类型的擦除规则

      • 泛型类的类型参数在编译时期会被擦除为原始类型。
      • 泛型方法的类型参数在编译时期会被擦除为Object类型。
      • 泛型数组的类型参数在编译时期会被擦除为Object类型。
      • 泛型类型的擦除时机是在编译时期，而不是运行时期。
      • 泛型类型的擦除后，会进行类型转换，例如List<String>会被转换为List。
      • 泛型类型的擦除后，无法获取到类型参数的具体信息，例如无法通过反射获取到List<String>的类型参数String。
      /**
 * @author Adopat
 * @description: Java 泛型类型擦除
 * @date 2023/03/20 15:32
 **/
public class Main {
    public static void main(String[] args) {
        MyClass<String> java = new MyClass<>("java");
        System.out.println(java.getClass());

    }
}

class MyClass<T> {
    private T data;

    public MyClass(T data) {
        this.data = data;
    }

    public T getData() {
        return data;
    }

    public void setData(T data) {
        this.data = data;
    }

    • 泛型类型擦除带来的问题和解决方案

      1. 无法在运行时检查泛型类型：由于泛型类型被擦除，导致在运行时无法检查具体的泛型类型，只能检查Object类型，这可能会导致类型安全问题。

        List<String> stringList = new ArrayList<>();
stringList.add("hello");
List<Object> objectList = (List<Object>) stringList;
objectList.add(1);
String s = stringList.get(1); // 运行时会抛出ClassCastException
        1
        2
        3
        4
        5

        在上面的例子中，我们将List<String>强制转换成了List<Object>，并向其中添加了一个整数。由于泛型类型被擦除，objectList在运行时无法检查其具体的泛型类型，因此可以添加一个整数。在获取索引为1的元素时，会抛出ClassCastException。

      2. 无法区分不同类型的泛型：由于类型被擦除，导致编译器无法区分不同类型的泛型，例如List<String>和List<Integer>在运行时都会被当做List<Object>处理。

        List<String> stringList = new ArrayList<>();
stringList.add("hello");
List<Integer> integerList = new ArrayList<>();
integerList.add(1);
System.out.println(stringList.getClass() == integerList.getClass()); // true
        1
        2
        3
        4
        5

        在上面的例子中，我们创建了一个List<String>和一个List<Integer>，并比较了它们的类对象。由于泛型类型被擦除，stringList和integerList在运行时都会被当做List<Object>处理，因此它们的类对象是相同的，输出结果为true。

      3. 无法创建泛型数组：由于泛型类型被擦除，导致无法创建泛型数组，例如new T[]会导致编译错误。

        T[] array = new T[10]; // 编译错误，无法创建泛型数组
        1

        在上面的例子中，我们尝试创建一个大小为10的泛型数组，但由于泛型类型被擦除，编译器无法确定具体的泛型类型，因此无法创建泛型数组。

      4. 无法使用基本类型作为泛型类型：由于泛型类型被擦除，导致无法使用基本类型作为泛型类型，例如List<int>会导致编译错误。

        List<int> intList = new ArrayList<>(); // 编译错误，无法使用基本类型作为泛型类型
        1

        在上面的例子中，我们尝试使用基本类型int作为泛型类型，但由于泛型类型被擦除，编译器无法确定具体的泛型类型，因此无法使用基本类型作为泛型类型。

      5. 无法使用运行时类型信息（RTTI）：由于泛型类型被擦除，导致无法使用运行时类型信息，例如instanceof操作符无法区分不同类型的泛型。

        List<String> stringList = new ArrayList<>();
System.out.println(stringList instanceof List<String>); // 编译错误，无法使用instanceof操作符检查泛型类型
        1
        2

        在上面的例子中，我们尝试使用instanceof操作符检查stringList的具体泛型类型，但由于泛型类型被擦除，无法使用运行时类型信息，因此编译器会报错。

    # 6.4 泛型类型的编译时期和运行时期的差异

    Java泛型类型在编译时期和运行时期有一些差异。在编译时期，Java编译器会进行类型检查和类型擦除；在运行时期，Java虚拟机只能获取到泛型类型的原始类型信息。

    /**
 * @author Adopat
 * @description: Java泛型类型的编译时期和运行时期的差异
 * @date 2023/03/20 15:32
 **/
public class Main {
    public static void main(String[] args) {
        MyClass<String> myClass = new MyClass<>("java");
        // class java.lang.String
        System.out.println(myClass.getData().getClass().getName());
        // class java_basic.generics.MyClass
        System.out.println(myClass.getClass());

    }
}

class MyClass<T> {
    private T data;

    public MyClass(T data) {
        this.data = data;
    }

    public T getData() {
        return data;
    }

    public void setData(T data) {
        this.data = data;

    # 6.5 泛型类型的反射机制

    Java 泛型类型的反射机制是指通过反射机制来获取泛型类型信息的方法。在 Java 中，泛型类型在编译时会被擦除，因此在运行时无法直接获取泛型类型的信息。但是，通过反射机制，可以间接地获取泛型类型的信息。

    • 获取类的泛型参数类型

      /**
 * @author Adopat
 * @description: getGenericSuperclass() 获取泛型超类
 * @date 2023/03/20 16:33
 **/
public class Test {
    public static void main(String[] args) {
        Student stu = new Student();
        Class clazz = stu.getClass();
        // student类：class java_basic.generics.Student
        System.out.println("student类："+clazz);
        // Student的父类:class java_basic.generics.Person
        System.out.println("Student的父类:"+clazz.getSuperclass());
        // Student的Generic父类:class java_basic.generics.Person
        System.out.println("Student的Generic父类:"+clazz.getSuperclass());
        // 使用getGenericSuperclass()方法获取类对象的泛型超类，这将返回一个Type对象，表示泛型超类
        Type type = clazz.getGenericSuperclass();
        // 使用ParameterizedType接口将Type对象强制转换为ParameterizedType对象。这样我们就可以访问泛型超类的实际类型参数。
        ParameterizedType p = (ParameterizedType) type;
        // 通过调用ParameterizedType对象的getActualTypeArguments()方法来获取泛型超类的第一个实际类型参数，并从结果数组中获取第一个元素。这将返回一个Class对象。
        Class c = (Class)p.getActualTypeArguments()[0];
        // c =class java_basic.generics.Student
        System.out.println("c ="+c);

    }
}
class Person<T>{

}
class Student extends Person<Student>{

}

    • 获取字段的泛型类型

      import java.lang.reflect.Field;
import java.lang.reflect.ParameterizedType;
import java.lang.reflect.Type;
import java.util.List;

/**
 * @author Adopat
 * @description: getGenericType() 用于获取字段、方法或构造函数的泛型类型。
 * @date 2023/03/21 09:28
 **/
public class GetGenericTypeDemo {
    public static void main(String[] args) throws NoSuchFieldException {
        Field field = ExampleDemo.class.getDeclaredField("list");
        Type type = field.getGenericType();
        if (type instanceof ParameterizedType) {
            ParameterizedType pType = (ParameterizedType) type;
            Type[] actualTypeArguments = pType.getActualTypeArguments();
            for (Type t : actualTypeArguments) {
                System.out.println(t.getTypeName());
            }
        }
    }
}
class ExampleDemo<T> {
    private List<T> list;
}

    • 获取方法的泛型参数类型和泛型返回类型

      import java.lang.reflect.Method;
import java.lang.reflect.ParameterizedType;
import java.lang.reflect.Type;
import java.util.List;

/**
 * @author Adopat
 * @description: getGenericReturnType()方法获取该方法的返回类型的泛型信息.
 * @date 2023/03/21 09:44
 **/
public class GetGenericReturnTypeDemo {
    public static void main(String[] args) throws NoSuchMethodException {
        Method method = ExampleDemo1.class.getDeclaredMethod("getList", String.class);
        // getGenericReturnType()方法获取该方法的返回类型的泛型信息
        Type returnType = method.getGenericReturnType();
        if (returnType instanceof ParameterizedType) {
            ParameterizedType pType = (ParameterizedType) returnType;
            Type[] actualTypeArguments = pType.getActualTypeArguments();
            for (Type t : actualTypeArguments) {
                System.out.println(t.getTypeName());
            }
        }
        // 使用getGenericParameterTypes()方法获取getList方法的参数类型的泛型信息
        Type[] parameterTypes = method.getGenericParameterTypes();
        for (Type type : parameterTypes) {
            if (type instanceof Class) {
                System.out.println(((Class) type).getName());
            } else if (type instanceof ParameterizedType) {
                ParameterizedType pType = (ParameterizedType) type;
                Type[] actualTypeArguments = pType.getActualTypeArguments();
                for (Type t : actualTypeArguments) {
                    System.out.println("类型: "+t.getTypeName());
                }
            }
        }

    }
}

    # 6.6 泛型的限制和约束

    • 限定通配符

      <? extends T>:确保类型必须是T及T的子类来设定类型的上界

      import java.util.ArrayList;

/**
 * @author Adopat
 * @description: PECS(Producer Extends Consumer Super)原则
 * @date 2023/03/20 15:16
 **/
public class GenericExample {
    public static void main(String[] args) {
        // Producer Extends：如果你需要一个只读List，用它来produce T，那么使用? extends T。
        ArrayList<? extends Fruits> list = new ArrayList();
        // 除了null 其他都不能增加
        list.add(null);
    }
}
class Fruits{
    private String name;

    public Fruits() {
    }

    public Fruits(String name) {
        this.name = name;
    }

    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }

    @Override
    public String toString() {
        return "Fruits{" +
                "name='" + name + '\'' +
                '}';
    }
}
class Apple extends Fruits{

}
class Orange extends Fruits{

    • 非限定通配符

      ?表示通配符，表示任意类型。通常用于方法参数或返回类型中，表示可以接受或返回任意类型的值。它的作用等同于没有使用泛型

      public void printList(List<?> list) {
    for (Object obj : list) {
        System.out.println(obj);
