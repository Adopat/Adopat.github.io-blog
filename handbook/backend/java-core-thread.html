    # 第十二章 并发编程

    # 12.1 一些基本概念

    # 12.1.1 CPU的一些常见参数及解析

    • CPU型号：CPU型号通常由制造商和系列名称组成，例如Intel的Core i7系列，AMD的Ryzen系列等。不同的CPU型号有不同的性能和功能，因此选择适合自己的CPU型号非常重要。

    • CPU频率：CPU频率指的是CPU的时钟频率，通常以GHz为单位。CPU频率越高，处理器每秒钟可以执行的指令数就越多，性能也就越好。

    • CPU核心数：CPU核心数指的是CPU中的物理处理器数量。例如，一个双核CPU就有两个物理处理器，一个四核CPU就有四个物理处理器。CPU核心数越多，可以同时处理的任务数量就越多，性能也就越好。

    • 线程数：线程数指的是CPU中的逻辑处理器数量。例如，一个物理处理器可以使用超线程技术模拟出两个逻辑处理器，因此一个双核四线程CPU实际上只有两个物理处理器，但可以模拟出四个逻辑处理器。线程数越多，可以同时处理的任务数量也就越多。

    • 缓存：CPU缓存是一种高速缓存，用于暂存CPU中经常使用的数据和指令。CPU缓存分为多级，包括L1、L2、L3等。缓存越大，可以暂存的数据和指令就越多，CPU也就越快。

    • 制造工艺：制造工艺指的是CPU芯片制造的工艺技术，通常以纳米（nm）为单位。制造工艺越小，CPU的功耗就越低，性能也就越好。

    • TDP：TDP（Thermal Design Power）指的是CPU的热设计功耗，即CPU在正常工作状态下的最大热量产生能力。TDP越高，CPU的发热量就越大，需要更好的散热系统来保持CPU的稳定工作。

    • 支持的指令集：指令集是CPU可以执行的指令的集合。不同的CPU支持不同的指令集，例如，Intel的SSE、AVX指令集等，AMD的MMX、SSE等指令集。支持更多的指令集，CPU可以执行更多的指令，性能也就越好。

    # 12.1.2 线程概念

    Java线程是一个轻量级的子进程，是程序执行的最小单位，是操作系统资源分配的基本单位。每个Java程序都至少有一个线程，称为主线程。Java线程是通过Thread类来实现的，可以通过继承Thread类或实现Runnable接口来创建线程。

    # 12.1.3 线程的生命周期

    Java线程的生命周期包括5个状态：新建状态、就绪状态、运行状态、阻塞状态和死亡状态。

    • 新建状态：当我们创建一个Thread对象时，线程处于新建状态。此时线程尚未启动，也没有进入就绪状态。新建状态的线程可以通过调用start()方法来启动线程。
    • 就绪状态：当线程调用start()方法后，线程处于就绪状态。此时线程已经准备好了，等待CPU调度执行。就绪状态的线程可以被CPU调度执行，也可以被阻塞。
    • 运行状态：当线程被CPU调度执行时，线程处于运行状态。此时线程正在执行任务，在这个阶段中线程会不断地执行任务，直到任务完成或者被阻塞。
    • 阻塞状态：线程在执行任务时，如果遇到某些情况（如等待IO、等待锁、等待信号等），就会进入阻塞状态。此时线程会停止执行，等待条件满足后再次进入
    • 就绪状态。
    • 死亡状态：当线程执行完任务或异常退出时，线程进入死亡状态。此时线程已经完成了自己的任务，不再执行任何操作。线程一旦进入死亡状态就不能再被启动。

    # 12.1.4 线程和进程的区别

    进程是程序在计算机中的一次执行活动，是资源分配的基本单位。一个进程可以包含多个线程，每个线程都是执行进程中的一个单独的任务。线程是进程中的执行单元，是CPU调度和分派的基本单位。线程拥有自己的栈空间和程序计数器，但共享进程的其他资源，如内存、文件等。多线程可以提高应用程序的性能和响应速度，因为它可以使多个任务同时执行。

    # 12.1.5 同步和异步区别

    同步和异步是指程序执行的方式。同步是指程序按照顺序执行，每个任务必须等待前一个任务完成后才能执行，任务之间的执行顺序是固定的。异步是指程序不按照顺序执行，任务可以同时执行，任务之间的执行顺序是不确定的。异步执行任务通常会提供一个回调函数，任务完成后会调用该函数。

    同步和异步的区别最大在于异步的话调用者不需要等待处理结果，被调用者会通过回调等机制来通知调用者其返回结果。

    # 12.1.6 阻塞和非阻塞区别

    阻塞和非阻塞是指程序在执行某个操作时的状态。阻塞是指程序在执行某个操作时被挂起，等待某个条件的满足才能继续执行。非阻塞是指程序在执行某个操作时不会被挂起，可以立即继续执行其他任务。阻塞和非阻塞通常用于I/O操作。例如，当程序从磁盘中读取文件时，如果是阻塞模式，程序会被挂起，等待文件读取完成后才能继续执行；如果是非阻塞模式，程序可以继续执行其他任务，但需要定时检查文件读取是否完成。

    举个生活中简单的例子：

    你妈妈让你烧水，小时候你比较笨啊，在哪里傻等着水开（ 同步阻塞 ）。等你稍微再长大一点，你知道每次烧水的空隙可以去干点其他事，然后只需要时不时来看看水开了没有（ 同步非阻塞 ）。后来，你们家用上了水开了会发出声音的壶，这样你就只需要听到响声后就知道水开了，在这期间你可以随便干自己的事情（ 异步非阻塞 ）。

    # 12.1.7 并发并行区别

    并行和并发都指的是同时处理多个任务的能力，但它们的实现方式和应用场景有所不同。 并发是指系统能够同时处理多个任务，但这些任务并不是同时进行的，而是通过快速地切换和分配CPU时间片来实现看似同时进行的效果。例如，多个线程在同一时间段内交替执行，每个线程执行一段时间后，就会让出CPU资源，切换到下一个线程执行。这样，虽然每个线程的执行时间不长，但由于它们的执行时间相互交错，看起来就像是同时执行一样。 而并行是指系统中有多个处理器或计算核心，可以同时处理多个任务，每个任务都可以在一个独立的处理器或核心上运行。例如，一个多核CPU可以同时执行多个线程或进程，每个线程或进程可以在不同的核心上并行执行，同时完成任务。

    综上所述，虽然并发和并行都是指同时处理多个任务，但并发是通过快速切换和分配CPU时间片来实现的，而并行是通过多个处理器或计算核心同时处理多个任务来实现的。在实际应用中，需要根据具体的场景和需求选择并发和并行的实现方式。

    # 12.1.8 多线程的目的

    多线程的意义并不在于提高访问速度，而在于提高系统的并发性和处理能力。多线程可以让多个任务同时执行，缩短响应时间，提高系统的并发性和吞吐量，从而提高系统的处理能力和性能。

    多线程加锁确实会降低访问速度，但是多线程的意义并不在于提高访问速度，而在于提高系统的并发性和处理能力。多线程可以让多个任务同时执行，缩短响应时间，提高系统的并发性和吞吐量，从而提高系统的处理能力和性能。当然，在多线程并发执行的过程中，可能会出现数据竞争等线程安全问题，需要采用加锁等方法来解决。但是，这并不影响多线程的优势和意义。总之，多线程是一种有效的提高系统并发性和处理能力的方式，加锁等线程安全机制则是保证多线程正确运行的前提。

    # 12.1.9 多线程的使用场景

    • CPU密集型任务

      如果任务需要大量的CPU计算，而且计算量比较大，这时候就需要使用多线程来提高程序的执行效率。例如：图像处理、视频编码、加密解密等。

    • IO密集型任务

      如果任务需要大量的IO操作，例如：读写文件、网络通信等，这时候也需要使用多线程来提高程序的执行效率。

    • GUI应用程序

      GUI应用程序需要响应用户的交互操作，同时还需要进行各种计算和数据处理。如果在主线程中处理所有的逻辑，就会导致界面卡顿或者无响应。因此，在GUI应用程序中使用多线程来处理计算和数据处理任务是非常必要的。

    • 处理大数据量

      如果需要处理大量的数据，单线程可能会导致内存溢出或者程序崩溃。这时候可以使用多线程来分批处理数据，从而避免内存溢出和程序崩溃。

    • 并发访问资源

      如果多个线程需要同时访问共享资源，例如：数据库、文件、共享变量等，这时候就需要使用多线程来处理并发访问问题，从而避免数据不一致或者死锁等问题。

    总之，多线程的使用场景非常广泛，只要是需要提高程序的执行效率或者避免并发问题的场景，都可以使用多线程来解决。

    # 12.1.10 并发并行同步异步阻塞非阻塞多线程多进程单核CPU多核CPU之间的联系

    • 并发和并行：并发和并行都是指多个任务同时执行的情况，但并发是指在单个CPU上快速切换多个任务，而并行则是指多个物理处理器或计算核心同时执行不同的任务。

    • 同步和异步：同步和异步是指任务之间的协作方式。同步是指一个任务必须等待另一个任务完成后才能继续执行，而异步则是指任务之间相互独立，不需要等待其他任务的完成。

    • 阻塞和非阻塞：阻塞和非阻塞是指任务等待结果时的状态。阻塞是指任务等待结果时被挂起，无法继续执行其他任务，而非阻塞则是指任务等待结果时可以继续执行其他任务。

    • 多线程和多进程：多线程和多进程都是实现并发处理的方式。多线程是指在同一个进程中创建多个线程，共享进程资源，而多进程则是指在操作系统中创建多个独立的进程，每个进程有自己的独立资源。

    • 单核CPU和多核CPU：单核CPU只有一个物理处理器，可以通过多线程来实现并发处理，但是无法实现真正的并行处理。而多核CPU则有多个物理处理器或计算核心，可以实现真正的并行处理，同时也可以使用多线程来实现更高效的并发处理。

    综上所述，多线程、多进程、单核CPU、多核CPU等都是实现并发和并行处理的方式，而同步、异步、阻塞、非阻塞则是任务之间协作方式的不同表现。

    注意 在单核CPU上，无论创建了多少个线程，同一时刻只会有一个线程能够获取到CPU的资源来执行任务 通过快速地切换和分配CPU时间片来实现看似同时进行的效果。例如，多个线程在同一时间段内交替执行，每个线程执行一段时间后，就会让出CPU资源，切换到下一个线程执行。这样，虽然每个线程的执行时间不长，但由于它们的执行时间相互交错，看起来就像是同时执行一样

    # 12.1.11 使用多线程编程需要注意的点

    • 线程安全：多个线程同时访问共享数据时，可能会出现数据不一致或者数据异常的情况，因此需要采用一些同步机制来保证线程安全，例如使用synchronized关键字或者Lock接口进行加锁操作。

    • 死锁问题：如果多个线程之间相互等待对方释放锁，就会发生死锁问题，导致程序无法继续执行。因此需要避免出现死锁问题，例如按照固定的顺序获取锁、尽量减少锁的持有时间等。

    • 线程间通信：多个线程之间需要进行通信，例如使用wait、notify或者BlockingQueue等机制，来实现线程之间的协作。

    • 线程池使用：为了避免频繁地创建和销毁线程所带来的资源消耗，可以使用线程池来管理线程，提高程序运行效率。

    • 性能问题：多线程的程序往往需要考虑性能问题，例如需要合理地设置线程数量、调整线程优先级、减少线程上下文切换等，才能达到更好的性能表现。 综上所述，多线程编程需要注意线程安全、死锁问题、线程间通信、线程池使用以及性能问题等方面，才能保证程序的正确性和性能表现。

      /**

public class MyThread extends Thread {

        t1.setName("鲁班");

        t3.setName("亚瑟");
        // 启动线程
        t1.start();
        t2.start();
        t3.start();
    }

}

    • 实现 Runnable 接口，重写 run() 方法

      /**
 * @author Adopat
 * @description: 创建线程的第二种方法 实现Runnable 接口
 * @date 2023/03/06 16:25
 **/
public class MyRunnable implements Runnable {
    @Override
    public void run() {
        for (int i = 0; i < 10; i++) {
            try {//sleep会发生异常要显示处理 暂停20毫秒
                Thread.sleep(20);
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
            System.out.println(Thread.currentThread().getName() + "打了:" + i + "个小兵");
        }
    }

    public static void main(String[] args) {
        //创建MyRunnable类
        MyRunnable mr = new MyRunnable();
        //创建Thread类的有参构造,并设置线程名
        Thread t1 = new Thread(mr, "张飞");
        Thread t2 = new Thread(mr, "貂蝉");
        Thread t3 = new Thread(mr, "吕布");
        //启动线程
        t1.start();
        t2.start();
        t3.start();

    }
}

    • 实现 Callable 接口，重写 call() 方法

      import java.util.concurrent.Callable;
import java.util.concurrent.ExecutionException;
import java.util.concurrent.FutureTask;

/**
 * @author Adopat
 * @description: 创建线程的第二种方法 实现Callable 接口
 * 需要实现接口中的 call() 方法。 call() 方法有返回值，返回值通过 FutureTask 进行封装。
 * @date 2023/03/06 16:35
 **/
public class CallerTask implements Callable<String> {
    @Override
    public String call() throws Exception {
        return "Hello,i am running!";
    }

    public static void main(String[] args) {
        //创建异步任务
        FutureTask<String> task=new FutureTask<String>(new CallerTask());
        //启动线程
        new Thread(task).start();
        try {
            //等待执行完成，并获取返回结果
            String result=task.get();
            System.out.println(result);
        } catch (InterruptedException e) {
            e.printStackTrace();
        } catch (ExecutionException e) {
            e.printStackTrace();
        }
    }
}

    # 12.2.2 Java线程常用方法

    • start()：启动线程，使线程进入就绪状态。

    • run()：定义线程的执行逻辑，线程启动后会自动调用该方法。

    • sleep(long millis)：让当前线程休眠一段时间，单位是毫秒。

    • interrupt()：中断线程，会设置线程的中断标志。

    • isInterrupted()：判断线程是否被中断。

    • join()：等待线程执行完成，主线程调用该方法后会等待被调用线程执行结束后再继续执行。

    • yield()：让出CPU执行权，让其他线程有机会执行。

    • setName(String name)：设置线程名称。

    • getName()：获取线程名称。

    • isAlive()：判断线程是否还在运行。

    • setPriority(int priority)：设置线程的优先级，范围是1~10。

    • getPriority()：获取线程的优先级。

    • setDaemon(boolean on)：设置线程是否为守护线程。

    • isDaemon()：判断线程是否为守护线程。

    • wait()：让线程等待，直到其他线程调用notify()或notifyAll()方法唤醒它。

    • notify()：唤醒一个等待的线程。

    • notifyAll()：唤醒所有等待的线程。

    • getState()：获取线程的状态，包括NEW、RUNNABLE、BLOCKED、WAITING、TIMED_WAITING、TERMINATED等状态。

      Java并发编程之可见性分析 volatile (opens new window)

      volatile 只能确保可见性，不能确保原子性 除了 volatile 和 synchronized，Lock 也能够保证可见性，它能保证同一时刻只有一个线程获取锁然后执行同步代码，并且在释放锁之前会将对变量的修改刷新到主存当中

    • 死锁问题：多个线程之间相互等待对方释放锁，导致程序无法继续执行。 Java中实现线程同步的方式有两种：synchronized关键字和Lock接口，它们都能够实现线程之间的同步操作。

      • 锁的概念

        锁是一种同步机制，用于协调对共享资源的访问。它可以防止多个线程同时访问共享资源，从而避免数据竞争和不一致的结果。

      • 产生锁的原因

        锁的产生原因是多个线程在共享同一块资源时，可能会在同一时间修改或读取该资源，导致结果不一致或异常。为了避免这种情况，需要使用锁来确保在任何时候只有一个线程可以访问共享资源。

      • 锁的分类

        重入锁（ReentrantLock）：是一种可重入的互斥锁，它允许线程多次获得同一把锁而不会发生死锁。

        Java 中的可重入锁是指同一个线程可以多次获取同一把锁而不会发生死锁的一种锁机制。在使用可重入锁时，线程首先需要获得锁，然后可以在不释放锁的情况下进入被保护的代码块。当线程再次请求该锁时，由于已经持有该锁，因此不需要再次等待，可以直接获得锁并进入被保护的代码块。 可重入锁的主要优点是避免了死锁的发生。在使用普通锁时，如果线程 A 持有了某个锁并在执行代码块时等待了另外一个锁，而这个锁又被线程 B 持有并等待释放，那么就会发生死锁。而使用可重入锁时，线程 A 可以直接再次获取已经持有的锁，从而避免了死锁的发生。 另外，可重入锁还可以提高代码的灵活性和可重用性。在某些情况下，我们需要在一个方法中调用另一个方法，而这两个方法都需要获取同一把锁来保证线程安全。如果使用普通锁，就需要在被调用的方法中再次获取锁，这样会降低代码的灵活性和可重用性。而使用可重入锁时，线程可以在被调用的方法中直接再次获取已经持有的锁，从而避免了重复获取锁的问题，提高了代码的灵活性和可重用性。 可重入锁在 Java 中的实现主要有两种，一种是 synchronized 关键字，另一种是 ReentrantLock 类。在使用 synchronized 关键字时，如果一个线程已经持有了锁，那么它可以继续调用同步方法或同步代码块而不会阻塞。而在使用 ReentrantLock 类时，线程需要使用 lock() 方法获取锁，并使用 unlock() 方法释放锁。如果一个线程已经获得了锁，那么它可以多次调用 lock() 方法，但必须相应地多次调用 unlock() 方法才能完全释放锁

        读写锁（ReadWriteLock）：允许多个线程同时读取共享数据，但只允许一个线程写入共享数据。读写锁适用于读多写少的场景。

        Java 中的读写锁（ReadWriteLock）是一种特殊的锁机制，它允许多个线程同时读取共享数据，但只允许一个线程写入共享数据。读写锁适用于读多写少的场景，可以提高程序的并发性能。 读写锁包含两个锁：读锁和写锁。当有线程持有读锁时，其他线程也可以同时持有读锁而不会发生阻塞。但当有线程持有写锁时，其他线程无法同时持有读锁或写锁，只能等待写锁释放。读写锁的主要目的是提高读取效率，因为在读取数据时不需要加锁，多个线程可以同时读取同一块数据。 读写锁的使用方式与普通锁类似，可以使用 synchronized 关键字或者 ReentrantLock 类来实现。在使用 synchronized 关键字时，可以将读取操作和写入操作分别放在不同的 synchronized 块中，也可以使用 synchronized 方法。在使用 ReentrantLock 类时，可以使用 readLock() 方法获取读锁，使用 writeLock() 方法获取写锁。 需要注意的是，在使用读写锁时，如果有线程持有写锁，那么其他线程无法同时持有读锁或写锁。这就可能导致写锁的饥饿问题，即因为写线程一直持有锁而导致读线程一直无法获取锁而被阻塞。为了避免这种情况，可以使用 ReentrantReadWriteLock 类中的 tryReadLock() 和 tryWriteLock() 方法来尝试获取锁，如果无法获取，则可以等待一段时间后重试，或者放弃获取锁。

        synchronized 关键字：是 Java 中最基本的锁机制，它可以用于同步方法和同步代码块。synchronized 关键字采用的是悲观锁策略，即假定每次操作都会出现并发问题。

      • 出现锁的解决方案

        1. 死锁检测：当多个线程相互等待对方释放锁时，会出现死锁。死锁检测可以帮助我们检测到死锁并及时解决。
        2. 可重入锁：可以允许一个线程多次获得同一把锁，从而避免死锁。
        3. 读写锁：可以允许多个线程同时读取共享数据，从而提高读取效率。只有在写入时才需要加锁，从而减少了加锁的次数。
        4. CAS（Compare And Swap）：是一种乐观锁策略，它通过先比较内存中的值是否与自己期望的值相同，如果相同则进行更新操作，否则重新读取内存中的值。CAS 操作是原子性的，可以避免多个线程同时修改同一块内存导致的数据竞争问题。

    # 12.4.1 线程通信的概念

    Java线程通信是指在多个线程之间传递数据或信号，使它们可以相互协作完成任务。在多线程编程中，多个线程之间需要进行协作来完成任务，这就需要线程之间进行通信。

    # 12.4.2 线程通信的目的

    线程通信的目的是为了协调多个线程之间的执行顺序和访问共享资源的方式，从而实现线程之间的相互协作。

      /**
 * @author Adopat
 * @description: 生产者消费者模型 线程通信的经典案例
 * @date 2023/03/09 14:54

class ProducerConsumer {
    // 用于线程同步的锁
    private final Object lock = new Object();
    // 共享资源
    private int count = 0;

    /**
     * 生产者方法
     * @throws InterruptedException
     */
    public void produce() throws InterruptedException {
        synchronized (lock) {
            // 如果共享资源已经满了，就等待消费者消费
            while (count == 1) {
                lock.wait();
            }
            // 生产一个产品
            count++;
            System.out.println("生产者生产了一个产品，当前产品数量：" + count);
            // 唤醒等待的消费者线程
            lock.notifyAll();
        }
    }

    /**
     * 消费者方法
     * @throws InterruptedException
     */
    public void consume() throws InterruptedException {
        // 如果共享资源已经为空，就等待生产者生产
        synchronized (lock) {
            while (count == 0) {
                lock.wait();
            }
            // 消费一个产品
            count--;
            System.out.println("消费者消费了一个产品，当前产品数量：" + count);
            // 唤醒等待的生产者线程
            lock.notifyAll();
        }
    }

    • Lock接口&Condition接口

      • Lock接口概念

        Java 中的 Lock 接口和 Condition 接口是 JDK 5 中提供的一种替代 synchronized 关键字的方式，用于实现多线程的同步和互斥。Lock 接口提供了比 synchronized 更加灵活、精细的锁机制，Condition 接口则是 Lock 接口的补充，用于实现更精细的线程等待和唤醒操作。

      • Lock接口优点

        支持更多的锁操作，如获取锁的超时、可中断的锁等等；支持公平锁和非公平锁，可以控制线程的等待顺序；可以配合 AQS（AbstractQueuedSynchronizer）实现自定义锁机制

      • Condition接口概念

        Condition 接口主要用于实现更精细的线程等待和唤醒操作，可以用于解决线程等待的粒度问题，避免了使用 wait() 和 notify() 方法时可能出现的死锁和竞态条件等问题。

      • Condition接口常用方法

        await()：使当前线程等待，直到被通知或者被中断；

        awaitUninterruptibly()：使当前线程等待，直到被通知；

        signal()：唤醒一个等待的线程；

        signalAll()：唤醒所有等待的线程。

      • 实例代码

        package java_concurrent;

/**
 * @author Adopat
 * @description: 使用Lock接口 Condition 接口 实现线程通信
 * @date 2023/03/09 16:11
 **/
import java.util.concurrent.locks.Condition;
import java.util.concurrent.locks.Lock;
import java.util.concurrent.locks.ReentrantLock;
public class LockExample {
    private Lock lock = new ReentrantLock();
    private Condition condition = lock.newCondition();
    private int count = 0;
    public void increment() throws InterruptedException {
        lock.lock();
        try {
            while (count != 0) {
                // count数量不为0 生产者等待
                condition.await();
            }
            count++;
            System.out.println(Thread.currentThread().getName() + " 生产了一个商品，现在商品数量为：" + count);
            // 唤醒消费者消费
            condition.signalAll();
        } finally {
            lock.unlock();
        }
    }
    public void decrement() throws InterruptedException {
        lock.lock();
        try {
            while (count == 0) {
                // count数量不为0 消费者等待
                condition.await();
            }
            count--;
            System.out.println(Thread.currentThread().getName() + " 消费了一个商品，现在商品数量为：" + count);
            // 唤醒生产者生产
            condition.signalAll();
        } finally {
            lock.unlock();
        }
    }
    public static void main(String[] args) {
        LockExample example = new LockExample();
        // 创建生产者线程
        Thread producerThread = new Thread(() -> {
            try {
                while (true) {
                    example.increment();
                    Thread.sleep(1000);
                }
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        });
        producerThread.setName("ProducerThread");
        producerThread.start();
        // 创建消费者线程
        Thread consumerThread = new Thread(() -> {
            try {
                while (true) {
                    example.decrement();
                    Thread.sleep(1000);
                }
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        });
        consumerThread.setName("ConsumerThread");
        consumerThread.start();
    }
}

    • BlockingQueue

      • 概念

        Java 中的 BlockingQueue是一个可以用于实现线程间通信的数据结构，它实现了生产者-消费者模型，即多个线程可以同时访问队列，当队列为空时消费者线程会被阻塞，当队列满时生产者线程会被阻塞，从而实现了线程间的同步和互斥。

      • 常用方法

        put(E e)：将元素插入到队列的末尾，如果队列已满，则阻塞等待；

        take()：从队列的头部获取元素，如果队列为空，则阻塞等待；

        offer(E e, long timeout, TimeUnit unit)：将元素插入到队列的末尾，如果队列已满，则阻塞等待一段时间；

        poll(long timeout, TimeUnit unit)：从队列的头部获取元素，如果队列为空，则阻塞等待一段时间。

      • 实例代码

        /**
 * @author Adopat
 * @description: 使用BlockingQueue实现线程通信
 * @date 2023/03/09 16:43
 **/
import java.util.concurrent.BlockingQueue;
import java.util.concurrent.LinkedBlockingQueue;
public class BlockingQueueExample {
    // 设置队列的容量
    private static final int CAPACITY = 10;
    private static BlockingQueue<Integer> queue = new LinkedBlockingQueue<>(CAPACITY);
    public static void main(String[] args) {
        // 创建生产者线程
        Thread producerThread = new Thread(() -> {
            try {
                for (int i = 1; i <= 20; i++) {
                    queue.put(i);
                    System.out.println(Thread.currentThread().getName() + " 生产了一个商品，现在商品数量为：" + queue.size());
                    Thread.sleep(1000);
                }
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        });
        producerThread.setName("ProducerThread");
        producerThread.start();
        // 创建消费者线程
        Thread consumerThread = new Thread(() -> {
            try {
                while (true) {
                    int value = queue.take();
                    System.out.println(Thread.currentThread().getName() + " 消费了一个商品，现在商品数量为：" + queue.size());
                    Thread.sleep(1000);
                }
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        });
        consumerThread.setName("ConsumerThread");
        consumerThread.start();
    }
}

    # 12.4.4 notify 和 notifyAll区别

    • 使用场景区别

      notify() 方法通常用于生产者-消费者模型中的生产者唤醒消费者，或者是多个线程竞争同一个资源时的唤醒操作。因为在这些场景中，多个线程之间是互斥的，只需要唤醒其中一个线程即可，其他线程还需要继续等待。 notifyAll() 方法通常用于多个线程之间的协作和通信，例如，在分布式系统中，多个节点之间需要等待某个条件满足后才能继续执行，此时就需要使用 notifyAll()方法唤醒所有等待的节点。

      需要注意的是，notify() 和 notifyAll() 方法都必须在同步代码块或同步方法中调用，并且必须持有对象锁。在调用 notify() 或 notifyAll() 方法后，被唤醒的线程不会立即从 wait() 方法中返回，而是要等待当前线程释放锁后才能继续执行。

    • 被唤醒的线程数量不同

      notify() 方法只会随机唤醒一个等待中的线程，而 notifyAll()方法会唤醒所有等待中的线程。

    • 实例代码

      package java_concurrent;

/**
 * @author Adopat
 * @description: notify 和 notifyAll区别
 * @date 2023/03/09 15:57
 **/
 class WaitNotifyExample {
    // 创建对象锁
    private final Object lock = new Object();
    private boolean flag = false;
    // wait() 方法的使用
    public void waitMethod() throws InterruptedException {
        synchronized (lock) {
            while (!flag) {
                lock.wait();
                System.out.println(Thread.currentThread().getName() + " is waiting...");
            }
            System.out.println(Thread.currentThread().getName() + " 被唤醒了！");
        }
    }
    // notify() 方法的使用,需要在同步代码块中执行
    public void notifyMethod() {
        synchronized (lock) {
            flag = true;
            lock.notify();
        }
    }
    // notifyAll() 方法的使用
    public void notifyAllMethod() {
        synchronized (lock) {
            flag = true;
            lock.notifyAll();
        }
    }
}
public class NotifyTest {
    public static void main(String[] args) throws InterruptedException {
        WaitNotifyExample example = new WaitNotifyExample();
        // 创建等待线程
        Thread waitThread = new Thread(() -> {
            try {
                example.waitMethod();
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        });
        waitThread.setName("WaitThread");
        waitThread.start();
        // 等待 1 秒钟
        Thread.sleep(1000);
        // 创建唤醒线程
        Thread notifyThread = new Thread(() -> example.notifyMethod());
        notifyThread.start();
        // 等待 1 秒钟
        Thread.sleep(1000);
        // 创建唤醒线程
        Thread notifyAllThread = new Thread(() -> example.notifyAllMethod());
        notifyAllThread.start();
    }
}

      1. 确保使用 synchronized 关键字来保护共享资源，避免死锁和竞态条件的出现。

    # 12.5. Java 线程池

    # 12.5.1 线程池的概念

    Java线程池是一种线程管理技术，它可以预先创建一定数量的线程并将它们保存在池中，当需要执行任务时，就可以从线程池中获取空闲的线程来执行任务，执行完毕后将线程归还给线程池，从而避免了线程的频繁创建和销毁，提高了线程的利用率和系统的性能。

    # 12.5.2 线程池的目的

    线程池的主要目的是优化线程的使用，避免线程的频繁创建和销毁，降低系统开销，提高系统的吞吐量和响应性。

    # 12.5.3 线程池的创建

    Java中提供了多种方式来创建线程池，如通过ThreadPoolExecutor类手动创建线程池，或者通过Executors类中提供的静态方法来创建线程池。

    • 手动创建线程池(推荐使用)

      package java_concurrent;

import com.google.common.util.concurrent.ThreadFactoryBuilder;

import java.util.concurrent.*;

/**
 * @author Adopat
 * @description: Java 线程池的创建 手动创建 使用 guva
 * @date 2023/03/09 13:09
 **/

public class MyThreadPool {
    public static void main(String[] args) {
        // 设置线程名字
        ThreadFactory namedThreadFactory = new ThreadFactoryBuilder()
                .setNameFormat("demo-pool-%d").build();
        ExecutorService singleThreadPool = new ThreadPoolExecutor(
                // 核心线程数
                10,
                // 最大线程数
                20,
                //线程空闲时间
                0L,
                // 时间单位
                TimeUnit.MILLISECONDS,
                // 任务队列
                new LinkedBlockingQueue<Runnable>(1024),
                // 创建线程使用的工厂
                namedThreadFactory,
                // 由于达到线程边界和队列容量而阻塞执行时使用的处理程序
                new ThreadPoolExecutor.AbortPolicy()
        );


        for (int i = 0; i < 20; i++) {
            singleThreadPool.execute(() -> System.out.println(Thread.currentThread().getName()));
        }
        // 线程关闭
        singleThreadPool.shutdown();


    }

}
