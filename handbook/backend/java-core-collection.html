    # 第十章 集合框架容器

    # 10.1 Collection 集合体系图

    • Set

      • TreeSet：基于红黑树实现，支持有序性操作，例如根据一个范围查找元素的操作。 但是查找效率不如 HashSet，HashSet 查找的时间复杂度为 O(1)，TreeSet则为 O(logN)。
      • HashSet：基于哈希表实现，支持快速查找，但不支持有序性操作。 并且失去了元素的插入顺序信息，也就是说使用 Iterator 遍历HashSet 得到的结果是不确定的。
      • LinkedHashSet：具有 HashSet的查找效率，且内部使用双向链表维护元素的插入顺序。

    • List

      • ArrayList：基于动态数组实现，支持随机访问。
      • Vector：和 ArrayList类似，但它是线程安全的。
      • LinkedList：基于双向链表实现，只能顺序访问，但是可以快速地在链表中间插入和删除元素。 不仅如此，LinkedList 还可以用作栈、队列和双向队列。

    • Queue

      • LinkedList：可以用它来实现双向队列。
      • PriorityQueue：基于堆结构实现，可以用它来实现优先队列。

    # 10.2 Map 集合体系图

    • TreeMap：基于红黑树实现。
    • HashMap：基于哈希表实现。
    • HashTable：和 HashMap 类似，但它是线程安全的， 这意味着同一时刻多个线程可以同时写入 HashTable 并且不会导致数据不一致。 它是遗留类，不应该去使用它。现在可以使用 ConcurrentHashMap 来支持线程安全， 并且 ConcurrentHashMap 的效率会更高，因为 ConcurrentHashMap引入了分段锁。
    • LinkedHashMap：使用双向链表来维护元素的顺序，顺序为插入顺序或者最近最少使用（LRU）顺序。

    # 10.3 容器常见实现类

    # 10.3.1ArrayList&LinkedList

    • 概念

      Java ArrayList是Java集合框架中的一个实现类，它可以动态地增加或者缩减容量以适应需要，可以存储任意类型的对象，且元素可以重复。

    • ArrayList使用场景

      1. 当需要一个可以自动扩容且长度可变的序列时。
      2. 当需要进行随机访问时（根据索引查找元素）。

    • ArrayList和LinkedList的区别和联系

      区别：

      1. 内部实现结构不同：ArrayList是通过数组实现的，LinkedList是通过链表实现的。
      2. 随机访问效率不同：ArrayList支持快速的随机访问，因为它的元素在内存中是连续存储的，而LinkedList需要遍历元素才能找到指定位置的元素。
      3. 插入和删除操作效率不同：LinkedList在插入和删除元素时效率更高，因为它只需要改变链表中的指针指向即可，而ArrayList需要移动数组中的元素，效率较低。
      4. 线程安全性不同：ArrayList不是线程安全的，而LinkedList可以通过Collections.synchronizedList()方法或者使用ConcurrentLinkedDeque类来实现线程安全。

      联系：

      都实现了List接口，具有相同的基本方法。都支持元素的添加、删除、遍历等操作。都可以存储任意类型的对象。

    • 常用方法

      • add(E e)：向列表的尾部添加指定元素。

      • add(int index, E element)：将指定的元素插入此列表中的指定位置。

      • remove(int index)：从此列表中移除指定位置上的元素。

      • remove(Object o)：从列表中移除指定元素的第一个匹配项（如果存在）。

      • get(int index)：返回列表中指定位置的元素。

      • set(int index, E element)：用指定元素替换列表中指定位置的元素。

      • size()：返回列表中的元素数。

      • clear()：从列表中删除所有元素。

      • indexOf(Object o)：返回此列表中第一次出现的指定元素的索引，如果此列表不包含该元素，则返回-1

    • 实例代码

      import java.util.ArrayList;
import java.util.LinkedList;

public class ListTest {
    public static void main(String[] args){
        // ArrayList只能存储引用数据类型
        ArrayList<String> arrayList = new ArrayList<>();
        LinkedList<String> linkedList = new LinkedList<>();
        // 添加元素
        arrayList.add("tom");
        arrayList.add("pony");
        arrayList.add("jack");
        // 打印ArrayList
        System.out.println(arrayList);
        // 更新元素
        arrayList.set(0,"justin");
        System.out.println(arrayList);
        // 返回大小
        int size = arrayList.size();
        System.out.println("size ="+size);
        // 通过索引删除元素
        String removeString = arrayList.remove(0);
        System.out.println(removeString);
        // 通过对象删除元素
        boolean removeObj = arrayList.remove("justin");
        System.out.println(removeObj);
        // 判断arrayList 是否为空
        boolean emptyFlag = arrayList.isEmpty();
        System.out.println(emptyFlag);
        // 清空arrayList
        arrayList.clear();
        System.out.println(arrayList.isEmpty());
        // LinkedList 是基于链表实现的，增删比较快 特有API
        linkedList.add("justin");
        linkedList.add("pony");
        linkedList.add("jack");
        // 获取第一个元素
        String firstString = linkedList.getFirst();
        // 获取最后一个元素
        String lastString = linkedList.getLast();
        // justin
        System.out.println(firstString);
        // jack
        System.out.println(lastString);
    }
}

    # 10.3.2HashSet&TreeSet

    • 概念

      Java HashSet是Java集合框架中的一个实现类，它实现了Set接口，可以存储任意类型的对象，但是不能包含重复的元素，可以为null。

    • HashSet使用场景

      1. 当需要存储一组唯一的元素时。
      2. 当不需要按照特定顺序对元素进行排序时。
      3. 当需要进行快速查找时

    • 区别和联系

      区别：

      1. 内部实现结构不同：HashSet是通过哈希表实现的，而TreeSet是通过红黑树实现的。
      2. 排序方式不同：HashSet不保证元素的顺序，而TreeSet可以按照元素的大小升序排序。
      3. 迭代性能不同：HashSet的迭代性能比TreeSet要好，因为HashSet是通过哈希表实现的，访问元素的时间复杂度是O(1)，而TreeSet是通过红黑树实现的，访问元素的时间复杂度是O(log n)。
      4. 插入和删除操作效率不同：HashSet在插入和删除元素时效率更高，因为它只需要计算哈希值即可，而TreeSet需要维护红黑树的平衡性，效率较低。

      联系：

      1. 都实现了Set接口，具有相同的基本方法。
      2. 都支持元素的添加、删除、遍历等操作。
      3. 都可以存储任意类型的对象。

      如果需要快速的插入和删除元素，可以使用HashSet；如果需要按照元素的大小排序，可以使用TreeSet。

    • 常用方法

      • add(E e)：向set中添加一个元素。

      • remove(Object o)：从set中删除一个元素。

      • size()：返回set中元素的数量。

      • isEmpty()：检查set是否为空。

      • contains(Object o)：检查set中是否包含指定的元素。

      • clear()：从set中删除所有元素。

      • iterator()：返回一个用于遍历set中元素的迭代器。

    • 实例代码

      import java.util.HashSet;
import java.util.Set;

public class SetTest {
    public static void main(String[] args){
        Set<String> set = new HashSet<>();
        set.add("小明");
        set.add("小东");
        set.add("小红");
        set.add("小明");
        // 打印set 集合
        System.out.println("set ="+set);
        // 返回set 大小
        int size = set.size();
        System.out.println("size ="+size);
        // 根据对象删除元素
        set.remove("小明");
        // set 是无序，不可重复的，set没有索引
        boolean isEmpty = set.isEmpty();
        System.out.println("isEmpty ="+isEmpty);
        // HashSet 不能保证元素的排列顺序，TreeSet是SortedSet接口的唯一实现类，可以确保集合处于排序状态
        // HashSet 底层使用的哈希表，TreeSet采用的数据结构是红黑树
        // HashSet中元素可以是null,但是只能有一个，TreeSet不允许放入null
        // 一般使用HashSet,如果需要排序功能是，才使用TreeSet

    }
}

class Teacher{
    private String name;
    private int age;

    public Teacher() {
    }

    public Teacher(String name, int age) {
        this.name = name;
        this.age = age;
    }

    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }

    public int getAge() {
        return age;
    }

    public void setAge(int age) {
        this.age = age;
    }
    /*
      equals重写规则
      1.判断地址是否一样
      2.非空判断和class 类型判断
      3.强转
      4.对象里面的字段一一匹配
     */
    @Override
    public boolean equals(Object o) {
        if (this == o) return true;
        if (o == null || getClass() != o.getClass()) return false;
        Teacher teacher = (Teacher) o;
        return age == teacher.age && Objects.equals(name, teacher.name);
    }

    @Override
    public int hashCode() {
        return Objects.hash(name, age);
    }

    @Override
    public String toString() {
        return "Teacher{" +
                "name='" + name + '\'' +
                ", age=" + age +
                '}';
    }
}

    # 10.3.3HashMap&TreeMap

    • 概念

      Java HashMap是Java集合框架中的一个实现类，它实现了Map接口，可以存储键值对，其中键和值都可以为null，但是键不能重复，如果重复则会覆盖原有的键值对。

    • HashMap使用场景

      1. 当需要存储键值对，并且需要根据键快速查找对应的值时。
      2. 当需要进行快速插入和删除键值对时

    • 区别和联系

      区别：

      1. 内部实现结构不同：HashMap是通过哈希表实现的，而TreeMap是通过红黑树实现的。
      2. 排序方式不同：HashMap不保证元素的顺序，而TreeMap可以按照键的大小升序排序。
      3. 插入和删除操作效率不同：HashMap在插入和删除元素时效率更高，因为它只需要计算哈希值即可，而TreeMap需要维护红黑树的平衡性，效率较低。
      4. 支持的元素类型不同：HashMap可以存储null键和null值，而TreeMap不允许插入null键，也不允许插入null值。

      联系：

      1. 都实现了Map接口，具有相同的基本方法。
      2. 都支持元素的添加、删除、遍历等操作。
      3. 都可以存储键值对。
      4. 都支持键值对的查找操作。

      如果需要快速的插入和删除键值对，可以使用HashMap；如果需要按照键的大小排序，可以使用TreeMap。

    • 常用方法

      • put(K key, V value)：向map中添加一个键值对。

      • remove(Object key)：从map中删除指定键的键值对。

      • get(Object key)：根据键获取对应的值。

      • containsKey(Object key)：检查map中是否包含指定的键。

      • clear()：从map中删除所有键值对。

      • keySet()：返回map中所有键的集合。

      • values()：返回map中所有值的集合。

      • entrySet()：返回map中所有键值对的集合。

    • 实例代码

      import java.util.*;

public class MapTest {
    public static void main(String[] args){
        // hashMap 无序
        hashMap();
        // treeMap 有序
        System.out.println("+++++++++++++++++++++++++++");
        treeMap();
        System.out.println("++++++++++++++++++++++++++++++");
        // linkedHashMap() 是有序的
        linkedHashMap();
    }
    public static void hashMap() {
        HashMap<String, String> hashMap = new HashMap<>();
        // 增加entry 元素
        hashMap.put("A小明", "广东深圳");
        hashMap.put("B小红", "广东东莞");
        hashMap.put("C小绿", "广东惠州");
        hashMap.put("D小蓝", "广东广州");
        // 打印hashMap
        //hashMap = {B小红=广东东莞, A小明=广东深圳, D小蓝=广东广州, C小绿=广东惠州}
        System.out.println("hashMap = " + hashMap);
        // 获取 value 集合
        Collection<String> collection = hashMap.values();
        System.out.println(collection);
        // 根据key 获取value
        String value = hashMap.get("A小明");
        System.out.println("value =" + value);
        // 返回hashMap 中元素数量
        int size = hashMap.size();
        System.out.println("size =" + size);
        // 判断是否含有某个key
        boolean isContainsKey = hashMap.containsKey("A小明");
        System.out.println("isContainsKeys =" + isContainsKey);
        // 获取所有的key集合
        Set<String> keySet = hashMap.keySet();
        // keySet =[B小红, A小明, D小蓝, C小绿] 
        System.out.println("keySet =" + keySet);
        // 返回一个set集合，集合的类型为Map.Entry
        Set<Map.Entry<String, String>> entrySet = hashMap.entrySet();
        // entrySet =[B小红=广东东莞, A小明=广东深圳, D小蓝=广东广州, C小绿=广东惠州] 
        System.out.println("entrySet =" + entrySet);
        for (Map.Entry entry : entrySet) {
            System.out.println("key=" + entry.getKey() + ",value" + entry.getValue());
        }
        // 判断是否为空
        boolean beforeisEmpty = hashMap.isEmpty();
        System.out.println("beforeisEmpty="+beforeisEmpty);
        // 清空Map
        hashMap.clear();
        // 清除之后是否为空
        boolean afterisEmpty = hashMap.isEmpty();
        System.out.println("afterisEmpty="+afterisEmpty);
    }
        public static void treeMap(){

            treeMap.put("A小明","广东深圳");

            treeMap.put("C小绿","广东惠州");
            treeMap.put("D小蓝","广东广州");
            // 打印treeMap
            //treeMap = {A小明=广东深圳, B小红=广东东莞, C小绿=广东惠州, D小蓝=广东广州}
            System.out.println("treeMap = "+treeMap);
            // 获取 value 集合
            Collection<String> collection = treeMap.values();
            System.out.println(collection);
            // 根据key 获取value
            String value = treeMap.get("A小明");
            System.out.println("value ="+value);
            // 返回treeMap 中元素数量
            int size = treeMap.size();
            System.out.println("size ="+size);
            // 判断是否含有某个key
            boolean isContainsKey = treeMap.containsKey("A小明");
            System.out.println("isContainsKeys ="+isContainsKey);
            // 获取所有的key集合
            Set<String> keySet = treeMap.keySet();
            System.out.println("keySet ="+keySet);
            // 返回一个set集合，集合的类型为Map.Entry
            Set<Map.Entry<String,String>> entrySet = treeMap.entrySet();
            for(Map.Entry entry:entrySet){
                System.out.println("key="+entry.getKey()+",value"+entry.getValue());
            }
            // 判断是否为空
            boolean beforeisEmpty = treeMap.isEmpty();
            System.out.println("beforeisEmpty="+beforeisEmpty);
            // 清空Map
            treeMap.clear();
            // 清除之后是否为空
            boolean afterisEmpty = treeMap.isEmpty();
            System.out.println("afterisEmpty="+afterisEmpty);

        }
    public static void linkedHashMap(){
        LinkedHashMap<String,String> linkedHashMap = new LinkedHashMap<>();
        // 增加entry 元素
        linkedHashMap.put("A小明","广东深圳");
        linkedHashMap.put("B小红","广东东莞");
        linkedHashMap.put("C小绿","广东惠州");
        linkedHashMap.put("D小蓝","广东广州");
        // 打印LinkedHashMap
        //linkedHashMap = {A小明=广东深圳, B小红=广东东莞, C小绿=广东惠州, D小蓝=广东广州}
        System.out.println("linkedHashMap = "+linkedHashMap);

    # 10.4 迭代器

    • 概念

      迭代器是一种遍历容器元素的方式，它可以在容器中按顺序遍历元素，而不需要知道容器内部的结构。

    • 分类

      • Iterator：是最基本的迭代器，可以遍历所有实现了Iterable接口的容器类，提供了hasNext()和next()方法进行遍历。
      • ListIterator：是专门用于遍历List集合的迭代器，提供了hasNext()、next()、previous()、hasPrevious()等方法，可以实现双向遍历。

    • 使用场景

      当需要遍历集合中的元素时，可以使用迭代器进行遍历。迭代器可以遍历所有实现了Iterable接口的容器类，例如ArrayList、HashSet、TreeSet等等。

    • 和普通for循环分类

      1. 迭代器可以遍历任何实现了Iterable接口的容器类，而普通for循环只能遍历数组或实现了List接口的容器类。
      2. 迭代器遍历过程中可以进行删除操作，而普通for循环不支持删除操作。
      3. 迭代器可以实现快速的遍历和查找，而普通for循环需要遍历整个集合。

    # 10.5 比较器

    • 概念

      Java容器中的比较器（Comparator）是一种用于比较两个对象大小的接口，它可以在容器中进行排序操作

    • 使用场景

      当需要对容器中的元素进行排序时，可以使用比较器。比较器可以用于实现自然排序（Natural Ordering）和自定义排序（Custom Ordering）.

    • 分类

      1. 自然排序（Natural Ordering）：Java中的许多类都实现了Comparable接口，该接口中定义了compareTo()方法，可以实现对象的自然排序。例如，字符串、数字等类型的排序都是自然排序。
      2. 自定义排序（Custom Ordering）：当需要对容器中的元素进行自定义排序时，可以使用Comparator接口。该接口中定义了compare()方法，可以实现自定义比较器。例如，对于一个自定义的Person类，可以通过实现Comparator接口来定义一个比较器，按照姓名或者年龄进行排序。

    • Comparable接口和Comparator接口的区别和联系

      区别：

      1. Comparable接口是在需要进行自然排序时实现的，而Comparator接口是在需要进行自定义排序时实现的。
      2. Comparable接口中的compareTo()方法只能实现一种排序方式，而Comparator接口中的compare()方法可以实现多种排序方式。
      3. Comparable接口中的compareTo()方法需要在对象本身中实现，而Comparator接口中的compare()方法可以单独实现。

      联系：

      1. Comparable接口和Comparator接口都是用于比较对象大小的接口。
      2. Comparable接口和Comparator接口都可以用于实现排序

      return a-b 升序解释

      compare原理分析（jdk默认规则,默认升序）：假如比较的数为a,b

      1.如果return <0 此时a<b, a和b的位置不变, 此时顺序为 a,b (a<b 升序)

      2.如果return >0 此时a>b，需要交换a和b位置，此时顺序为 b a (b<a 升序)

      3.如果return ==0 此时a等于b，a和b位置不变。 所以 return a-b 为升序

      return b-a 降序解释

      如果 b-a>0 说明 b>a 此时 顺序不变 b a(b>a 降序)

      如果 b-a<0 说明 b<a 此时交换顺序 a,b (a>b 降序)

      所以 return b-a 为降序

      可以理解为 a,b 两数比较，默认排序规则为升序，当 a<b 是顺序不变,a>b 是交换顺序

    # 10.6 排序

    • 常见的排序方法

      1. Collections.sort()：使用自然排序（Natural Ordering）对List进行排序。
      2. Collections.sort(List list, Comparator c)：使用定制排序（Custom Ordering）对List进行排序。
      3. Arrays.sort()：使用自然排序（Natural Ordering）对数组进行排序。
      4. Arrays.sort(T[] a, Comparator c)：使用定制排序（Custom Ordering）对数组进行排序。
      5. Arrays.parallelSort()：使用并行排序（Parallel Sorting）对数组进行排序。
      6. Arrays.parallelSort(T[] a, Comparator c)：使用并行排序（Parallel Sorting）和定制排序（Custom Ordering）对数组进行排序。
      7. External Sort API：使用外部排序（External Sorting）对大量数据进行排序。 这些排序方法可以对Java容器中的对象进行排序，而且可以按照自然排序或定制排序的方式排序，还可以使用并行排序提高排序效率。

    # 10.8 并发集合

    • 概念

      并发集合是指可以被多个线程同时访问的集合，具有线程安全的特性，能够在多线程并发的情况下保证数据的一致性。Java提供了多种并发集合类，如ConcurrentHashMap、ConcurrentLinkedQueue等。

    • 出现原因

      在多线程环境下，如果多个线程同时访问同一个集合，可能会发生数据不一致的情况，比如一个线程在修改集合元素时，另一个线程在遍历集合，这时就需要使用线程安全的并发集合来解决这个问题。

    • 并发集合分类

      1. ConcurrentMap：线程安全的Map集合，它的实现类有ConcurrentHashMap和ConcurrentSkipListMap。
      2. ConcurrentNavigableMap：线程安全的可导航Map集合，它的实现类有ConcurrentSkipListMap。
      3. ConcurrentLinkedQueue：线程安全的队列，它的实现类有ConcurrentLinkedQueue和ConcurrentLinkedDeque。
      4. BlockingQueue：阻塞队列，它的实现类有ArrayBlockingQueue、LinkedBlockingQueue、PriorityBlockingQueue、DelayQueue和SynchronousQueue。
      5. TransferQueue：传输队列，它的实现类有LinkedTransferQueue。
      6. BlockingDeque：阻塞双端队列，它的实现类有LinkedBlockingDeque。

    • 实例代码

      import java.util.ArrayList;
import java.util.concurrent.CopyOnWriteArrayList;

/**
 * @author Adopat
 * @description: 并发集合
 * @date 2023/03/13 14:34
 **/
public class ConcurrentCollectionExample {
    public static void main(String[] args) throws InterruptedException {
        nonConcurrentCollection();
        concurrentCollection();


    }

    /**
     * 不使用并发集合
     */
    public static void nonConcurrentCollection() throws InterruptedException {
        ArrayList<Object> list = new ArrayList<>();
        for (int i = 0; i < 10000; i++) {
            Thread thread = new Thread(new Runnable() {
                @Override
                public void run() {
                    list.add("element");
                }
            });
            thread.start();

        }
        Thread.sleep(1000);
        System.out.println("不使用并发集合: "+list.size());
    }
    /**
     * 使用并发集合
     */
    public static void concurrentCollection() throws InterruptedException {
        CopyOnWriteArrayList<String> list = new CopyOnWriteArrayList<>();
        for (int i = 0; i < 10000; i++) {
            Thread thread = new Thread(new Runnable() {
                @Override
                public void run() {
                    list.add("element");
                }
            });
            thread.start();
        }
        Thread.sleep(1000);
        System.out.println("使用并发集合 ："+list.size());
    }
    
}

      为什么要使用并发集合？ 在不使用并发集合的情况下，多个线程同时对List进行操作，可能会导致数据不一致的情况，比如多个线程同时调用add方法，可能会覆盖掉其他线程添加的元素，或者可能会出现数组越界的异常等。而使用并发集合可以保证多线程并发的情况下数据的一致性和线程安全性。并发集合采用了一些特殊的算法和机制，如CAS、分段锁等来保证线程安全性和高性能。因此，在多线程情况下，为了保证程序的正确性和性能，我们应该使用并发集合。
