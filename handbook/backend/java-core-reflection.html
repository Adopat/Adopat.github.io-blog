    # 第五章 反射

    # 5.1 反射概念

    反射是指在程序运行时，动态地获取一个类的信息，包括类名、属性、方法等，然后通过这些信息来操作该类。Java中的反射机制是通过反射API提供的类来实现的，主要包括Class、Constructor、Field和Method等。

    Class类是Java反射API中最基本的类，它代表一个类的定义，可以获取类的信息，包括它的名称、属性、方法等。Constructor类代表一个类的构造方法，可以通过它来实例化一个对象。Field类代表一个类的属性，可以获取和设置属性的值。Method类代表一个类的方法，可以调用方法并获取方法的返回值。

    下面是一个简单的反射示例，演示了如何通过反射获取一个类的信息：

    public class MyClass {
    private String name;
    public void setName(String name) {
        this.name = name;
    }
    public String getName() {
        return name;
    }
}

public class ReflectionDemo {
    public static void main(String[] args) {
        Class clazz = MyClass.class;
        System.out.println("类名：" + clazz.getName());
        Field[] fields = clazz.getDeclaredFields();
        System.out.println("属性列表：");
        for (Field field : fields) {
            System.out.println(field.getName());
        }
        Method[] methods = clazz.getDeclaredMethods();
        System.out.println("方法列表：");
        for (Method method : methods) {
            System.out.println(method.getName());
        }
    }
}

    # 5.2 反射的作用

    反射可以帮助程序在运行时动态地获取和操作类的信息，从而实现一些动态化的功能，例如：

    1. 实现框架自动化配置、自动化装配等功能。
    2. 动态生成代理对象，实现动态代理。
    3. 实现插件化开发，动态加载和卸载插件。
    4. 实现单元测试，动态地调用某个类的方法，并对方法的返回值进行验证。
    5. 实现代码生成，例如通过反射生成JavaBean的getter和setter方法。

    # 5.3 反射的原理

    Java反射机制是通过反射API提供的类来实现的，主要包括Class、Constructor、Field和Method等。Class类是Java反射API中最基本的类，它代表一个类的定义，可以获取类的信息，包括它的名称、属性、方法等。当一个类被加载到JVM中时，JVM会为该类生成一个Class对象，并将该对象存储在方法区中。反射就是通过这个Class对象来获取和操作类的信息。

    Constructor类代表一个类的构造方法，可以通过它来实例化一个对象。Field类代表一个类的属性，可以获取和设置属性的值。Method类代表一个类的方法，可以调用方法并获取方法的返回值。这些类都是通过Class类的方法来获取的。

    反射的原理是通过Java的类加载机制来实现的。当一个类被加载到JVM中时，JVM会为该类生成一个Class对象，并将该对象存储在方法区中。反射就是通过这个Class对象来获取和操作类的信息。反射可以帮助程序在运行时动态地获取和操作类的信息，从而实现一些动态化的功能。

    # 5.4 反射的使用

    1. 获取类对象

      可以通过Class类的静态方法forName()来获取一个类的Class对象，也可以通过类名的.class属性来获取。下面是一个示例：

      // 第一种，使用 Class.forName 静态方法。当你知道该类的全路径名时，你可以使用该方法获取 Class 类对象。
Class clazz1 = Class.forName("MyClass");
// 第二种，使用 .class 方法。这种方法只适合在编译前就知道操作的 Class。
Class clazz2 = MyClass.class;
// 第三种，使用类对象的 getClass() 方法。
String str = new String("Hello");
Class clz = str.getClass();

    3. 获取属性和方法

      通过Class对象获取Field和Method对象，然后可以通过它们来获取和设置属性值，调用方法等操作。下面是一个示例：

      // 通过 Class 对象的 getFields() 方法可以获取 Class 类的属性，但无法获取私有属性

Field[] fields = clz.getFields();
for (Field field : fields) {
    System.out.println(field.getName());
}
// 而如果使用 Class 对象的 getDeclaredFields() 方法则可以获取包括私有属性在内的所有属性：
Class clz = Apple.class;
Field[] fields = clz.getDeclaredFields();
for (Field field : fields) {
    System.out.println(field.getName());
}

    4. 反射使用完整代码

      import java.lang.reflect.Constructor;
import java.lang.reflect.Field;
import java.lang.reflect.Method;

/**
 * @author Adopat
 * @description: Java basic reflection
 * @date 2023/03/17 15:02
 **/
public class ReflectionDemo {
    public static void main(String[] args) throws Exception {
        // 第一步，获取反射类的 Class 对象：
        Class clazz = Class.forName("java_basic.reflection.Apple");
        // 第二步，通过 Class 对象获取构造方法 Constructor 对象：
        Constructor constructor = clazz.getConstructor(String.class);
        // 第三步，通过 Constructor 对象初始化反射类对象：
        Apple apple = (Apple) constructor.newInstance("红富士");
        // 第四步，获取要调用的方法的 Method 对象：
        Method getName = clazz.getMethod("getName");
        Method setName = clazz.getMethod("setName", String.class);
        System.out.println("修改前 ："+apple);
        // 第五步，通过 invoke() 方法执行：
        setName.invoke(apple, "山西红富士");
        // 获取属性
        Field[] declaredFields = clazz.getDeclaredFields();
        for (Field declaredField : declaredFields) {
            System.out.println("declaredField: " + declaredField.getName());
        }
    }
}
class Apple {
    private String name;

    public Apple() {
    }

    public Apple(String name) {
        this.name = name;
    }

    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }

    @Override
    public String toString() {
        return "Apple{" +
                "name='" + name + '\'' +
                '}';
    }
}

    # 5.5 安全性问题

    由于反射可以访问和操作类的私有属性和方法，因此可能存在安全性问题。为了防止反射攻击，需要在代码中加入一些安全性检查，例如限制反射的使用范围、禁止访问私有属性和方法等。

    下面是一个示例，演示了如何通过反射来访问类的私有属性：

    public class MyClass {
    private String name;
    public void setName(String name) {
        this.name = name;
    }
    private String getName() {
        return name;
    }
}

public class ReflectionDemo {
    public static void main(String[] args) throws Exception {
        Class clazz = MyClass.class;
        MyClass obj = new MyClass();
        Field field = clazz.getDeclaredField("name");
        field.setAccessible(true);
        field.set(obj, "张三");
        Method method = clazz.getDeclaredMethod("getName");
        method.setAccessible(true);
        String name = (String) method.invoke(obj);
        System.out.println(name);
    }
}

    为了防止反射攻击，可以在代码中加入一些安全性检查，例如：

    1. 禁止访问私有属性和方法，例如通过设置setAccessible(false)来禁止访问。
    2. 限制反射的使用范围，例如只允许在特定的包或类中使用反射。
    3. 对反射进行权限检查，例如只允许特定的用户或角色使用反射。
    4. 使用安全性框架，例如Spring Security等，来防止反射攻击。

    # 5.6 性能问题

    由于反射需要在运行时动态获取类的信息，因此会对程序的性能产生一定的影响。为了优化反射性能，可以采取以下措施：

    1. 缓存Class对象，避免重复获取。

    2. 缓存Constructor、Field和Method对象，避免重复获取。

    3. 使用基本类型的包装类来代替原始类型，避免自动装箱和拆箱的开销。

    4. 尽可能使用静态绑定，避免使用动态绑定。

    # 5.7 应用场景

    反射在Java编程中有着广泛的应用，主要包括以下几个方面：

    1. 框架开发：反射可以帮助框架实现自动化配置、自动化装配等功能，使框架更加灵活和易用。

    2. 动态代理：反射可以帮助实现动态代理，动态生成代理对象，从而可以在运行时动态地代理某个对象。

    3. 单元测试：反射可以帮助实现单元测试，动态地调用某个类的方法，并对方法的返回值进行验证。

    4. 代码生成：反射可以帮助生成代码，例如通过反射生成JavaBean的getter和setter方法。

    5. 插件化开发：反射可以帮助实现插件化开发，动态地加载和卸载插件，从而实现动态扩展系统功能。
