    # 1.1 关键字

    以下是Java中常见的关键字及说明：

    关键字 说明
    abstract 用于声明抽象类，以及抽象方法。
    boolean 用于将变量声明为布尔值类型，只有 true 和 false 两个值。
    break 用于中断循环或 switch 语句。
    byte 用于声明一个可以容纳 8 个比特的变量。
    case 用于在 switch 语句中标记条件的值。
    catch 用于捕获 try 语句中的异常
    char 用于声明一个可以容纳无符号 16 位比特的 Unicode 字符的变量。
    class 用于声明一个类
    continue 用于继续下一个循环，可以在指定条件下跳过其余代码。
    default 用于指定 switch 语句中除去 case 条件之外的默认代码块。
    do 通常和 while 关键字配合使用，do 后紧跟循环体。
    double 用于声明一个可以容纳 64 位浮点数的变量。
    else 用于指示 if 语句中的备用分支。
    enum 用于定义一组固定的常量（枚举）
    extends 用于指示一个类是从另一个类或接口继承的
    final 用于指示该变量是不可更改的
    finally 和 try-catch 配合使用，表示无论是否处理异常，总是执行 finally 块中的代码。
    float 用于声明一个可以容纳 32 位浮点数的变量。
    for 用于声明一个 for 循环，如果循环次数是固定的，建议使用 for 循环。
    if 用于指定条件，如果条件为真，则执行对应代码。
    implements 用于实现接口
    import 用于导入对应的类或者接口。
    instanceof 用于判断对象是否属于某个类型（class）
    int 用于声明一个可以容纳 32 位带符号的整数变量。
    interface 用于声明接口。
    long 用于声明一个可以容纳 64 位整数的变量。
    native 用于指定一个方法是通过调用本机接口（非 Java）实现的
    new 用于创建一个新的对象。
    null 如果一个变量是空的（什么引用也没有指向），就可以将它赋值为 null，和空指针异常息息相关。
    package 用于声明类所在的包
    private 一个访问权限修饰符，表示方法或变量只对当前类可见。
    protected 一个访问权限修饰符，表示方法或变量对同一包内的类和所有子类可见。
    public 一个访问权限修饰符，除了可以声明方法和变量（所有类可见），还可以声明类。main() 方法必须声明为 public。
    return 用于在代码执行完成后返回（一个值）。
    short 用于声明一个可以容纳 16 位整数的变量。
    static 表示该变量或方法是静态变量或静态方法
    strictfp 并不常见，通常用于修饰一个方法，确保方法体内的浮点数运算在每个平台上执行的结果相同。
    super 可用于调用父类的方法或者字段
    switch 通常用于三个（以上）的条件判断。
    synchronized 用于指定多线程代码中的同步方法、变量或者代码块
    this 可用于在方法或构造函数中引用当前对象
    throw 主动抛出异常
    throws 用于声明异常。
    transient 修饰的字段不会被序列化
    try 于包裹要捕获异常的代码块。
    void 用于指定方法没有返回值。
    volatile 保证不同线程对它修饰的变量进行操作时的可见性，即一个线程修改了某个变量的值，新值对其他线程来说是立即可见的。
    while 如果循环次数不固定，建议使用 while 循环。
    goto 保留字,不能使用
    const 保留字,不能使用

float f = l; // long转换为float

    # 1.4 基本数据类型缓存池

    • 数据类型缓存池概念

      Java基本数据类型缓存池指的是Java虚拟机在启动时，预先创建了一定范围内的基本数据类型对象，这些对象被缓存起来，以便在程序运行时被重复利用，从而提高程序的性能。

    • 数据类型缓存池分类

      Java基本数据类型缓存池的范围和实现方式可能因不同的JVM版本而异，但通常包括以下数据类型：

      • boolean类型：缓存了true和false两个对象；
      • byte类型：缓存了-128到127之间的所有整数对象；
      • char类型：缓存了0到127之间的所有字符对象；
      • short类型：缓存了-128到127之间的所有整数对象；
      • int类型：缓存了-128到127之间的所有整数对象；
      • long类型：缓存了-128到127之间的所有整数对象；
      • float类型：缓存了所有小数点后只有0或1位的浮点数对象；
      • double类型：缓存了所有小数点后只有0或1位的浮点数对象。

      以下是一些示例代码，演示Java基本数据类型缓存池的使用：

      Integer a = 100; // 自动装箱，从缓存池中获取Integer对象
Integer b = 100; // 自动装箱，从缓存池中获取Integer对象
Integer c = 200; // 自动装箱，创建新的Integer对象
Integer d = new Integer(100); // 创建新的Integer对象

System.out.println(a == b); // true，因为a和b引用的是同一个Integer对象
System.out.println(a == c); // false，因为a和c引用的是不同的Integer对象
System.out.println(a == d); // false，因为a和d引用的是不同的Integer对象

    • Java中的&和&&区别

      Java中的&和&&都是逻辑与运算符，它们的主要区别在于：

      1. 运算规则不同：&是按位与运算符，它会对两个操作数的每个二进制位进行与运算；&&是逻辑与运算符，它只有在两个操作数都为true时才会返回true。

      2. 短路行为不同：&是非短路运算符，即无论第一个操作数的值是什么，都会对第二个操作数进行运算；&&是短路运算符，即如果第一个操作数的值为false，则不再计算第二个操作数。

      以下是一些示例代码，演示&和&&的区别：

      int a = 10;
int b = 3;

boolean c = (a > b) & (a / b > 2); // 按位与运算，结果为false
boolean d = (a > b) && (a / b > 2); // 逻辑与运算，第一个表达式为true，但第二个表达式为false，结果为false

boolean e = (b > a) & (a / b > 2); // 按位与运算，结果为false
boolean f = (b > a) && (a / b > 2); // 逻辑与运算，第一个表达式为false，不再计算第二个表达式，结果为false

      而&&是逻辑与运算符，它只有在两个操作数都为true时才会返回true。在第一个表达式中，第二个操作数需要进行除法运算，但由于第一个表达式已经为false，所以不再计算第二个操作数，整个表达式的结果为false。在第二个表达式中，由于第一个操作数的值为false，所以不再计算第二个操作数，整个表达式的结果为false。

      需要注意的是，在使用&和&&时，应该根据实际需求选择合适的运算符，避免出现意外的结果。如果不确定是否需要短路运算，建议使用&&运算符。

    拷贝数组和复制数组都是将一个数组的内容复制到另一个数组中，但它们之间有以下区别：

    1. 拷贝数组是创建一个新的数组，将原数组的内容复制到新数组中。原数组和新数组是不同的对象，它们的引用地址不同。而复制数组是将原数组的内容复制到另一个已经存在的数组中，原数组和新数组是同一个对象，它们的引用地址相同。

    2. 拷贝数组的目标数组长度可以比原数组长度短或长，当目标数组长度短于原数组长度时，只会复制目标数组长度的元素。而复制数组的目标数组长度必须和原数组长度相同。

    3. 拷贝数组可以使用Arrays.copyOf()方法或System.arraycopy()方法进行操作。而复制数组只能使用System.arraycopy()方法进行操作。

    4. 拷贝数组的返回值是一个新的数组对象，而复制数组没有返回值，它只是将原数组的内容复制到目标数组中。

    总之，拷贝数组和复制数组都是将一个数组的内容复制到另一个数组中，但它们在实现方式和操作结果上存在一些区别。
