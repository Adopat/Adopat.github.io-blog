    # 第八章 正则表达式

    # 8.1 语法基础

    正则表达式的基本语法规则如下：

    • 元字符：元字符是正则表达式中具有特殊含义的字符，例如"."表示任意字符，"\d"表示数字等。
    • 字符集：用方括号"[]"表示，表示匹配方括号中的任意一个字符，例如"[abc]"匹配"a"、"b"或"c"。
    • 量词：用来指定匹配次数，例如"*"表示0次或多次，"+"表示1次或多次，"?"表示0次或1次。
    • 边界符：用来指定匹配的边界，例如"^"表示字符串的开头，"$"表示字符串的结尾。
    • 捕获组
    String regex = "a\\d+"; // 匹配以"a"开头，后面跟1个或多个数字的字符串
String input = "a123";
boolean match = input.matches(regex); // 返回true
// 捕获组 
// 按指定模式在字符串查找
String line = "This order was placed for QT3000! OK?";
// \D 非数字字符匹配。等效于 [^0-9]
// \d 匹配数字
String pattern = "(\\D*)(\\d+)(.*)";

// 创建 Pattern 对象
Pattern r = Pattern.compile(pattern);

// 现在创建 matcher 对象
Matcher m = r.matcher(line);
if (m.find( )) {
    // group(0)），它总是代表整个表达式  Found value: This order was placed for QT3000! OK?
    System.out.println("Found value: " + m.group(0) );
    // Found value: This order was placed for QT
    System.out.println("Found value: " + m.group(1) );
    // Found value: 3000
    System.out.println("Found value: " + m.group(2) );
    // Found value: ! OK?
    System.out.println("Found value: " + m.group(3) );
} else {
    System.out.println("NO MATCH");
}

    在其他语言中，\\ 表示：我想要在正则表达式中插入一个普通的（字面上的）反斜杠，请不要给它任何特殊的意义。

    在 Java 中，\\ 表示：我要插入一个正则表达式的反斜线，所以其后的字符具有特殊的意义

    所以，在其他的语言中（如 Perl），一个反斜杠 \就足以具有转义的作用，而在 Java 中正则表达式中则需要有两个反斜杠才能被解析为其他语言中的转义作用。也可以简单的理解在 Java 的正则表达式中，两个\\代表其他语言中的一个\，这也就是为什么表示一位数字的正则表达式是 \\d，而表示一个普通的反斜杠是\\。

    # 8.2 匹配模式

    • 贪婪匹配：默认情况下，正则表达式会尽可能匹配更多的字符，这种匹配方式称为贪婪匹配。

    • 懒惰匹配：在量词后面加上"?"可以将匹配方式改为懒惰匹配，即尽可能匹配更少的字符。

    • 断言匹配：断言是一种特殊的匹配方式，用来匹配位置而不是具体的字符，例如"^"表示匹配字符串的开头，"(?<=...)"表示向前断言，"(?<!...)"表示向前否定断言。

      在正则表达式中，断言是一种特殊的匹配方式，用于匹配位置而不是具体的字符。根据断言的方向和条件，可以将其分为向前断言和向后断言，以及肯定断言和否定断言(向后断言)。

      1. 向前断言

      向前断言是指匹配当前位置前面的字符，但不包括这些字符在匹配结果中。向前断言的语法格式为"(?<=...)"，其中"..."表示具体的条件。

      例如，"(?<=a)\d+"表示匹配前面有一个字符"a"的，后面跟1个或多个数字的字符串，但不将字符"a"包括在匹配结果中。

      1. 向后断言

      向后断言是指匹配当前位置后面的字符，但不包括这些字符在匹配结果中。向后断言的语法格式为"(?<!...)"，其中"..."表示具体的条件。

      例如，"(?<!\d)b"表示匹配前面是1个或多个数字的，后面跟字符"b"的字符串，但不将数字包括在匹配结果中。

      1. 肯定断言

      肯定断言是指匹配满足某种条件的字符。肯定断言的语法格式为"(?=...)"，其中"..."表示具体的条件。

      例如，"\d+(?=元)"表示匹配以数字结尾且后面跟字符"元"的字符串，但不将字符"元"包括在匹配结果中。

      1. 否定断言

      否定断言是指匹配不满足某种条件的字符。否定断言的语法格式为"(?!...)"，其中"..."表示具体的条件。

      例如，"\d+(?!元)"表示匹配以数字结尾但后面不跟字符"元"的字符串，不将字符"元"包括在匹配结果中。

      总之，断言是一种特殊的匹配方式，可以根据具体的需求选择合适的断言类型进行匹配。

    // 贪婪匹配
String regex1 = "a\\d+"; // 匹配以"a"开头，后面跟1个或多个数字的字符串
String input1 = "a123b456";
Pattern pattern1 = Pattern.compile(regex1);
Matcher matcher1 = pattern1.matcher(input1);
while (matcher1.find()) {
    System.out.println(matcher1.group()); // 输出"a123b456"
}

// 懒惰匹配
String regex2 = "a\\d+?"; // 匹配以"a"开头，后面跟1个或多个数字的字符串，懒惰匹配
String input2 = "a123b456";
Pattern pattern2 = Pattern.compile(regex2);
Matcher matcher2 = pattern2.matcher(input2);
while (matcher2.find()) {
    System.out.println(matcher2.group()); // 输出"a1"
}

// 断言匹配
String regex3 = "(?<=a)\\d+"; // 匹配以"a"开头，后面跟1个或多个数字的字符串，前面必须是"a"
String input3 = "a123b456";
Pattern pattern3 = Pattern.compile(regex3);
Matcher matcher3 = pattern3.matcher(input3);
while (matcher3.find()) {
    System.out.println(matcher3.group()); // 输出"123"
}

    # 8.3 实战应用

    正则表达式在实际开发中的应用场景非常广泛，例如字符串匹配、数据验证、文本替换等。

    // 字符串匹配
String regex = "\\d{4}-\\d{2}-\\d{2}"; // 匹配日期格式"yyyy-MM-dd"
String input = "Today is 2022-01-01";
Pattern pattern = Pattern.compile(regex);
Matcher matcher = pattern.matcher(input);
if (matcher.find()) {
    System.out.println(matcher.group()); // 输出"2022-01-01"
}

// 数据验证
String regex = "^\\d{6}$"; // 匹配6位数字
String input = "123456";
boolean match = input.matches(regex); // 返回true

// 文本替换
String regex = "\\b(\\w+)\\b"; // 匹配单词
String input = "Hello world!";
Pattern pattern = Pattern.compile(regex);
Matcher matcher = pattern.matcher(input);
String result = matcher.replaceAll("<$1>"); // 将单词用"<>"包裹起来
System.out.println(result); // 输出"<Hello> <world>!"

    # 8.4 性能优化

    正则表达式的性能问题主要涉及回溯、分支、嵌套等因素对性能的影响，以及如何优化正则表达式的性能。

    // 回溯优化
String regex = "(a|b)+"; // 匹配由"a"或"b"组成的字符串
String input = "abababababababababababababababababababababababababababababab";
Pattern pattern = Pattern.compile(regex);
Matcher matcher = pattern.matcher(input);
while (matcher.find()) {
    // do something
}

// 分支优化
String regex = "a(\\d{2}|\\d{3})b"; // 匹配"a"、2或3个数字、"b"组成的字符串
String input = "a123b";
Pattern pattern = Pattern.compile(regex);
Matcher matcher = pattern.matcher(input);
if (matcher.matches()) {
    // do something
}

// 嵌套优化
String regex = "\\(([^()]+|\\([^()]*\\))*\\)"; // 匹配括号内的内容，支持嵌套
String input = "(a(b(c)d)e)f";
Pattern pattern = Pattern.compile(regex);
Matcher matcher = pattern.matcher(input);
while (matcher.find()) {
    // do something
}

    # 8.6 库和框架

    Java自带的java.util.regex包提供了基本的正则表达式功能，第三方库和框架则提供了更加丰富和高效的正则表达式功能。

    // Java自带的java.util.regex包
String regex = "\\d{4}-\\d{2}-\\d{2}"; // 匹配日期格式"yyyy-MM-dd"
String input = "Today is 2022-01-01";
Pattern pattern = Pattern.compile(regex);
Matcher matcher = pattern.matcher(input);
if (matcher.find()) {
    System.out.println(matcher.group()); // 输出"2022-01-01"
}

// 第三方正则表达式库：https://github.com/VerbalExpressions/JavaVerbalExpressions
VerbalExpression regex = VerbalExpression.regex().startOfLine().then("http").maybe("s").then("://").maybe("www.").anythingBut(" ").endOfLine();
String input = "Visit us at http://www.google.com";
if (regex.test(input)) {
    System.out.println("Matched!");
}

// 第三方文本处理框架：https://lucene.apache.org/core/
String regex = "\\b(\\w+)\\b"; // 匹配单词
String input = "Hello world!";
Pattern pattern = Pattern.compile(regex);
Matcher matcher = pattern.matcher(input);
String result = matcher.replaceAll("<$1>"); // 将单词用"<>"包裹起来
System.out.println(result); // 输出"<Hello> <world>!"
