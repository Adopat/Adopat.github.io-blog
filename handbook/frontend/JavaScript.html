    # 第四章 JavaScript基础

    # 4.1 JavaScript简介

    • ECMAscript概述

      ECMAscript简称ES，是JavaScript的标准，我们经常说的ES5，ES6等等，可以称作JavaScript的版本，我们在之前学过的所有JavaScript特性，都是基于ES5版本的，今天我们开始讲解的是ES6标准的特性。ES6已更名为ES2015，ES7等后续的版本，我们都可以统称为ES2015+

    • JavaScript定义

      是一种脚本语言，可以用来创建动态更新的内容，控制多媒体，制作图像动画。在语言分类上JavaScript是一门动态语言，解释型语言 客户端语言。

    • JS执行线程的运行机制及其他线程的搭配

      (1) 主线程按顺序执行代码，当碰见 setTimeout、setInterval、XmlHttpRequest、DOM 事件 等 Api 时，会将其交给对应的定时器线程、XmlHttpRequest 线程、事件线程处理，然后主线程继续执行后面的代码

      (2) 定时器线程、XmlHttpRequest 线程、事件线程的任务触发后，会将回调函数添加至 JS 线程的任务队列中

      (3) 主线程内的任务全部执行完成后，会调用事件循环器拉取任务队列中的任务到主线程，至此一个事件循环周期结束，然后重复 (1) (2) (3) 来完成后续周期。

    • 静态语言与动态语言

      静态语言和动态语言是编程语言的两种常见类型。静态语言是指在编译时就已经确定了变量的数据类型，而动态语言则是在运行时才确定变量的数据类型。常见的静态语言包括Java、C++，而常见的动态语言包括Python、Ruby JavaScript。

    • 服务端语言与客户端语言

      服务端语言和客户端语言是根据程序运行的位置来划分的。服务端语言运行在服务器上，用于处理请求并生成响应；客户端语言则运行在用户的计算机上，用于显示网站或应用程序的界面。常见的服务端语言包括PHP、Node.js，而常见的客户端语言包括JavaScript。

    • 解释型语言与编译型语言

      解释型语言和编译型语言也是根据程序运行方式来划分的。解释型语言是指程序在运行时逐行翻译并执行，因此运行速度较慢，但代码更容易调试和修改，常见的解释型语言包括Python、Perl；编译型语言则是指程序在编译时被翻译成机器码，再运行这些机器码，因此运行速度较快，但修改和调试比较困难。常见的编译型语言包括C、C++，Java 既可以编译亦可解释。

      虽然JavaScript是一种解释型语言，但它的执行顺序并不完全是从上到下依次执行。这是因为JavaScript引入了异步编程的概念，其中回调函数是一种常用的实现方式。

      在异步编程中，回调函数会被先注册并且延迟执行，直到某些事件（如网络请求返回数据、定时器到期等）发生后才会被触发执行。因此，在这种情况下，JavaScript的程序执行顺序就不再是简单的从上到下依次执行，而是由事件驱动的非线性执行。

      需要注意的是，JavaScript仍然是一种解释型语言，而回调函数只是改变了程序执行顺序的一种机制，并没有违背解释型语言的特性。由于异步编程的使用越来越广泛，理解JavaScript的事件驱动模型和回调函数机制对于开发高效可靠的应用程序至关重要。

    • 变量的默认值

      JavaScript中的变量如果没有初始化时没有默认值的

      Java中变量的默认值和JavaScript中变量的默认值有以下区别和联系：

      1. 数据类型： Java和JavaScript都是弱类型语言，但Java具有静态类型，而JavaScript是动态类型。Java中的变量必须先声明其数据类型，然后才能使用。在声明变量时，如果未对其进行显式初始化，则其将自动设置为其数据类型的默认值。相比之下，JavaScript中的变量可以在任何时间声明，并且其数据类型由其所包含的值决定。

      2. 默认值：Java中的变量在声明时会自动分配一个默认值，而JavaScript中的变量则不会。在Java中，数值类型（byte、short、int、long、float、double）的默认值为0，布尔类型的默认值为false，字符类型的默认值为'\u0000'（空字符），而对象引用类型的默认值为null。在JavaScript中，未显式初始化的变量默认值为undefined。

      3. 可赋值性：在Java中，如果您尝试访问未初始化的变量，您将收到编译时错误。这是因为Java中的所有变量都必须初始化，否则编译器将无法确定它们是否已准备好使用。在JavaScript中，未初始化的变量将默认为undefined，但可以随时赋值新值。

      4. 局部变量与成员变量：在Java中，成员变量（即类级别变量）的默认值与局部变量的默认值不同。对于成员变量，如果未显式初始化，其默认值与局部变量相同，即0、false、'\u0000'或null。在JavaScript中，所有变量的默认值都相同，无论它们是全局变量还是局部变量。

      综上所述，虽然Java和JavaScript都具有变量默认值的概念，但它们的实现方式略有不同。Java的默认值是根据数据类型自动分配的，而JavaScript的默认值始终为undefined。此外，在Java中，变量必须先声明其数据类型才能使用，并且未初始化的变量将导致编译时错误。在JavaScript中，变量可以在任何时间声明，并且未初始化的变量将默认为undefined，但它们可以随时赋值新值。

    • 变量的默认值的设置方法

      在JavaScript中，变量可以通过不同的方式来赋予默认值：

      1. 使用条件运算符（ternary operator）：这是一种简单的方法，只需在变量声明时使用三元运算符，即可为其赋予默认值。示例代码如下：
      let myVar = someValue ? someValue : defaultValue;
      1
      1. 使用逻辑或运算符（logical OR operator）：当变量的值为false、null、undefined等“假值”时，可以使用逻辑或运算符为其赋予默认值。示例代码如下：
      let myVar = someValue || defaultValue;
      1
      1. 使用ES6中的默认参数（default parameters）：在函数定义中，可以为函数的参数指定默认值。当函数调用时该参数未被传入实参，则会使用该默认值。示例代码如下：
      function myFunc(myParam = defaultValue) {
  // ...
}
      1
      2
      3

      需要注意的是，在使用逻辑或运算符和默认参数时，如果变量已经被赋值为一个“假值”，则会直接返回该“假值”而不使用默认值。因此，应该谨慎地选择适合自己的默认值赋值方法。

    • Null理解

      在 JavaScript 中，null 是一个特殊的值，表示“空值”或“没有对象”。当你想要明确地表示一个变量没有引用任何对象时，可以将其赋值为 null。

      与 undefined 不同，null 是一个有效的 JavaScript 值。如果你使用 typeof 运算符检查 null 变量的类型，它会返回 "object"，这是因为 JavaScript 将 null 视为一个空对象引用。

      需要注意的是，null 和 undefined 在 JavaScript 中有不同的含义。undefined 表示未定义的值，即声明但未初始化的变量，或者没有返回任何值的函数。而 null 表示一个已经定义了但值为空的变量。

      在编写代码时，你应该尽可能避免使用 null 和 undefined。可以通过使用默认值、条件语句和错误处理来避免这些值的出现。如果必须使用 null，那么请确保你的代码能够正确处理它们。

    • Undefined理解

      在JavaScript中，undefined是一个原始值，表示一个变量或属性未被赋值。如果一个变量已经声明，但没有被显式地初始化为任何值，那么它的默认值就是undefined。例如：

      let x;
console.log(x); // undefined
      1
      2

      另外，在访问对象的属性时，如果该属性不存在，那么返回的也是undefined。例如：

      const obj = { a: 1 };
console.log(obj.b); // undefined

      2

      需要注意的是，尽管undefined是一个原始值，但它实际上是一个全局变量，在ECMAScript标准中有明确规定。这意味着在代码中可以使用undefined来表示未定义的值，但不应该将其直接赋值为其他值。推荐使用typeof操作符来检测一个变量是否为undefined。例如：

      let x;

  console.log('x is undefined');

        Java 是静态类型语言，必须在定义数组时指定数据类型。这种限制使得 Java 数组只能存储同一种类型的元素，因此不同类型的数据无法混合存储。

        JavaScript 和 Python 的设计符合动态类型语言的灵活性和简单性的特点。在很多情况下，我们需要在一个数组中存储不同类型的数据，比如处理表单数据，或者将不同类型的数据放在一个容器中进行处理等。而静态类型语言则更加注重类型安全，在编译期间就要保证类型匹配，以避免运行时出现错误。

        因此说，数组的设计是否支持存储不同类型的数据与静态语言和动态语言有关。动态类型语言的灵活性使得数组可以存储不同类型的数据，这是一种更加自由的设计选择。

        设计eval()函数的主要目的是为了支持动态编程，它使得代码可以在运行时生成和修改。使用eval()函数，开发人员可以通过字符串形式的代码来实现动态构建函数、变量和对象等操作。这种能力在某些情况下非常有用，比如在编写插件、模板引擎、动态表单验证等方面。

        然而，由于eval()函数可以执行任何有效的JavaScript或Python代码，它也可能导致安全性问题，因为恶意用户可以通过传递恶意代码来执行攻击。此外，由于eval()函数需要在运行时进行代码解析和编译，它通常比静态编译的代码慢，并会占用更多的资源。

        因此，在使用eval()函数时，需要谨慎考虑其安全性和性能方面的问题，并尽量避免不必要的使用。如果确实需要使用eval()函数，应该对输入进行验证和过滤，以确保只执行合法的代码。

    # 4.2.14 前端存储技术

    • 概念

      后端常用数据库做数据存储，譬如MySql、MongoDB，缓存技术存储数据，如Redis、Memcached；前端存储数据目前常用的是Cookie、Storage、IndexedDB

    • Cookie

      HTTP Cookie（也叫Web Cookie或浏览器Cookie）是服务器发送到用户浏览器并保存在本地的一小块数据，它会在浏览器下次向同一服务器再发起请求时被携带并发送到服务器上。通常，它用于告知服务端两个请求是否来自同一浏览器，如保持用户的登录状态。Cookie使基于无状态的HTTP协议记录稳定的状态信息成为了可能。

    • Storage

      作为 Web Storage API的接口，Storage 提供了访问特定域名下的会话存储（session storage）或本地存储（local storage）的功能，例如，可以添加、修改或删除存储的数据项。

      如果你想要操作一个域名的会话存储（session storage），可以使用 window.sessionStorage如果想要操作一个域名的本地存储（local storage），可以使用 window.localStorage。

      sessionStorage和localStorage的用法是一样的，区别在于sessionStorage会在会话关闭也就是浏览器关闭时失效，而localStorage是将数据存储在本地，不受关闭浏览器影响，除非手动清除数据

    • IndexedDB

      随着浏览器的功能不断增强，越来越多的网站开始考虑，将大量数据储存在客户端，这样可以减少从服务器获取数据，直接从本地获取数据。

      现有的浏览器数据储存方案，都不适合储存大量数据：Cookie的大小不超过 4KB，且每次请求都会发送回服务器；LocalStorage在 2.5MB 到 10MB 之间（各家浏览器不同），而且不提供搜索功能，不能建立自定义的索引。所以，需要一种新的解决方案，这就是 IndexedDB 诞生的背景。

      通俗地说，IndexedDB就是浏览器提供的本地数据库，它可以被网页脚本创建和操作。IndexedDB允许储存大量数据，提供查找接口，还能建立索引。这些都是LocalStorage 所不具备的。就数据库类型而言，IndexedDB不属于关系型数据库（不支持 SQL查询语句），更接近 NoSQL数据库。

    • 回调函数使用场景

    # 4.5.3 Promise对象

    • Promise基本概念

      • Promise概念

        Promise是一种用于处理异步操作的JavaScript对象。它提供了一种更优雅的方式来编写和组织异步代码，使其更易于理解和维护。

      • Promise的作用

        Promise的作用是封装一个异步操作，并支持链式调用，以便在异步操作完成时能够获取结果或错误信息。

      • Promise优势

        1. 简化异步处理流程，避免回调地狱。
        2. 支持链式调用，使异步操作组合更加灵活。
        3. 提供了统一的错误处理机制，方便捕获和处理错误。
        4. 可以通过Promise.all等方法实现多个异步操作的并发执行。

      • Promise使用场景

        1. 处理AJAX请求 - 在使用AJAX进行Web开发时，Promise可以用于处理异步操作并等待数据返回。
        2. 处理文件读写操作 - 在Node.js中，Promise可用于处理文件读取和写入操作。
        3. 处理动画效果 - Promise可用于处理动画效果，例如在前端网页中实现一个动态加载的进度条。
        4. 处理多个异步操作 - Promise提供了一种优雅的方式来处理多个异步操作，例如在一个应用程序中使用多个API调用。
        5. 错误处理 - Promise提供了一种便捷的错误处理机制，例如在产生错误时能够在Promise链中正确地处理异常情况。
        6. 流程控制 - Promise可以帮助您控制代码执行的流程，例如在某个异步操作完成后再执行下一个异步操作。
        7. 缓存结果 - Promise可以缓存非常耗时的操作，例如对于从服务器获取的数据，可以将其缓存到本地存储中，以减少重复请求。

      • Promise的状态

        • pending 状态：表示进行中。Promise 实例创建后的初始态；
        • fulfilled 状态(成功态resolved)：表示成功完成。在执行器中调用 resolve 后达成的状态；
        • rejected 状态：表示操作失败。在执行器中调用 reject 后达成的状态。

      • Promise状态转换

        • pending ----- resolve() --> resolved;
        • pending ----- reject() --> rejected;

    # 4.6.6 类

    JavaScript中的类是一种面向对象编程的概念，它允许开发者定义一个抽象模板来创建对象实例。 它们使用class关键字定义。 类可以包含属性和方法，属性是类的状态，方法是类的行为。

    在JavaScript中，类与原型继承紧密相关。声明一个类时，实际上是在创建一个新的函数，使用构造函数语法创建实例。类的方法和属性被添加到该函数的prototype属性上，这些方法和属性可以被该类的所有实例共享。

    使用类的好处之一是可以方便地实现面向对象编程的三大特性：封装、继承和多态。 这使得代码更加模块化，易于维护和扩展。

    下面是一个简单的JavaScript类的示例：

    class Person {
  constructor(name, age) {
    this.name = name;
    this.age = age;
  }
  
  sayHello() {
    console.log(`Hello, my name is ${this.name} and I'm ${this.age} years old.`);
  }
}

const person1 = new Person("Alice", 30);
person1.sayHello(); // 输出 "Hello, my name is Alice and I'm 30 years old."

    # 4.6.7 模块化

    JavaScript中的模块化是一种组织和封装代码的方法，它可以将代码分割成可重用的部分，并使得每个部分仅暴露出需要对外公开的接口。这样做可以降低代码的耦合度，提高代码的可维护性和可扩展性。

    在ES6之前，JavaScript并没有原生支持模块化，通常使用IIFE（立即执行函数表达式）或者CommonJS规范等方式来实现模块化。而在ES6中，引入了一套新的模块化规范，即ES6模块化规范。

    ES6模块化规范定义了如何导出和导入模块。一个模块可以通过export关键字将指定变量、函数或对象暴露给其他模块使用；另一个模块则可以通过import关键字来引入所需的模块中的变量、函数或对象。ES6模块化支持静态分析，可以在编译时进行优化，减少运行时的开销。

    下面是一个简单的ES6模块化示例：

    // module.js
export const PI = 3.14;
export function sayHello(name) {
  console.log(`Hello, ${name}!`);
}

// main.js
import { PI, sayHello } from './module.js';
console.log(PI); // 输出3.14
sayHello('ChatGPT'); // 输出“Hello, ChatGPT!”
