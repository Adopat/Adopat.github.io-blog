    # 3.1.2 css中常见的数值单位

    • 像素（px）：像素是一个相对固定的单位，它基于屏幕的分辨率来计算。在大多数情况下，1 像素等于屏幕上的一个物理像素点。
    • 百分比（%）：百分比可以被用于相对于另一个元素的大小来指定一个元素的大小。例如，如果一个元素的宽度设置为50%，那么它将会是其容器宽度的一半。
    • em：em 是一个相对的单位，它基于字体的大小来计算。1 em 等于当前字体的大小。因此，如果字体大小为16像素，那么1 em 就等于16 像素。
    • rem：rem 也是一个相对单位，但它始终相对于根元素的字体大小计算。根元素通常是 <html> 标签。因此，如果根元素的字体大小为16像素，那么1 rem 就等于16像素。
    • vw 和 vh：vw 和 vh 分别表示视口的宽度和高度的百分比。例如，如果一个元素的宽度设置为50vw，那么它将是视口宽度的一半。
    • pt：pt 是一个绝对单位，1 pt 相当于 1/72 英寸。
    • cm、mm 和 in：这些单位表示厘米、毫米和英寸，它们也是绝对单位。1cm = 37.8px = 25.2/64in
    • 数值单位参考资料 css的值和单位 (opens new window)

    # 3.2.4 伪类和伪元素选择器

    • 伪类

      伪类是选择器的一种，它用于选择处于特定状态的元素，比如当它们是这一类型的第一个元素时，或者是当鼠标指针悬浮在元素上面的时候。它们表现得会像是你向你的文档的某个部分应用了一个类一样，帮你在你的标记文本中减少多余的类，让你的代码更灵活、更易于维护。

      伪类就是开头为冒号的关键字：

      伪类实例代码

      <style>
        /* 伪类 */
        article p:first-child {
            font-size: 120%;
            font-weight: bold;
        }

        article p:last-child {
            font-size: 120%;
            font-weight: bold;
            background-color: rebeccapurple;
        }
        /* 用户行为伪类 */
        a:link,
        a:visited {
            color: rebeccapurple;
            font-weight: bold;
        }

        a:hover {
            color: hotpink;
        }
    </style>

    # 3.2.5 选择器权重

    一个选择器的优先级可以说是由三个不同的值（或分量）相加，可以认为是百（ID）十（类）个（元素）——三位数的三个位数：

    • ID：选择器中包含 ID 选择器则百位得一分。(一个页面只能有一个独特的ID)
    • 类：选择器中包含类选择器、属性选择器或者伪类则十位得一分。（很多元素都可以有相同的类）
    • 元素：选择器中包含元素、伪元素选择器则个位得一分。

    注意 通过在样式的结尾设置!important，可以将此样式优先级设置为最高，实例代码如下所示：

    <!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Document</title>
    <style>
        h1{
            color:red !important;
        }
    </style>
</head>
<body>
    <h1 style="color:yellow">hello world</h1>
</body>
</html>

    # 3.3.1 颜色属性

    • 颜色设置方式

      • 颜色关键字

        为了纪念 Eric Meyer 的女儿，色值 #663399 在 CSS4 中被定义为 「rebeccapurple」 (opens new window)

        例如red、blue、green等，但是这种方式并不是一种严格的、可靠的方法来描述颜色。

        原因是，不同的浏览器和操作系统对这些颜色关键字的支持程度可能会有所不同，并且在不同的设备上显示出来的颜色也可能存在差异。此外，由于颜色关键字只能表达有限的颜色范围，无法满足所有的设计需求。

        因此，为了更加精细化地控制颜色，CSS提供了其他多种方式来设置颜色，如RGB、十六进制、HSL等。这些方式可以实现更加精确的颜色描述，并且具有更好的跨平台和浏览器兼容性。

      • 十六进制颜色值

        使用6个十六进制数字（0-9和A-F）来表示颜色，前两位表示红色通道，中间两位表示绿色通道，后两位表示蓝色通道。例如，红色可以表示为#FF0000，绿色可以表示为#00FF00。

      • HSL颜色模式（Hue-Saturation-Lightness）

        使用HSL颜色模式可以更直观地控制颜色的亮度、饱和度和色相。HSL颜色值由三个参数组成：色相（hue）、饱和度（saturation）和亮度（lightness），分别表示颜色的基本色调、纯度和明暗程度。例如，红色可以表示为hsl(0, 100%, 50%)，其中0表示红色的色相，100%表示完全饱和度，50%表示中等亮度。

      • RGB

        RGB表示红绿蓝（Red-Green-Blue），即使用红色、绿色和蓝色三个通道的不同强度来混合生成各种颜色。在CSS中，RGB颜色值由三个参数组成，分别表示红色、绿色和蓝色通道的强度，取值范围为0~255，其中0表示没有该颜色通道的贡献，255则表示该颜色通道的最大贡献。

        例如，红色可以表示为rgb(255, 0, 0)，其中第一个参数表示红色通道的强度为255，其他两个通道的强度为0。深灰色可以表示为rgb(60, 60, 60)，其中三个通道的强度均为60。通过调整三个通道的强度比例，我们可以生成各种不同的颜色。

        使用CSS RGB可以实现更精细化的颜色控制，并且具有良好的跨平台和浏览器兼容性。

      • RGBA颜色值（Red-Green-Blue-Alpha）

        RGBA颜色值和RGB颜色值类似，但是多了一个透明度参数（alpha），用于控制颜色的不透明度。透明度的取值范围是从0（完全透明）到1（完全不透明）。例如，半透明黑色可以表示为rgba(0, 0, 0, 0.5)，其中0.5表示50%的不透明度。

      • HSLA颜色值（Hue-Saturation-Lightness-Alpha）

        类似于RGBA颜色值，HSLA颜色值也多了一个透明度参数。例如，半透明红色可以表示为hsla(0, 100%, 50%, 0.5)。

      网络安全字体（Web Safe Fonts）是指在大多数操作系统和浏览器中都能够正常显示的一组字体。这些字体经过广泛使用和测试，被认为是最稳定、最可靠的字体选择，因此在网页设计中非常常用。

    • flexbox（弹性布局）：通过定义容器和子元素的属性来实现灵活的布局方式。常用于实现各种复杂的自适应布局。
    • grid（网格布局）：将页面划分为多个网格来进行布局，提供更加灵活的布局能力。
    • box-sizing：设置盒子模型的计算方式，可以是内容盒子（content-box）或边框盒子（border-box）。常用于控制元素大小的计算方法。
    • margin、padding：设置元素的外边距和内边距，常用于调整元素的间距和布局效果。
    • width、height：设置元素的宽度和高度，常用于控制元素的大小和布局效果。

    这些属性可以组合使用，实现各种复杂的布局效果。在实际开发中，应该根据需求选择合适的方法，并遵循良好的代码规范和可维护性原则。同时，在进行复杂布局时，也要注意兼容性问题和性能优化。

    # 3.4.1 盒子模型的基本概念

    • border：边框,包裹在padding外面的边框线条，可以通过border属性来设置。

    # 3.4.3 盒子模型的分类

    • 标准模型

      指定的 width 和 height 为content内容的宽度和高度，此时

      width = content width + padding left + padding right + border left + border right height = content height + padding top + padding bottom + border top + border bottom

    • 替代模型

      指定的 width 和 height 就是盒子的宽度和高度

    • 两种模型如何选择

      1. 标准模型盒子：适合需要对内边距和边框做特殊处理的情况。例如，有时候需要让元素的背景色只覆盖到内容区域，而不包括内边距和边框，这时就可以使用标准模型盒子。
      2. 替代模型盒子：适合像图片这样的替代元素。由于替代元素本身就包含了padding和border，所以使用替代模型可以更方便地控制元素的大小。另外，在某些情况下也可以使用替代模型来简化布局代码。

    • 两种模型如何切换

      • box-sizing: content-box（默认值）：使用标准模型进行尺寸计算。
      • box-sizing: border-box：使用替代模型进行尺寸计算。

    # 3.4.4 盒子模型的应用

    可以通过CSS样式来控制盒子模型的各个属性，例如：

    • width、height：设置元素的宽度和高度。
    • background-color：设置元素的背景色。
    • border-style、border-width、border-color：分别设置边框的样式、宽度、颜色。
    • padding、margin：分别设置内边距和外边距的大小。

    # 3.4.4 盒子模型的兼容性

    不同浏览器对盒子模型的解析方式可能存在差异，主要表现在盒子模型计算尺寸时是否包含边框和内边距。在CSS3中，可以通过box-sizing属性来设置计算尺寸时是否包含边框和内边距。例如：

    • box-sizing: content-box（默认值）：计算尺寸时不包含边框和内边距。
    • box-sizing: border-box：计算尺寸时包含边框和内边距。

    # 3.5 布局技术

    # 3.5.1 正常布局流

    • 概念

      正常布局流（normal flow）是指在不对页面进行任何布局控制时，浏览器默认的 HTML 布局方式。没有使用CSS进行布局设置的话就是默认的正常布局流。

    • 块元素和内联元素

      块级元素

      • 块级元素以新行开始，占据父容器的整个可用宽度。
      • 可以通过设置宽度、高度、边距和填充来控制它们的尺寸和位置。
      • 块级元素可以包含其他块级元素和内联元素。
      • 常见的块级元素有<div>、<h1>-<h6>、<p>、<ul>、<ol>等。

      内联元素

      • 内联元素不会以新行开始，只占据它们实际内容所需的宽度。
      • 无法设置宽度和高度，它们的大小由其内容决定。
      • 内联元素不能包含其他块级元素，仅能包含其他内联元素或文本节点。
      • 常见的内联元素有<a>、<span>、<img>、<strong>、<em>等。

    # 3.5.2 display属性

    • 概念

      CSS 的 display 属性用于定义 HTML 元素应该如何显示，可以控制元素在页面中的布局和盒模型类型。

    • 常见属性值

      • block: 元素将被渲染为块级元素，并且在文档流中占据整个父容器的宽度。块级元素通常用于构建网页布局的主要结构（例如段落、标题、列表等）。
      • inline: 元素将被渲染为行内元素，并且只占据其内容所需的宽度。行内元素通常用于包含文本或其他行内元素（例如超链接、强调文本、图片等）
      • inline-block: 元素将被渲染为行内块级元素，这意味着它具有块级元素的尺寸和边距，但是仍然流动在同一行上。行内块级元素通常用于创建水平排列的元素（例如按钮、图标等）。
      • none: 元素将被隐藏，不会在页面上显示，也不会占据空间。这通常用于通过 JavaScript 控制元素的可见性。
      • flex: 元素将被渲染为弹性容器，可以通过 flex-direction 和其他属性对其子元素进行灵活的布局。
      • grid: 元素将被渲染为网格容器，可以通过 grid-template-columns 和其他属性对其子元素进行网格布局。
      • table: 元素将被渲染为表格容器，可以通过 table-cell 等其他属性控制表格的行和列

    # 3.5.3 flex 弹性布局

    • 概念

      flexbox 是 CSS 弹性盒子布局模块（Flexible Box Layout (opens new window) Module）的缩写，它被专门设计出来用于创建横向或是纵向的一维页面布局。要使用 flexbox，你只需要在想要进行 flex 布局的父元素上应用display: flex ，所有直接子元素都将会按照 flex 进行布局。

    • 常用属性

      • display 属性：指定一个元素使用 flex 布局，值为 flex。
      • flex-direction 属性：指定主轴的方向，可选值有 row（默认值，水平方向）、column（垂直方向）、row-reverse 和 column-reverse。
      • justify-content 属性：定义了项目在主轴上的对齐方式，可选值有 flex-start（默认值，左对齐）、flex-end（右对齐）、center（居中对齐）、space-between（两端对齐）、space-around（每个项目周围都有空间）和 space-evenly（每个项目之间和周围都有空间）。
      • align-items 属性：定义了项目在交叉轴上的对齐方式，可选值有 stretch（默认值，拉伸填满整个容器高度）、flex-start（顶部对齐）、flex-end（底部对齐）、center（垂直居中对齐）和 baseline（基线对齐）。
      • flex-wrap 属性：指定了项目是否换行，可选值有 nowrap（默认值，不换行）、wrap（换行，第一行在上方）和 wrap-reverse（换行，第一行在下方）。
      • align-content 属性：定义了多根轴线的对齐方式，可选值有 stretch（默认值，轴线占满整个交叉轴）、flex-start（轴线顶部对齐）、flex-end（轴线底部对齐）、center（轴线垂直居中对齐）、space-between（轴线之间平均分布）和 space-around（轴线周围平均分布）。
      • flex-grow 属性：定义了项目的放大比例，默认值为 0，即不放大。如果有多余空间，该元素将按照比例分配剩余空间。
      • flex-shrink 属性：定义了项目的缩小比例，默认值为 1，即可以缩小。如果空间不足，该元素将按照比例缩小。
      • flex-basis 属性：指定了项目在主轴上的初始大小。默认值为 auto，即使用项目本身的宽度或高度。
      • flex 属性：是 flex-grow、flex-shrink 和 flex-basis 三个属性的缩写形式。例如，flex: 1 表示 flex-grow: 1; flex-shrink: 1; flex-basis: 0%;。

    # 3.5.4 Grid布局

    • 概念

      CSS 网格是一个用于 web 的二维布局系统。利用网格，你可以把内容按照行与列的格式进行排版。另外，网格还能非常轻松地实现一些复杂的布局。

    • 常用属性

      • display: grid;：将元素变为一个网格容器。
      • grid-template-columns 和 grid-template-rows：定义网格的列和行。可以使用像像素、百分比、自动(用于自适应内容)、fr(分数单位)和minmax()函数来定义网格的大小。
      • grid-column-start、grid-column-end、grid-row-start 和 grid-row-end：这些属性可以用来定义单个单元格的位置，以及跨越多个单元格的区域
      • grid-template-areas：通过对网格单元格进行命名，使得布局在代码中更易于理解和修改。
      • grid-gap：用于定义网格单元格之间的间隔。(不推荐使用，建议直接使用 gap)
      • grid-auto-flow：定义当网格容器中的所有单元格都被放置完毕时，新的单元格应该如何排列。
      • align-items 和 justify-items：用于控制网格单元格内部的对齐方式。
      • align-content 和 justify-content：用于控制整个网格在父容器内的对齐方式。

      父级容器不管是绝对定位，相对定位还是固定定位，都可以实现作为参照物的效果，在实际项目开发中，常用方法如下所示：

    • 各种定位方式的使用场景

      • static：默认值，元素在文档流中正常排列。

      • relative：相对定位，元素在文档流中正常排列，但可以通过top、bottom、left和right属性调整位置，且不影响其它元素的布局。

      • absolute：绝对定位，元素脱离文档流，相对于最近的已定位祖先元素（如果没有则相对于body）进行定位。通常用于创建浮动层或者弹出窗口等需要完全独立于文档流的元素。

      • fixed：固定定位，元素相对于浏览器视口进行定位，随页面滚动而固定不动。通常用于创建导航菜单、悬浮按钮等需要一直显示在页面上的元素。

      • sticky：粘性定位，元素首先在文档流中正常排列，当滚动到指定位置时，变为固定定位，直到滚动回到初始位置时恢复为相对定位。通常用于创建头部导航条或者侧边栏等需要在页面滚动过程中保持可见的元素。

    # 3.6 响应式设计

    # 3.6.1 响应式页面

    • 概念

      一套代码可以同时适应多个设备。这样的网页就是响应式网页。这样的网页是通过媒体查询来进行实现的。

    • 响应式优缺点

      优点：

      1. 提高用户体验：响应式设计可以使网站在不同设备上表现出色，并且可以更好地适应不同的屏幕尺寸和分辨率，提高用户使用该网站的体验。
      2. 增加可访问性：响应式设计可以使网站易于访问，因为用户可以在任何设备上浏览网站而无需担心兼容性问题。
      3. 维护成本低：相对于维护多个独立的站点，维护一个响应式网站只需要一个代码库，因此可以降低开发和维护成本。
      4. 有利于SEO：Google等搜索引擎偏好响应式设计的网站，因为这样的网站具有更好的页面质量、内容丰富度和用户友好性。
      5. 更好的跨平台兼容性：响应式设计使得网站能够在不同操作系统和设备之间保持一致的外观和功能，从而提高跨平台兼容性。

      缺点：

      1. 页面加载速度慢：当在较小的设备上加载大量的内容和资源时，响应式网站可能会导致页面加载速度变慢。
      2. 设计限制：为了实现响应式设计，开发人员必须考虑多种设备尺寸和分辨率，这可能会对设计带来一定限制，比如需要放弃某些复杂的特效或布局。
      3. 浏览器兼容性问题：不同浏览器对CSS3等新技术的支持有所不同，这可能导致在某些浏览器上显示不正常。
      4. 开发复杂度高：响应式设计需要考虑多个屏幕大小和分辨率，因此需要更多的测试和调试，开发难度相对较高。
      5. 难以创建精细的用户体验：由于响应式设计需要考虑多种设备和屏幕尺寸，因此很难为所有用户提供完美的用户体验。

    • 什么时候选用响应式页面

      1. 网页本身并不复杂。
      2. 对用户体验要求不高。
      3. 希望多终端访问，又希望降低开发成本。

    • 过度效果

      CSS3的过渡效果可以让一个元素的某一个或多个css属性从一个值平滑过渡到另一个值。可以用一个transition来实现元素的过渡效果。
