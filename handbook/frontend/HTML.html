    # 2.1 基本概念

    # 2.1.1 HTML概念

    HTML (opens new window)（HyperText Markup Language，超文本标记语言）是一种用来告知浏览器如何组织页面的标记语言。HTML 可复杂、可简单，一切取决于 web 开发者。HTML 由一系列的元素 (opens new window)组成。

    # 2.1.2 HTML 元素组成

    • 常规元素

      一个HTML元素由三部分部分组成:开始标签;内容，结束标签

    • 空元素

      不是所有元素都拥有开始标签、内容和结束标签。一些元素只有一个标签，通常用来在此元素所在位置插入/嵌入一些东西。这些元素被称为空元素 (opens new window)。

      以下是一些常见的空元素

      1. <br>:表示换行，常用于在文本中插入换行符。
      2. <img>：表示图像，用于在HTML页面中插入图像。
      3. <input>：表示输入框，用于在HTML页面中添加表单元素。
      4. <meta>：表示元数据，用于在HTML页面中添加元数据信息。
      5. <link>：表示链接，用于在HTML页面中引入CSS样式表等外部资源。
      6. <hr>：表示水平分割线，用于在HTML页面中插入水平分割线。

    # 2.1.4 HTML中的实体引用

    • 实体引用

      在HTML中，有些字符比如小于号(<)，大于号(>)，引号(")等需要进行转义，否则会被浏览器解析为HTML标签或属性，从而导致页面出错。为了解决这个问题，HTML提供了实体引用的方法，可以将这些字符转换为特定的实体引用。实体引用的语法是以“&”开头，以“;”结尾，中间是实体引用的名称或编号。下面是一些常用的实体引用：

      原义字符 等价字符引用
      < &lt;
      > &gt;
      " &quot;
      ' &apos;
      & &amp;
      (空格) &nbsp;
      版权符号@ &copy;
      注册符号® &reg;

    • 实体引用和转义字符的区别和联系

      实体引用和转义字符都是用来表示特殊字符的方法，但它们有以下区别：

      • 格式不同：实体引用以“&”开头，“;”结尾，例如“<”表示小于号“<”；而转义字符以“\”或“%”等符号开始，例如“\n”表示换行符。
      • 用途不同：实体引用用于XML、HTML等文档中，用来表示特殊字符，例如小于号“<”、大于号“>”等；而转义字符通常用于编程语言中，用来表示特殊字符或转义字符本身，例如换行符“\n”、双引号“\”等。
      • 范围不同：实体引用只能表示已定义的特殊字符或自定义实体引用，而转义字符可以表示任何字符，包括字母、数字和符号等。
      • 易读性不同：实体引用在文本中更易于阅读和理解，而转义字符则需要更多的注意力来识别和理解。

      总之，实体引用用于XML、HTML等文档中表示特殊字符，转义字符用于编程语言中表示特殊字符或转义字符本身。两者虽然有些相似，但在使用场景、格式、用途等方面还是有一定的区别。

    # 2.16 HTML语法特性

    1. HTML对换行不敏感，对tab不敏感

      HTML只在乎标签的嵌套结构，嵌套的关系。谁嵌套了谁，谁被谁嵌套了，和换行、tab无关。换不换行、tab不tab，都不影响页面的结构。

      也就是说，HTML不是依靠缩进来表示嵌套的，而是看标签的嵌套关系。但是，我们发现有良好的缩进，代码更易读。建议大家都正确缩进标签。

    2. 空白折叠现象

      HTML中所有的文字之间，如果有空格、换行、tab都将被折叠为一个空格显示。如何要多个空格使用 &nbsp;

    现在我们将 src 属性从 <video> 标签中移除，转而将它放在几个单独的标签 <source> (opens new window) 当中。在这个例子当中，浏览器将会检查 <source> 标签，并且播放第一个与其自身 codec 相匹配的媒体。你的视频应当包括WebM 和 MP4 两种格式，这两种在目前已经足够支持大多数平台和浏览器。

    每个 <source> 标签页含有一个 type 属性，这个属性是可选的，但是建议你添加上这个属性 — 它包含了视频文件的 MIME types (opens new window) ，同时浏览器也会通过检查这个属性来迅速的跳过那些不支持的格式。如果你没有添加 type 属性，浏览器会尝试加载每一个文件，直到找到一个能正确播放的格式，这样会消耗掉大量的时间和资源。

    video标签常见的属性

    • width (opens new window) 和 height (opens new window)

      ​ 你可以用属性控制视频的尺寸，也可以用 CSS (opens new window) 来控制视频尺寸。无论使用哪种方式，视频都会保持它原始的长宽比 — 也叫做纵横比。如果你设置的尺寸没有保持视频原始长宽比，那么视频边框将会拉伸，而未被视频内容填充的部分，将会显示默认的背景颜色。

    • autoplay (opens new window)

      ​ 这个属性会使音频和视频内容立即播放，即使页面的其他部分还没有加载完全。建议不要应用这个属性在你的网站上，因为用户们会比较反感自动播放的媒体文件。

    • loop (opens new window)

      ​ 这个属性可以让音频或者视频文件循环播放。同样不建议使用，除非有必要。

    • muted (opens new window)

      ​ 这个属性会导致媒体播放时，默认关闭声音。

    • poster (opens new window)

      ​ 这个属性指向了一个图像的 URL，这个图像会在视频播放前显示。通常用于粗略的预览或者广告。

    • preload (opens new window)

      ​ 这个属性被用来缓冲较大的文件，有 3 个值可选： "none" ：不缓冲 "auto" ：页面加载后缓存媒体文件 "metadata" ：仅缓冲文件的元数据

    注意我们并没有使用 autoplay 属性在这个版本的例子中 — 如果当页面一加载就开始播放视频的话，就不会看到 poster 属性的效果了。

      • autoplay，表示允许自动播放视频。
      • clipboard-write，表示允许网站写入用户剪贴板，该功能通常用于将链接或其他内容复制到剪贴板。
      • encrypted-media，表示允许加密的媒体播放，如通过数字版权管理技术（DRM）保护的音频或视频。
      • gyroscope，表示允许允许陀螺仪来控制视频播放，例如支持六轴运动传感器的设备。
      • picture-in-picture，表示允许允许画中画播放，允许视频浮动在文档中并继续播放。

    • allowfullscreen 表示允许全屏选项，即允许用户将视频放大至全屏模式。

    • checked：指定单选框或复选框为选中状态。
    • autofocus：指定页面加载时自动聚焦到指定表单元素上。
    • hidden：隐藏一个元素。
    • multiple：指定选择框的多选模式。
    • spellcheck：指定是否启用拼写检查。
    • draggable：指定元素是否可拖动。
    • contenteditable：指定元素是否可编辑。
    • autoplay:这个属性会使音频和视频内容立即播放，即使页面的其他部分还没有加载完全。建议不要应用这个属性在你的网站上，因为用户们会比较反感自动播放的媒体文件。
    • loop:这个属性可以让音频或者视频文件循环播放。同样不建议使用，除非有必要。
    • controls:用户必须能够控制视频和音频的回放功能。你可以使用 controls 来包含浏览器提供的控件界面，同时你也可以使用合适的 JavaScript API (opens new window) 创建自己的界面。界面中至少要包含开始、停止以及调整音量的功能。
    • defer:延迟执行脚本，即脚本会在页面解析完毕后再执行

    • 使用JavaScript进行表单数据校验

      <!-- novalidate 属性关闭浏览器的自动校验；这允许我们使用脚本控制表单校验。 -->
    <form novalidate>
        <p>
            <label for="mail">
                <span>Please enter an email address:</span>
                <input type="email" id="mail" name="mail">
                <!-- aria-live (en-US) 属性确保我们的自定义错误信息将呈现给所有人，包括使用屏幕阅读器等辅助技术的人。 -->
                <span class="error" aria-live="polite"></span>
            </label>
        </p>
        <button>Submit</button>
    </form>
    <script>
        // 有许多方式可以获取 DOM 节点；在此我们获取表单本身和
        // email 输入框，以及我们将放置错误信息的 span 元素。

        var form = document.getElementsByTagName('form')[0];
        var email = document.getElementById('mail');
        var error = document.querySelector('.error');

        email.addEventListener("input", function (event) {
            // 当用户输入信息时，校验 email 字段
            if (email.validity.valid) {
                // 如果校验通过，清除已显示的错误消息
                error.innerHTML = ""; // 重置消息的内容
                error.className = "error"; // 重置消息的显示状态
            }
        }, false);
        form.addEventListener("submit", function (event) {
            // 当用户提交表单时，校验 email 字段
            if (!email.validity.valid) {

                // 如果校验失败，显示一个自定义错误
                error.innerHTML = "I expect an e-mail, darling!";
                error.className = "error active";
                // 还需要阻止表单提交事件，以取消数据传送
                event.preventDefault();
            }
        }, false);

    </script>
